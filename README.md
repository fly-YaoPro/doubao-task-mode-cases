# 🖥️ doubao-task-mode-cases

> **豆包 PC 端办公任务模式 · 达人选题案例手册**
>
> 不只是案例库。是一套完整的选题设计方法论——从 45 个 Skill 的能力地图，到 131 条真实客户反馈提炼的 12 条提报规范，再到按 Skill 分桶的 60+ 个案例文件。

---

## 这是什么

豆包 PC 端已支持安装 **45 个 Skill**（视觉设计、视频生成、数据看板、网页应用、金融分析、舆情监控、本地电脑操作、飞书办公套件……）。任务模式让这些 Skill 能**定时触发、跨工具编排、上下文持久化**——但"能做什么"和"用户想看什么"之间有巨大的鸿沟。

这个仓库填的就是这道鸿沟。

**对达人**：提供经过客户验证的选题方向和四段式提报模板  
**对案例设计者**：提供从调研→筛选→撰写→入库的完整方法论  
**对豆包团队**：提供 45 个 Skill 的路由/门控/边界速查和组合关系地图

## 核心能力

### 🎯 选题筛选（别什么都写）

五道硬筛 × PC 场景三维 × 7 项客户反馈门控，确保每个选题有传播力和差异化。支持时间窗口过滤（高考/考研/校招/四六级按季节匹配）。

### 🔬 设计前调研（先看别人怎么做）

竞品对标（Claude Artifacts/小红书/抖音爆款基线）+ 人群痛点深挖（知乎/行业论坛真实原声）——双轨道强制前置，不调研不开案。

### 📝 提报规范（12 条可执行清单）

PC-1 反查、Skill 生命周期匹配、同质化预警、雷区过滤、使用对象明确、四段式写法……每条都是从客户驳回中提炼的真金白银。

### 🗺️ 能力地图（45 个 Skill 速查）

每个 Skill：定位一句话、路由表、硬边界、门控要点、内部 lark 依赖、案例目录对应。21 个 lark-* 飞书能力层（不单独做案例，作为交付层/数据层在组合中出现）。

### 🔗 组合关系（Skill 编排弹药库）

video-extract → creative-video、browser-task → creative-design、finance × cron、sentiment-tracker → newmedia-writing……已知的 8 条组合链路，案例设计的最高级玩法。

## 快速开始

**写一条客户提报**：开 [references/客户选题反馈与经验沉淀.md](references/客户选题反馈与经验沉淀.md) → 对照 12 条规范 → 按四段式格式输出

**查一个 Skill 的能力边界**：开 [references/能力地图.md](references/能力地图.md) → 总览速查表定位 → 对应分组查看路由和门控

**设计一个新案例**：开 [SKILL.md](SKILL.md) → 5.0 调研前置 → 4.3 客户反馈门控 → 5.1-5.6 方法论 → 铁律检查

**新 Skill 入库**：能力地图第十二节流程 → 精读 SKILL.md → 补入对应分组 → 按需建案例桶

---

## 维护机制

- 新 Skill 上线 → 遥师告知 → 补入能力地图 + 建空桶
- 新一批客户反馈 → 追加到反馈文档第五节 → 提炼新模式
- 同类型反馈累计 3 条 → 触发 SKILL.md 对应章节修订
- 每季度更新一次时间窗口判断逻辑
