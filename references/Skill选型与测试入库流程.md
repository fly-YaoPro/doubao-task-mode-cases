# Skill选型与测试入库流程

## 七、Skill测试与案例入库流程（新增）

当豆包上线新Skill或更新已有Skill时，执行以下流程判断是否入库案例：

```
发现新Skill/更新 → 读SKILL.md + 全部ref文件
  ↓
识别Skill的核心能力动作用什么？(如"文字→口播视频""数据→网页看板")
  ↓
匹配任务模式的能力支撑(支撑一/二/三)
  ↓
过五道硬筛 + PC场景三维
  ↓
设计2-3个候选案例 → 选最能展示"这个Skill+任务模式"组合差异化的1个
  ↓
  出测试案例文档（造假背景+默认五轮共创脚本+预期路由+关注点）
  ↓
实际测试 → 出测试反馈文档（场景反馈+场景发散）
  ↓
确认入库或标记为[已出测试提示词，待确认]
```

## 提示词模式要求

- 默认按[提示词双模式与普通用户共创流程](提示词双模式与普通用户共创流程.md)输出普通用户五轮对话；
- 预期真值与验收项单独保存，不在第一轮对话中泄露；
- 用户明确要求一段话执行、严格回归测试或高风险口径核对时，追加专家直出提示词；
- 同题对比两个Skill时，前四轮逐字一致，第五轮只替换Skill名称。
