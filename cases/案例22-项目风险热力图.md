## 造假背景

老陈，30人软件公司交付总监，手上同时盯5个外部项目。兼PMO——自己管两个，另外三个手下PM在跑但要盯。每周五下午打开Jira/飞书/微信，手抄关键数据到Excel：进度%、预算%、人力占用、技术风险1-5分。一行行看，黄了的拍PM，红了的打电话。一套流程一个半小时。

但这套流程有盲区：Excel是静态的——看不出一个格子从黄渐变到红的过程。等他看到红的时候已经晚了。上周的数据覆盖式更新后丢了，回头看趋势得翻旧邮件附件。五个项目之间没法一眼对比风险高低。

翻车经历：一个ToB交付项目，验收前两周。老陈每周评技术风险=黄连续四周——但他没注意到这个黄已经黄了一个月。验收前三天技术负责人说底层架构要重写，延期两周赔了8万。如果有张图让他看到这个格子连续三周黄了，他四周前就会介入。

破局：每周五喂数据→visualization渲染交互热力图（横轴=项目/纵轴=维度/色阶=风险等级）。hover看数值+变化幅度，点击格子展开三周趋势折线。自动标注连续三周黄色的高危信号。

## 场景一：首次录入+热力图

提示词：
```
我手上管5个项目，每周五要评估风险。帮我记住这些项目，然后用可视化给我一张交互热力图：横轴5个项目，纵轴4个维度（进度/预算/人力/技术风险），颜色1分=绿→5分=红五级色阶。

本周（7月11日）：
A-智慧园区（招商局，我管）：进度3/预算2/人力3/技术1。前端正常，甲方上周加了3个需求
B-数据中台（万科，小杨）：进度2/预算4/人力2/技术3。预算超15%，技术上了新版Flink我不放心
C-移动App（华润，小赵）：进度4/人力4/预算3/技术2。UI改三版未定，前端等设计稿闲着——进度严重落后
D-OA升级（内部，我管）：进度1/预算1/人力2/技术1。正常
E-电商平台（某品牌，小杨）：进度2/预算2/人力3/技术1。正常

要求：hover每个格子显示项目名/维度/评分/PM/备注。
```

预期：热力图渲染正确、C项目进度和人力两格应为橙色/红色——最扎眼区域

## 场景二：追加历史+趋势展开

提示词：
```
我找到了前两周数据，加上去，然后热力图支持点击格子展开趋势折线。

上周7/4：A(3,2,3,1)/B(2,3,2,3)/C(3,3,3,2)/D(1,1,2,1)/E(2,2,3,1)
上上周6/27：A(3,2,2,1)/B(1,3,2,2)/C(2,2,2,1)/D(2,1,2,1)/E(2,2,3,1)

点任意格子→下方展开三周趋势折线。并帮我标注连续三周≥3分的高危格子。
```

验证：B预算1→2→3→4一路上扬。C进度2→3→4一路下滑。C人力2→3→4。D全绿最不起眼。高危标注至少含B预算/C进度/C人力。

## 模式分析

能力模式七变体：任务模式调度visualization Skill，每周更新交互热力图+趋势线。和app-builder分工：app-builder做独立网页数据看板（CRUD），visualization做对话内交互数据探索（洞察）。这里选visualization因为老陈的需求是发现风险而非管理数据——他要的是探索和洞察。
