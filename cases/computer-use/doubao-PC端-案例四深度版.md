# 案例四（深度版）：连接蓝牙耳机并修复麦克风

> 这个案例的用户表述最模糊、诊断路径最多、最能体现豆包PC端的"动手能力"。设计了三组递进式测试对话，从简单到复杂。

---

## 一、环境准备

### 需要准备的设备/状态

| 设备 | 状态 | 用途 |
|------|------|------|
| 蓝牙耳机1（如Sony WH-1000XM4） | 已配对但**断开连接** | 测试"找到并连接"流程 |
| 蓝牙耳机2（如AirPods） | 已配对且**已连接**但未设为默认音频设备 | 测试"已连接但没声音"的诊断 |
| 笔记本内置麦克风 | 正常工作 | 对比测试 |
| 蓝牙音箱 | 已配对，故意留在附近干扰 | 测试多设备场景下的目标设备识别 |

### 预先制造的问题（选一个作为初始状态）

**问题A（推荐首测）**：蓝牙耳机已连接，但Windows音频输出仍指向"扬声器"
- 制造方法：连接蓝牙耳机后，右键音量图标→声音设置→将输出设备手动切回"扬声器"
- 用户视角：耳机连上了，灯也亮了，但就是没声音

**问题B（进阶测试）**：蓝牙耳机连接正常，音频输出也正确，但麦克风被禁用
- 制造方法：Win+I→系统→声音→输入→管理声音设备→禁用蓝牙耳机的麦克风
- 用户视角：能听不能说，会议时对方听不到我

**问题C（边界测试）**：蓝牙耳机既没声音也没麦克风，且附近有多个蓝牙设备
- 制造方法：同时打开蓝牙音箱、蓝牙耳机，且都不设默认
- 用户视角：不知道哪个设备在响、不知道怎么切换

---

## 二、核心测试对话序列（推荐用问题A起步）

### 【对话轮次1】模糊求助——用户在说不清的状态下发起

**用户说**：
```
我的蓝牙耳机连上了，但是没声音。帮我弄一下。
```

**预期豆包行为**：
```
🔍 正在检查音频状态...

检测结果：
  📡 蓝牙：已开启
  🎧 已配对蓝牙音频设备：
     - WH-1000XM4（状态：已连接 ✓）
     - AirPods（状态：未连接）
     - JBL Flip 5（状态：未连接）
  
  🔊 当前音频输出：扬声器（Realtek）← 问题在这里！
     WH-1000XM4已连接但未被设为默认输出设备
  
  已为你将默认输出切换为：WH-1000XM4 🎧
  请戴上耳机试一下，有声音了吗？
```

**关键验证点**：
- 豆包是否正确识别了问题根因（已连接但未设默认设备）——而不是盲目重装驱动
- 是否列出了所有蓝牙设备让用户确认目标
- 修改后是否给用户验证机会（"有声音了吗"）

---

### 【对话轮次2】用户发现麦也有问题

**用户说**：
```
好了有声音了。但麦克风好像也不行，刚才开会别人说听不到我。
帮我也修一下麦克风。
```

**预期豆包行为**：
```
🎤 正在检测麦克风状态...

当前输入设备：WH-1000XM4 Handsfree

⚠️ 关于蓝牙耳机麦克风的说明：
Windows下蓝牙耳机有两个音频模式：
  🎵 Stereo模式：高音质立体声（听音乐用，麦克风不可用）
  📞 Handsfree模式：单声道（通话用，麦克风可用）
  
目前状态：
  输出：WH-1000XM4 Stereo ✓（听音乐音质好）
  输入：WH-1000XM4 Handsfree ✓（麦克风已启用）

正在测试麦克风...
  🎙️ 开始3秒录音测试...
  ▶️ 播放刚才的录音...

你能听到刚才录的声音吗？

如果录音不清楚，可能原因：
  1. 耳机麦克风距离嘴太远
  2. 环境中风噪/背景音干扰
  3. 笔记本内置麦克风和蓝牙麦克风冲突（可能两个同时开着）
  
当前检测到其他麦克风：
  - 笔记本内置麦克风（Realtek）：已静音，避免冲突 ✓
  
需要我帮你切换回笔记本内置麦克风吗？（音质稳定但需靠近电脑说话）
```

**关键验证点**：
- 豆包是否解释了Stereo和Handsfree的区别（这是蓝牙耳机最常见的用户困惑）
- 是否有麦克风测试流程（录制→播放→确认）
- 是否检查了多个麦克风冲突的问题
- 是否给出了替代方案（笔记本内置麦克风）

---

### 【对话轮次3】用户切换场景

**用户说**：
```
好的麦克风可以用了。
对了，我平时主要用耳机听音乐，开会才用麦克风。
能不能帮我设置成：听音乐自动用Stereo模式，开会自动切到能说话的模式？
```

**预期豆包行为**：
```
💡 这个需求很好，但Windows原生不支持"按应用自动切换蓝牙模式"。

不过我有两个折中方案：

方案一【推荐】：设置快捷键切换
  我可以为你在桌面创建一个快捷方式：
  🎵 "切换音乐模式"（输出=Stereo，输入=笔记本内置麦克风）
  📞 "切换会议模式"（输出=Handsfree，输入=蓝牙耳机麦克风）
  双击即可切换，不需要每次手动去设置。

方案二：设置默认规则
  当前默认值：输出=Stereo，输入=Handsfree
  日常听音乐和开会都可直接使用（开会时声音会降低但可通话）
  如果对音乐音质要求高再手动切。

你想用哪个方案？
```

**关键验证点**：
- 豆包是否诚实地告诉用户"Windows原生不支持"——而不是承诺做不到的事
- 是否主动提出了两个可操作的折中方案
- 方案一（桌面快捷方式切换）是否是真正可执行的操作（创建脚本文件并放在桌面）

---

## 三、边界情况测试（单独测试，不按对话顺序）

### 测试A：用户说"上次还能用，今天突然不行了"

**用户说**：
```
我的蓝牙耳机上次还能用的，今天突然连不上了。
帮我看看怎么回事。
```

**预期豆包行为**：
```
🔍 正在排查蓝牙连接问题...

先做快速检查：
  📡 蓝牙开关：已开启 ✓
  🔋 耳机是否开机/有电：请确认耳机指示灯是否亮起？
  📏 耳机是否在范围内：通常10米以内

如果以上都正常，尝试以下修复：
  1. 🔄 关闭蓝牙→等待5秒→重新开启
  2. 🔄 重启耳机（长按电源键5秒，看到指示灯红蓝交替闪烁）
  3. 🔄 在Windows蓝牙设置中"删除设备"→重新配对

先用方案1试试？（我已帮你关闭了蓝牙，5秒后会自动重新开启）

---（5秒后）---

📡 蓝牙已重新开启
🎧 WH-1000XM4：搜索不到 ← 设备没出现

试试方案2：请重启耳机（长按电源键5秒），重启后告诉我。
```

**关键验证点**：
- 是否按"操作成本从低到高"的顺序排查（重启蓝牙→重启耳机→重新配对）
- 每一步是否给用户清晰的执行指令（"长按5秒看到红蓝交替闪烁"）
- 需要用户动手的部分（按键）和可以远程操作的部分（关蓝牙）是否明确区分

### 测试B：多蓝牙设备环境下的目标识别

**用户说**：
```
我有好几个蓝牙设备，帮我全部设好：
- 耳机 WH-1000XM4：默认听音乐和开会
- 音箱 JBL Flip 5：看视频用
- 不要动 AirPods（那是连手机的）
```

**预期豆包行为**：
```
📡 当前已配对蓝牙设备：

  🎧 WH-1000XM4 → 已设为默认输出设备
  🔊 JBL Flip 5 → 当前未连接，已设为次选
  🎧 AirPods → 已连接（仅用于手机，不修改设置）

⚠️ Windows只能设置一个默认输出设备，不能按使用场景自动切换。
建议方案：
  - WH-1000XM4 设为默认设备（满足大部分场景）
  - JBL Flip 5 保留在设备列表，想用的时候可以快速切换
  （我可以帮你在桌面放一个"切换到音箱"的快捷方式）
  - AirPods 不修改任何设置 ✓

按这个方案执行吗？
```

**关键验证点**：
- 是否尊重用户的"不要动AirPods"指令
- 是否诚实地告知Windows的局限性

## 四、测试通过/不通过的判定标准

| 场景 | 通过 | 不通过 |
|------|------|--------|
| 设备连接但没声音 | 正确识别并修复"默认输出指向扬声器"的问题 | 盲目重装驱动、要求用户重启、建议"换根线"（蓝牙哪有线） |
| 麦克风不可用 | 解释Stereo/Handsfree模式区别+提供测试流程 | 只说"麦克风已启用"但用户实际测试仍不行 |
| 要求自动切换模式 | 诚实告知"Windows不支持"并提供折中方案 | 编造一个不存在的功能或只说"不支持"不给替代方案 |
| 突然连不上 | 按成本从低到高排查（重启蓝牙→重启耳机→重新配对） | 第一步就建议"重新配对"（用户之前的数据/偏好设置会丢失） |
| 多设备管理 | 尊重用户要求+诚实告知系统限制 | 擅自修改被禁用设备、承诺系统做不到的事 |

## 五、比普通对话版本的差异点总结

| 维度 | 之前简版 | 现在深度版 |
|------|---------|-----------|
| 对话轮次 | 1轮（用户说→豆包修好） | 3轮+2个边界测试（模拟真实多轮交互） |
| 问题复杂度 | 单一问题：没声音 | 链式问题：没声音→麦克风不行→想自动切换 |
| 用户表述 | 清晰的 | 模糊的："上次还能用""帮我全部设好" |
| 豆包诊断 | 一步定位 | 多步排查（成本从低到高） |
| 失败路径 | 无 | 有（Windows不支持自动切换→给出折中方案） |
| 边界情况 | 无 | 多设备环境、设备断连重连 |
