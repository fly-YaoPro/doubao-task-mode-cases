## 案例五：交互式打击垫/节奏创作器

### 为什么选这个

在v0社区里，任何"能出声"的AI生成应用都会获得额外关注——因为大家默认AI只能生成静态UI。一个可点击发声的打击垫直接打破这个预期。

### 测试提示词

```
帮我做一个节奏创作器/打击垫。

布局：4×4 共16个打击垫按钮，每个按钮可以绑定一个音色。

音色预设（默认8个音色分配到前两排）：
底鼓、军鼓、踩镲、拍手、碎镲、通鼓（低）、通鼓（高）、铃鼓
后两排空着，用户可以自己加。

交互：
- 点击一个打击垫→发出对应音色（用Web Audio API生成，不用加载外部音频文件）
- 每个垫子有对应颜色（底鼓深红、军鼓浅灰、踩镲金色...）
- 点击时垫子有"按下"动画（scale缩小+颜色变亮，0.1s回弹）
- 底部有一个"录制"按钮：点录制→开始点垫子→再点停止→自动循环播放刚才的节奏
- 录制时可以叠加（第一轮录鼓、第二轮录镲），每轮录音用不同颜色的轨道显示在下方
- "播放/暂停"按钮+速度调节滑块（60-180 BPM）
- "清除"按钮重置所有录音

视觉：
- 垫子用大号圆角方块（80×80px），间距8px，居中排列
- 按下时垫子凹陷感（内阴影+scale），松开回弹
- 已录制的轨道在下方用时间轴小条显示，每条轨道不同颜色
- 整体深色背景（#1a1a2e），垫子用霓虹色系，像Ableton Push或Launchpad的感觉
- 当前播放位置有一条竖线在时间轴上移动

注意：音频用Web Audio API的oscillator生成简单打击音色（不需要加载外部音频文件）。
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **音频生成** | 点击垫子有声音，16个垫子音色不同 | app-builder可能不支持Web Audio API |
| **按下动画** | 点击时scale缩小+颜色变亮+0.1s回弹 | 动画不流畅或没有 |
| **录制循环** | 录制的节奏正确循环播放，可叠加录制 | 时间同步问题、叠加覆盖而非叠加 |
| **速度调节** | 滑块改变BPM，循环速度实时变化 | BPM只变数字不变实际速度 |
| **视觉品质** | 霓虹深色+打击垫网格+Ableton风格 | 白底默认按钮排列 |


## 案例六：活动座位表拖拽编排器

### 为什么选这个

婚礼策划、公司年会、班级聚餐——排座位是让一个人崩溃的活。Excel排座位=二维表格里打字，根本看不出空间关系。拖拽式座位编排器是真正的"AI生成一个你愿意付费的工具"级别的案例。

### 测试背景（造假用）

```
活动：公司2026年年中团建晚宴
桌数：6张圆桌，每桌8人
参加人员：48人（6个部门）

部门人员：
技术部（8人）：张三、李四、王五、赵六、钱七、孙八、周九、吴十
产品部（7人）：陈PD、林PM、黄UX、郑UI、刘DATA、杨GROWTH、朱RESEARCH
设计部（5人）：冯UI、褚UX、卫GRAPHIC、蒋MOTION、沈CREATIVE
市场部（10人）：韩MARKET、杨BD、朱PR、秦CONTENT、许SOCIAL、何SEO、吕ADS、施EVENT、张COPY、孔ANALYTICS
运营部（8人）：曹OPS、严CS、华SUPPORT、金QA、魏LOGISTICS、陶TRAINING、姜ONBOARDING、戚RETENTION
销售部（10人）：谢SALES、邹BD、喻ACCOUNT、柏BDR、水AE、窦SDR、章CLOSER、苏NEGOTIATOR、潘PIPELINE、葛FORECAST

约束：
- 每桌尽量混合不同部门的人（不要同一桌全是技术部）
- 陈PD（产品总监）和谢SALES（销售总监）不要放一桌（之前吵过架）
- 韩MARKET（市场总监）和曹OPS（运营总监）最好放一桌（关系好，能带动气氛）
- 每桌至少有一个女生（已知：黄UX、郑UI、杨GROWTH、秦CONTENT、华SUPPORT、姜ONBOARDING、戚RETENTION、苏NEGOTIATOR）
```

### 测试提示词

```
帮我做一个活动座位表编排工具。

活动：公司年中团建晚宴
桌数：6张圆桌，每桌8人，共48人

功能：
- 左侧显示所有待安排人员列表（按部门分组）
- 右侧显示6张圆桌的俯视图，每桌8个座位（圆形排列）
- 从左侧拖拽人员到右侧某个座位=分配座位
- 已经分配的人从左列表移到对应桌的座位上
- 可以拖拽已分配的人换座位（同桌内换、跨桌换）
- 违反约束时座位或人名显示红色警告（比如：如果谢SALES和陈PD在同一桌，两人名字都变红）
- 每桌显示"已安排 X/8"，满了变绿
- 顶部显示统计：已安排/总共 48人

约束条件（在座位分配时自动检测）：
1. 每桌尽量不同部门混坐（同部门超过3人同桌→黄色警告）
2. 陈PD和谢SALES不在同一桌（同台→红色警告）
3. 韩MARKET和曹OPS最好同一桌（不同桌→蓝色提示）
4. 每桌至少1个女生

预设48人数据（部门+姓名+性别）：
技术部(8)：张三/男、李四/男、王五/男、赵六/男、钱七/男、孙八/男、周九/男、吴十/男
产品部(7)：陈PD/男、林PM/男、黄UX/女、郑UI/女、刘DATA/男、杨GROWTH/女、朱RESEARCH/男
设计部(5)：冯UI/男、褚UX/男、卫GRAPHIC/男、蒋MOTION/男、沈CREATIVE/男
市场部(10)：韩MARKET/男、杨BD/男、朱PR/男、秦CONTENT/女、许SOCIAL/男、何SEO/男、吕ADS/男、施EVENT/男、张COPY/男、孔ANALYTICS/男
运营部(8)：曹OPS/男、严CS/男、华SUPPORT/女、金QA/男、魏LOGISTICS/男、陶TRAINING/男、姜ONBOARDING/女、戚RETENTION/女
销售部(10)：谢SALES/男、邹BD/男、喻ACCOUNT/男、柏BDR/男、水AE/男、窦SDR/男、章CLOSER/男、苏NEGOTIATOR/女、潘PIPELINE/男、葛FORECAST/男

【视觉要求】
- 白色背景，6张圆桌居中排列（2行×3列），每桌之间间距充足
- 每张圆桌画成圆形（直径约200px），8个座位均匀分布在圆周上
- 空座位=浅灰圆点，已分配=彩色圆点（按部门颜色）+人名缩写
- 左侧人员列表用卡片式，部门分组+颜色标签，拖拽时卡片有阴影跟随
- 违反约束的座位/人名闪烁红色边框提醒
- 整体像"婚礼策划软件"的专业感，不是表格
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **拖拽交互** | 左侧拖到右侧座位，能放下 | app-builder可能不支持拖拽 |
| **圆形座位排列** | 8个座位均匀分布在圆周上 | 座位排成一行而非圆形 |
| **约束检测** | 人员冲突时自动标红/标黄 | 约束不生效 |
| **跨桌拖拽** | 已分配的人可以拖到另一桌 | 拖拽跨区域失败 |


## 案例七：个人"Life OS"仪表盘

### 为什么选这个

Notion的Life OS模板常年霸榜——人们喜欢"一个页面看全自己的生活"。但Notion需要自己搭、自己配。如果AI能一分钟生成一个漂亮的个人仪表盘，截图发朋友圈的概率极高。

### 测试提示词

```
帮我做一个个人生活仪表盘（Life OS）。

这是我现在的状态数据：

🕐 今天 2026年6月29日 周一
☀️ 杭州 晴 32°C

📋 今日待办
- 完成Q2绩效自评（截止今天）
- 预约牙科洗牙
- 回复云帆科技王总邮件
- 健身房（腿日）
- 取干洗衣服

📊 本周习惯追踪
冥想：█░░░░ 1/7天
运动：██░░░ 2/7天  
阅读：████░ 5/7天 ✓
喝水：██░░░ 2/7天（日均3杯，目标8杯）

💪 本月健身
已去健身房 8 次，目标 12 次
完成度：66%
力量训练记录：深蹲95kg、卧推70kg、硬拉120kg（本周刷新PR！）

🎯 2026年度目标进度
- 读50本书：21/50（42%）🟢 超前
- 体脂率降到15%：当前18.2%（🟡 需要加速）
- 考下PMP证书：已报名8月考试（🟢 按计划）
- 存款20万：当前8万/20万（🟡 40%，需加速）

💰 本月支出
总计 ¥8,520
餐饮 ¥2,800 | 房租 ¥3,500 | 交通 ¥480 | 娱乐 ¥1,200 | 其他 ¥540

【视觉要求——非常重要】
- 像iOS 17的"健康"App或Things 3的极简美学
- 整体背景用非常浅的暖灰（#f6f6f4），不要纯白
- 每个模块是一张白色圆角卡片（border-radius 16px），有非常subtle的阴影
- 卡片之间间距20px，留白充足
- 标题用深灰（#1c1c1e），正文用中灰（#6e6e73），数字用深色粗体
- 进度条用圆角细条（height:6px），绿色=超前/正常，黄色=需要关注
- 习惯追踪用方形格子（像GitHub contributions的简化版），已完成=实心绿，未完成=空心灰
- 待办事项前有圆形checkbox，完成=绿色填充+对勾+文字删除线
- 顶部日期和天气用大号文字+微妙渐变背景色块
- 整体风格：苹果系统级应用的克制、干净、留白多
- 不要任何emoji做装饰元素，靠排版和留白本身的美感

数据都是示例，但布局要是"真的能用"的。
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **美学品质** | 苹果级克制设计：留白、圆角、微阴影、无多余装饰 | 默认白底+蓝色链接 |
| **习惯追踪** | 方块网格显示7天完成情况 | 变成文字列表 |
| **进度条** | 圆角细条+颜色编码 | 粗进度条或默认样式 |
| **信息层次** | 日期天气>待办>习惯>健身>年度>支出，视觉层次分明 | 所有模块平等排列无主次 |


## 案例八：Secret Santa / 礼物交换抽签器

### 为什么选这个

每年圣诞节、年会、团建都有交换礼物环节——组长在微信群里扔骰子、手动排、检查有没有人抽到自己。一个AI生成的抽签器有动画、有结果保密机制、有历史记录，比微信群投票高级太多。

### 测试提示词

```
帮我做一个Secret Santa礼物交换抽签器。

活动：2026年公司圣诞交换礼物
参与人数：16人

功能流程：
第1步：主持人设置参与人名单（可以手动输入或粘贴名单）
第2步：设置规则：
  - 不能自己抽到自己
  - 不能和去年同一个人（如果有去年的记录可以导入）
  - 预算：100-200元
  - 交换日期：12月24日下午3点
第3步：点"开始抽签"→每个人名字一块虚拟刮刮卡/翻转卡片
第4步：每个参与者点自己的卡片→卡片翻转/刮开→显示"你抽到了XXX！预算100-200元，12月24日交换"
第5步：主持人后台可以看到完整配对表（仅主持人可见）

预设16人：张三、李四、王五、赵六、钱七、孙八、周九、吴十、陈十一、林十二、黄十三、郑十四、刘十五、杨十六、朱十七、秦十八
去年配对（可导入）：张三→李四、王五→赵六、钱七→孙八...（制造一些"不能重复"的约束）

【视觉要求——非常重要】
- 圣诞主题但不俗气：深绿+金色+暖白，不是大红大绿
- 每个人的卡片是一张精致的小卡片（200×120px），正面是金色花纹+名字，背面是"🎄点击揭晓"
- 点击卡片有翻转动画（3D rotateY），翻转后显示配对结果
- 卡片翻转后，背面用漂亮的衬线字体显示配对名字
- 顶部有雪花飘落微动画（纯CSS，不要JS大动画）
- 主持人后台页面简洁，表格显示配对+约束状态（"因去年重复需重抽"用红色标记）
- 整体像一张精致的圣诞贺卡，不是工具页面
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **抽签逻辑** | 随机配对，满足"不抽自己""不重复去年"约束 | 约束不满足或死循环 |
| **翻转动画** | 3D卡片翻转流畅 | 直接切换无动画 |
| **主持人后台** | 完整配对表+约束状态 | 没有后台或所有参与者都能看到 |
| **视觉品质** | 圣诞主题精致不土，翻转卡片有仪式感 | 大红大绿默认色 |


## 案例五~八 吸引力排序

**案例七（Life OS）> 案例五（打击垫）> 案例八（抽签器）> 案例六（座位编排）**

- 案例七截图发朋友圈/小红书级别——每个人都有"我的生活仪表盘"的展示欲
- 案例五打破"AI只能生成静态UI"的预期——能出声的AI应用太少了
- 案例八年会/圣诞刚需，且有翻转动画的趣味性
- 案例六拖拽交互对app-builder挑战最大（可能不支持drag & drop）
