## 过筛

| 筛 | 判断 |
|----|------|
| 第0筛（生产力） | ✅ 宠物医院/猫舍经营活动 |
| 第一筛（对话能做吗） | 单只宠物单次问"Cookie该什么时候打疫苗"对话能做。但20只宠物、疫苗/驱虫/体检三个维度、跨月跨年——对话模式下每次都得重新报一遍宠物信息才能算，没人会这么用 |
| 第二筛（App能做吗） | 有宠物健康管理App，但需要手动录入每只宠物的每次疫苗记录并手动设置提醒——等于把Excel换了个界面 |
| 第三筛（大众在乎吗） | ✅ 养过宠物的都懂——忘打一针从头来，疫苗钱白花+猫狗多挨针，真金白银的痛 |
| 第四筛（数据可获取吗） | ✅ 安全区。用户口述宠物信息+疫苗记录即可，无需登录任何系统 |
| 第五筛（信息源质量） | N/A，无外部信息源依赖 |
| PC-1（物理场景） | ✅ 管理20只宠物看板/录入记录/查看日历——天然坐在电脑前操作 |
| PC-2（信息密度） | ✅ 20只宠物×4个健康维度（疫苗/驱虫/体检/绝育）=80个数据点，手机根本看不全 |
| PC-3（操作复杂度） | ✅ 需要在看板各视图（表格/日历/详情页）之间切换，录入时键盘输入大量信息 |

**对话模式 vs 任务模式差距**：20倍+。对话模式每次都要重新输入宠物信息→手动计算日期→手动写通知话术。任务模式一次建档、跨月跨年自动跟踪、到点推送完整话术。更重要的是：**对话模式下你根本不会记得去问**——"Cookie 还有两天要打第二针了"这个事实不在你脑子里，所以你不会打开对话框。


 ## 造假背景

**机构**："喵星球宠物医院"，深圳南山一家社区宠物医院（虚构），院长小周，执业兽医，手下两个护士。同时经营一个英短猫舍作副业，常年保有15-20只种猫和幼猫。

**现在流程**：医院端——前台小姑娘阿婷用一本A4大本子手写登记，每页一只猫/狗，翻页找到→看上次日期→掰手指算→打电话通知主人。阿婷每个月还要手抄一份"本月到期清单"贴在柜台后面，月底那张纸早就划烂了。猫舍端——小周自己用手机备忘录记，但"记着记着就忘了改"。两家业务的数据完全不通——猫舍种猫的疫苗记录在手机备忘录里，医院客户的疫苗记录在那本A4大本子上，小周两头查。

**血泪教训1**：上个月一只布偶幼猫"团子"，主人从猫舍接走时打了第一针猫三联。按规程21天后打第二针。阿婷漏记了，主人也忘了。第28天主人才想起来问，小周只能说"超了7天，第一针白打了，重头来"——主人花了¥360打了两针，实际只需要¥120一针。主人气炸，在小红书写了篇避雷帖。
**血泪教训1**：两个月前一只布偶幼猫"汤圆"，在其他宠物店打了第一针猫三联。转到小周医院时，主人说"上次好像是一个月前打的"，阿婷当第一针登记。等小周发现不对去查——超了12天，第一针白打。主人花了¥360打了三针（实际只需¥120一针）。主人气炸，在小红书写了篇避雷帖。

**血泪教训2**：三个月前（2026年3月），猫舍种公"年糕"狂犬疫苗3月15日到期，小周备忘录记成3月20日。3月18日年糕食欲骤降，一查才发现抗体可能断了。紧急补打+隔离观察一周，错过了一只母猫的配种窗口。少了一窝小猫 = 少卖¥15000-20000。

**任务模式破局（双线并行）**：

**线一（管理看板·PC端）**：小周用豆包的应用生成功能，口述一段提示词，自动生成一个宠物健康管理看板——所有动物的疫苗/驱虫/体检记录在一张网页表格里，颜色标记紧急程度，日历视图一眼看到本月谁到期。小周在电脑上打开这个看板，录入新宠物、更新疫苗记录、查谁该打针——再也不用翻A4本子和手机备忘录两头跑。

**线二（智能提醒·任务模式）**：任务模式记住所有宠物信息和规则，每天早上推今日待办+自动写好通知话术。小周只需要复制粘贴发给主人，或直接给阿婷让她群发。

 ## 传播素材

**30秒短视频脚本框架**：

0-5秒（痛苦画面）：阿婷翻着翻烂的A4大本子，手指一行行找。小周掏出手机备忘录又放下——医院客户数据在纸本上、猫舍数据在手机里，两头查。旁边电话响了："我家Cookie是不是该打针了？"

5-10秒（加重痛苦）：小周翻手机备忘录——"年糕疫苗到期日我记得是3月20日…"画面切到猫舍，年糕精神不好。旁白："记晚了两天，错过一次配种，少了一窝小猫。"

10-18秒（管理看板·PC端）：小周对着电脑屏幕——一个干净整洁的宠物健康管理看板。所有宠物按到期紧急程度颜色标记，绿色一片、红色两只。他用鼠标点开年糕，疫苗历史一清二楚。旁白："小周用豆包一句话生成了一个管理看板，所有宠物的疫苗驱虫一目了然。"

18-25秒（智能提醒·手机端）：小周手机弹出推送："今日到期：团子（猫三联第二针）→ 通知刘先生。通知文案已生成：'刘先生好～团子该打第二针猫三联啦…'"

25-30秒（成果+价格锚定）：阿婷笑着用手机一键复制话术发给主人。小周在看板上点击"团子"，录入第二针完成——系统自动算出第三针日期。最后定格在看板界面+豆包推送同框。"忘一针=¥360白花。AI帮你记着、帮你管着、连微信文案都写好了。"

**传播一句话**："开宠物医院最怕的不是猫抓——是忘了谁该打针。AI给他生成了一个管理看板，所有宠物一目了然，还每天早上告诉他：今天要通知哪几个主人。"


## 模式分析

这个案例是**能力模式六 × 能力模式八的交叉案例**，在现案例库中没有同类：

1. **模式六（生命周期管理）**：任务模式记住所有宠物信息→自动推算日期→定时提醒+生成通知话术。和案例15（房产中介客户跟进）同模式但人群不同，且"忘打疫苗白花钱"的共情门槛远低于"忘回访客户丢单"。

2. **模式八（应用生成看板）——本案例核心增量**：用doubao-app-builder一句提示词生成可视化管理看板，医生可以在PC上看得清清楚楚——不是一条文字推送，是一个可以操作、可以筛选、颜色标记、日历可视化的完整管理工具。和案例19（电商周报看板）同属模式八但场景完全不同——一个是"数据分析看板"，一个是"数据库管理看板"。

3. **画面感天然优势**：猫狗出镜天然比"40组客户Excel表"和"四平台后台数据"有传播力，30秒短视频可以拍得很可爱，这在案例库中是个空白。

**入库建议**：优先实测。如果场景四（看板生成）和场景三（提醒话术）都过关，这个案例的份量足以作为模式六和模式八交叉的代表案例——它展示了"应用生成做管理看板 + 任务模式做智能提醒"的完整组合，而不是两个孤立的Skill调用。
