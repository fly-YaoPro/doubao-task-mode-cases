## 案例一：交互式粒子生命模拟器（最高吸引力）

### 为什么选这个

在Claude Artifacts社区最火的帖子中，交互式模拟器（粒子系统、Conway生命游戏、流体模拟）是获得"wtf this was generated by AI?"反应最多的类型。视觉冲击力直接、分享传播性强。

### 测试提示词

```
帮我做一个粒子生命模拟器。

画面中间是一个方形画布，里面有几百个彩色小粒子。

规则：
- 粒子有三种颜色：红、绿、蓝，代表不同的"物种"
- 同色粒子靠近时会互相吸引
- 不同色粒子靠近时会轻微排斥
- 粒子数量保持恒定（比如每种颜色各100个）
- 粒子碰到画布边缘会反弹
- 粒子移动越快颜色越亮、越慢颜色越暗

交互：
- 鼠标点击画布会在那个位置生成一批新粒子（随机颜色）
- 鼠标按住拖动会产生一个"引力场"，把周围的粒子拉过来然后甩出去
- 右下角有一个"重置"按钮

视觉：
- 画布背景是深黑色（#0a0a0f），粒子有微弱光晕
- 粒子太小看不到细节，太大像色块——直径3-4px刚好
- 整体氛围像"显微镜下的微生物世界"
```

### 测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **粒子运动** | 几百个粒子同时运动不卡顿、帧率稳定 |
| **颜色规则** | 同色吸引、异色排斥的规则正确执行 |
| **鼠标交互** | 点击生成粒子、拖拽产生引力场 |
| **视觉品质** | 深黑背景+粒子光晕+颜色亮度随速度变化 |

## 案例二：个人年度账单/数据可视化生成器

### 为什么选这个

"年度报告"是每年朋友圈刷屏的内容——网易云年度听歌、支付宝年度账单、微信读书年度阅读。但这些都是大厂做的。**普通人能不能给自己生成一个专属于自己的年度数据可视化页面？**

### 测试背景（造假用）

```
用户：一个喜欢看书的职场人
2026上半年阅读数据：

阅读统计：
- 总共读了 23 本书
- 虚构类 12 本，非虚构类 11 本
- 最厚的书：《原则》592页
- 最快的书：《小王子》2天读完
- 平均每天阅读 42 分钟
- 最喜欢的时间段：晚上10点-12点
- 纸质书 8 本，电子书 15 本

月度趋势：
1月3本、2月2本、3月5本、4月4本、5月6本、6月3本

图书类型分布：
文学小说 8本、商业管理 5本、心理学 4本、历史 3本、科幻 3本

最喜欢的3本：
《人类简史》《1984》《思考，快与慢》

2026下半年目标：再读30本
```

### 测试提示词

```
帮我做一个 "2026上半年阅读报告" 可视化页面。

我的阅读数据：

📚 总览
- 读了23本书（虚构12本+非虚构11本）
- 平均每天读42分钟
- 电子书15本、纸质书8本
- 最厚：《原则》592页 | 最快：《小王子》2天读完
- 最爱时段：晚上10-12点

📈 每月趋势
1月3本、2月2本、3月5本、4月4本、5月6本、6月3本

📊 类型分布
文学小说8本、商业管理5本、心理学4本、历史3本、科幻3本

⭐ Top3
《人类简史》《1984》《思考，快与慢》

🎯 下半年目标：再读30本

【视觉要求——非常重要】
- 像网易云年度听歌报告那种设计感，不是Excel表格
- 每个数据模块是一张独立卡片，向下滚动展示（长页面叙事风格）
- 顶部大标题 "2026 · 我的阅读之旅"，下面一行小字 "上半年的23本书"
- 数据用大号数字展示（如"23本"用48px粗体），搭配简洁图标（书本📚、时钟🕐等emoji）
- 月度趋势用小柱状图（不是整页大图，是嵌在卡片里的小图）
- 类型分布用彩色横条（每种类型一条，长度代表数量）
- 配色温暖：米白背景（#fef9ef）+ 卡片暖白+ 数据强调用深棕/暗红
- 底部有进度条："下半年目标30本，已读完23，完成76%"
- 整体像一本精致的电子杂志，滚动阅读体验

数据不要太多，关键数字突出就行。
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **视觉叙事感** | 长页面滚动，每段一个数据亮点，不是所有数据堆一页 | 把所有数字挤在一屏 |
| **数据可视化** | 小柱状图+横条图嵌入卡片，不是整页大图 | 图表太大占满屏幕，比例失调 |
| **配色品质** | 温暖米白+深棕，有"书籍/纸质"的质感 | 白底黑字默认样 |
| **移动端** | 手机上卡片单列、数字缩小但仍可读 | 48px数字在手机上溢出 |


## 案例三：团队午餐投票/随机选择器（实用+社交趣味）

### 为什么选这个

这是v0社区一个高赞案例的变体。每个团队都有"中午吃什么"的世纪难题——微信群投票太简陋、外面App太重。一个AI生成的有趣投票页，能投票、能随机选、能记录历史选择（避免"上周刚吃过这家"）。

### 测试提示词

```
帮我做一个团队午餐投票器。

功能：
- 预设8-10家附近的餐厅（可以后续添加/删除）
- 每人每天可以投1票（给今天最想吃的餐厅）
- 实时显示每个餐厅的得票数和进度条
- 底部有一个"🎲 随机选择"按钮——在所有没被"上周选过"的餐厅里随机选一个（避免重复）
- 选完后自动记录历史，下周随机时排除上周选过的

餐厅清单（预设）：
兰州拉面、黄焖鸡、沙县小吃、老乡鸡、喜家德水饺、杨国福麻辣烫、麦当劳、711便当、张亮麻辣烫、自选快餐

【视觉要求——非常重要】
- 整体明亮活泼但不幼稚，像Slack/Figma那种"专业但有趣"的感觉
- 每个餐厅一张小卡片，左侧餐厅名+emoji，右侧票数+进度条
- 进度条用渐变色（从暖黄到橙色），票越多越暖
- 顶部显示"🕐 今日投票倒计时 11:30 截止"
- 投过票的卡片边框变为主色（表示"你投了这家"）
- "随机选择"按钮要大、要突出，hover有弹性动画
- 选中的结果用大号弹窗动画展示（像抽奖转盘停下来的感觉）
- 底部有一个小小的历史记录折叠区（"上周我们吃了..."）
```

### 测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **投票交互** | 点击投票后票数+1、进度条增长、卡片高亮 |
| **去重逻辑** | 随机选择时排除上周选过的，再随机 |
| **视觉趣味性** | 进度条渐变、随机弹窗动画、卡片hover效果 |
| **历史记录** | 折叠区正确显示上周选择 |


## 案例四：交互式知识卡片/Flashcard学习工具

### 为什么选这个

学习工具是Claude Artifacts社区另一大类。但传统的闪卡太无聊——这个案例加了一个"记忆曲线"：不是机械翻卡，而是答对/答错自动调整复习频率。

### 测试提示词

```
帮我做一个知识闪卡学习工具。

功能：
- 每张卡片正面是问题，背面是答案
- 点卡片翻转查看答案
- 翻转后出现两个按钮："记住了👍"和"再练一次🔄"
- 点"记住了"的卡片下次出现间隔变长（1天→3天→7天→30天）
- 点"再练一次"的卡片下次出现间隔缩短
- 顶部显示今天的进度（"今日已学 12/50 张"）
- 旁边显示间隔分布：🟢今天36张、🟡3天后8张、🔴7天后6张

预设20张英文单词卡片：ambivalent、ubiquitous、ephemeral、pragmatic、eloquent、resilient、nostalgia、serendipity、ambiguous、benevolent、meticulous、procrastinate、unprecedented、phenomenon、hypothetical、paradigm、juxtapose、unanimous、spontaneous、sustainable

每条卡片内容：英文单词+中文释义+一个例句

【视觉要求】
- 卡片居中，白色卡片+微圆角+投影，翻转有3D翻转动画（不是直接切换）
- 卡片背面用浅蓝底色区别于正面白色
- 顶部进度条+彩色小圆点标识间隔分布
- 整体干净安静，像Anki/Quizlet的感觉
- 底部"今天已经学完！🎉"的完成状态
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **翻转动画** | 3D翻转动画流畅，不是瞬间切换 | 直接切换无动画 |
| **间隔逻辑** | 点"记住"后该卡片排到后面，下次间隔正确 | 间隔计算错误或存不住状态 |
| **进度展示** | 进度条+间隔分布正确更新 | 进度不更新 |
| **完成状态** | 全部学完后显示完成页 | 学完了还能继续翻卡 |


## 四个案例吸引力排序

**案例一（粒子模拟器）> 案例三（午餐投票）> 案例二（阅读报告）> 案例四（闪卡学习）**

- 案例一视觉冲击力最强，最适合作为demo展示——"这是AI生成的你敢信？"
- 案例三最接地气，每个团队都能用——"明天我们团队就用这个选午餐"
- 案例二最有情感共鸣——"我也想要一份自己的年度报告"
- 案例四最实用——但视觉吸引力不如前三
