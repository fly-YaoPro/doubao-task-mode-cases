# 案例十四：独立投资人·每日持仓股日报自动生成

> 拆分自「金融方向案例」（海外版豆包案例的本地化测试集），Skill 路由：doubao-daily-stock

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 投资管理——职业/半职业活动 |
| 第一筛 | 对话能做一次个股日报，但5只持仓股=5次手动对话×每天→不可能坚持 |
| 第二筛 | 无App能记住持仓列表+每天自动跑5只股日报+自动对比持仓表现 |
| 第三筛 | ✅ A股散户2亿+，每天看持仓股发生了什么是最基本需求 |
| PC-1 | ✅ 看盘+阅读报告天然在电脑前 |
| PC-2 | ✅ 5股×7个分析维度=极高信息密度 |

**对话模式 vs 任务模式**：对话=每天手动"帮我做XX股的日报"×5次。任务模式=一次配置持仓→每天收盘后自动跑5只→打包推送。持仓股成本/关注问题持久化，不用每天重复。

### 造假背景

**人物**：同上老徐。30%仓位在AI应用概念（中科创达）、25%在低空经济（亿航智能）、25%在消费电子（立讯精密）、20%在现金。

**困境**：收盘后需要知道每只持仓股今天发生了什么——涨跌原因、资金进出、有没有新公告、技术面有没有变化。以前每天手动翻5只股票的同花顺F10+新闻+公告——至少1小时。有时候出差/开会没看，第二天发现某只票出了利空公告已经跌了3个点。

**造假数据·持仓列表**：

```
持仓1：中科创达（300496.SZ）
- 仓位：30%
- 成本：¥68.50
- 关注问题：AI agent业务进展、智能汽车业务增速、海外客户拓展
- 波动特征：高波动（日均振幅4-6%），受AI概念情绪影响大

持仓2：亿航智能（EH.US，美股）
- 仓位：25%
- 成本：$18.20
- 关注问题：适航证进展、交付量数据、海外市场拓展
- 波动特征：极高波动（日均振幅6-10%），受政策和订单催化驱动

持仓3：立讯精密（002475.SZ）
- 仓位：25%
- 成本：¥32.80
- 关注问题：苹果订单占比变化、汽车电子业务进展、毛利率趋势
- 波动特征：中等波动（日均振幅2-3%），受果链情绪+业绩预期驱动
```

### 场景测试

**预判路由**：doubao-daily-stock → 对话框日报

**为什么匹配**：skill明确支持"单一上市股票的当日/近期个股日报"，覆盖行情表现、涨跌原因、资金变化、新闻公告、板块联动、技术面、预期与风险——完全匹配持仓股日报需求。

**提示词**：

```
帮我创建每日持仓股日报自动生成任务：

【路由】doubao-daily-stock（对每只持仓股独立调用）

【持久化持仓信息】（一次配置，长期复用，可随时增删改）
持仓1：中科创达 300496.SZ | 仓位30% | 成本¥68.50
- 关注：AI agent业务进展 / 智能汽车增速 / 海外客户

持仓2：亿航智能 EH.US（美股）| 仓位25% | 成本$18.20
- 关注：适航证进展 / 交付量 / 海外市场

持仓3：立讯精密 002475.SZ | 仓位25% | 成本¥32.80
- 关注：苹果订单占比 / 汽车电子进展 / 毛利率趋势

【触发】每个交易日收盘后30分钟（A股15:30，美股次日北京时间早6:00）

【A股执行时间】：15:30
【美股执行时间】：次日北京时间6:00（对应美东时间前一日16:00收盘后）

【执行流程】

Step 1 · 对每只持仓股独立调用doubao-daily-stock
每只股产出对话版日报，包含：

核心结论（3-5点，第一点直接回答"今天为什么涨跌"）
一、今日行情、量能与资金流向
二、板块联动与个股地位
三、技术面分析（MA/MACD/RSI/压力支撑位）
四、消息面与隔夜观察（最近3日新闻/公告/催化）
五、投行一致预期（如有覆盖）
六、综合评估（结合用户关注问题的针对性分析）
七、短期风险提示（3-5条，至少1条直接对应用户关注问题）

Step 2 · 生成持仓组合摘要
在所有个股日报之前，先推送一份组合层面摘要：

```
📊 {日期} 持仓日报

组合总况：
- 今日组合涨跌：+1.2%（跑赢沪深300的+0.3%）
- 贡献最大：中科创达 +3.5%
- 拖累最大：无

个股概览（按涨跌排序）：
🟢 中科创达 +3.5% | ¥71.20（距成本+3.9%）| AI agent新客户签约催化
🟢 立讯精密 +0.8% | ¥33.10（距成本+0.9%）| 苹果MR供应链消息
🔴 亿航智能 -2.1% | $17.82（距成本-2.1%）| 适航证预期推迟

重点关注：
⚠ 亿航智能跌破成本线→下方支撑$17.50（前低）
📌 中科创达接近前高¥73→如果放量突破加仓窗口打开
```

Step 3 · 打包推送
- 先推组合摘要
- 再逐只推个股日报（每只一条消息，方便单独查看）
- 推送到飞书"老徐·持仓管理"

【需要记住的】
- 持仓列表+每只的成本/仓位/关注问题
- 每只股的历史日报存档（方便回溯"一周前那根大阴线是什么原因"）
- 技术面关键位（前高/前低/支撑/压力）——跨日追踪是否突破/跌破

【异常处理】
- 如果某只股停牌→日报中标注停牌+原因，不生成行情分析
- 如果某只股数据源缺失（如美股某些字段不可得）→标注"暂无"，不编造
- 如果当日为非交易日→跳过
- 如果某只股触发关注信号（如跌破成本线5%以上/放量突破前高）→额外推一条红色预警

【合规】
- 日报中不包含"建议买入/卖出/加仓/减仓"指令
- 技术面只做分析，不给具体操作点位（如"跌破XX元应止损"改为"下方关键支撑在XX元"）
- 每份日报开头固定风险提示
```

**测试关注点**：

| 关注项 | 描述 | 风险 |
|--------|------|------|
| 行情数据准确性 | 收盘价/涨跌幅/成交额/换手率是否正确（skill硬要求：必须交叉核验2-3源） | 🔴 |
| 涨跌原因归因 | "今天为什么涨跌"的因果逻辑是否成立（不是"板块涨了所以它也涨了"的循环论证） | 🔴 |
| 资金流数据 | A股主力/北向资金数据是否准确；美股不套用A股资金表格 | 🟡 |
| 技术面指标 | MA/MACD/RSI数值是否正确（skill要求优先取权威源披露值） | 🟡 |
| 新闻/公告时效 | 是否覆盖了最近3日内的关键公告（不是上周的旧闻） | 🟡 |
| 用户关注问题回应 | 每只股的"综合评估"是否针对用户设置的关注问题（skill的question_focus机制） | 🟡 |
| 跨日一致性 | 今天的技术面关键位是否和昨天日报里的衔接 | 🟢 |
| 合规表述 | 有没有出现"建议买入/卖出"等违规措辞 | 🔴 |

### 场景发散

- **基金研究员**：覆盖池30只股票→每只出周报而非日报（日频太密），每周一出上周周报打包
- **雪球/富途大V**：自选股日报→自动变成每日发帖素材（需要额外做文案转口语化）
- **上市公司IR**：自动监控同行竞品3-5家的每日动态→每天早上出竞品动态简报
- **港美股投资者**：同时监控港股+美股持仓→注意时区差异（港股收盘16:00，美股收盘次日4:00）
