## 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 电商运营优化——主图点击率直接决定流量和转化 |
| 第一筛 | 对话能做一张主图，但无法自动轮换+自动读数据+自动对比+自动出周报 |
| 第二筛 | 无App能自动生成主图变体→替换→追踪数据→统计判断 |
| 第三筛 | ✅ 所有淘系卖家都懂——主图点击率差1%，月销差几千块 |
| PC-1 | ✅ 天猫后台+数据看板在电脑端 |
| PC-2 | ✅ 多SKU×多版主图×多指标=高信息密度 |
| PC-3 | ✅ 需在店铺后台、数据工具、设计工具之间切换 |

**对话模式 vs 任务模式**：对话=手动做两张图→手动替换→一周后手动翻后台找数据→手动对比（可能忘了）。任务模式=每周自动生成→自动轮换计划→自动读数据→自动出分析周报。差距不在"能不能做"，而在"有没有人真的坚持每周做"。


## 造假背景

**公司**："云启电器专营店"（虚构），天猫家电代理商。代理3个品牌线——轻量吸尘器、净水器、加湿器，月销2000万。

**人物**：小林，运营主管。手下1个助理，管3个品牌×约40个SKU的店铺运营。

**困境**：吸尘器线的主力SKU"轻量无线吸尘器M200"（月销2000+台）的主图用了快半年——白底+产品+一行字"15KPa大吸力·45分钟续航"。她一直怀疑这张图还能优化：
- 要不要试试"猫毛场景"？这款吸尘器的核心用户其实是养猫的——吸沙发猫毛巨好用
- 要不要把"15KPa"改成更感性的说法？大部分人不知道15KPa是什么概念
- 要不要加个对比数据？"吸力保留率96%"听起来比"15KPa"更可信

但这些想法永远是想法——因为她没有时间一张张做图、一张张替换、一周后看数据对比。主图优化=要专门空出一个下午来做——而她已经三个月没有"一个空出来的下午"了。

**造假数据·M200产品信息**：

```
SKU：M200
品类：轻量无线手持吸尘器
现主图（已用6个月）：白底+产品居中+"15KPa大吸力·45分钟续航"
现主图点击率：3.2%（行业均值3.5%，低于平均）
现主图转化率：4.1%
现主图加购率：8.7%
月销：2000+台
售价：¥599

核心用户画像：25-35岁女性，养猫/养狗，租房/小户型
真实核心卖点（从用户评价挖掘）：
- 吸猫毛/狗毛巨好用（评论区出现频率最高的词："猫毛""狗毛""毛发"）
- 轻——单手可以举起来吸窗帘（1.2kg）
- 无线——不用到处找插座，充满电能吸全屋
- 吸力不衰减——用半年了吸力和刚买的时候一样（评论区高频提及）
- 好清洗——尘杯一按就开，不脏手

潜在可测试的主图方向：
- 方向A：猫毛场景（击中核心用户痛点）
- 方向B：吸力数据可视化（建立参数信任）
- 方向C：轻量化场景（单手举高吸窗帘）

需要避免的：
- 不能出现价格（天猫主图规则）
- 不能出现虚假对比数据
- 不能出现竞品名称
```


## 造假素材准备

### 基础产品图（用doubao-creative-design生成）

需要一张M200吸尘器白底图作为主图变体的基础参考：

```
一台轻量无线手持吸尘器的白底产品图。

产品外观：主机是深灰色磨砂塑料+银色金属连接杆+白色集尘杯。整体造型现代简约，主机部分呈圆柱形（直径约8cm），手柄在顶部，集尘杯在下方透明可见（能看到里面没有灰尘的干净状态）。连接杆是银色铝合金，底部电动刷头是灰色+白色滚刷。刷头可以看到红色的软绒刷条。

尺寸比例：手持主机+连接杆+刷头=全长约115cm。产品竖直站立，刷头在底部，主机在顶部。

光线：柔光箱从左上角打光，深灰磨砂表面有细腻高光，金属杆有银色反光，透明集尘杯有轻微折射光。

专业家电白底图，淘宝主图标准，1:1比例。产品占画面85%。画面极简干净，没有任何背景元素。
```

### 变体A·场景图原料（猫毛场景）

需要一张"吸尘器在吸沙发猫毛"的场景图，作为变体A的底图。用以下提示词生成：

```
一台手持无线吸尘器正在清洁一张浅灰色布艺沙发。沙发上散落着明显的猫毛（浅橘色猫毛在灰色布面上很显眼），吸尘器的刷头刚好推到猫毛密集处，刷头前侧已经干净了，后侧还散落着猫毛——一个清晰的清洁前/清洁中对比。

视角：45°俯拍，能看到沙发坐垫的全景+吸尘器在工作中的角度。沙发旁边有一团阳光从窗户打进来，猫毛在阳光下特别明显。

吸尘器产品外观（保持和M200一致）：深灰磨砂主机+银色金属杆+白色透明集尘杯+灰色红边刷头。

风格：真实家居场景——沙发上有轻微褶皱靠垫，茶几边缘露出一角，地上有一双毛绒拖鞋。像真实用户在自家拍的，不像摆拍的广告。暖调自然光，画面温暖有生活气息。3:4比例。
```

### 变体B·数据图底图

变体B用白底图+叠加数据图表即可，基础图就用第一节的白底产品图，不需要额外生成。


## 两版主图详细设计

### 变体A·猫毛场景痛点型

**设计思路**：放弃"15KPa"这种用户看不懂的参数，改用"猫毛"这个养猫人一眼就懂的痛点词。画面从白底产品照换成真实场景，让养猫用户刷到主图时说"这不就是我家吗"。

**生成提示词**：

```
基于上传的猫毛场景图，用I2I生成一张淘宝主图（1:1）。

保留项：
- 保留吸尘器产品外观——深灰磨砂主机、银色金属杆、白色集尘杯、灰色红边刷头不变
- 保留沙发猫毛场景——浅灰布艺沙发上的橘色猫毛、清洁前/清洁中的对比
- 保留暖调自然光+阳光光斑

编辑项：
- 在画面上方1/3处叠加主卖点文字：白色粗体"养猫家庭的救星"，加轻微的半透明深灰底色条
- 在画面下方1/5处叠加辅助卖点：白色细体"无线轻量·每天吸出一只猫"
- 右下角极小的品牌角标"云启电器"
- 整体画面提亮10%，猫毛部分加一点锐度让毛发更清晰

风格：真实家居+商业主图之间的平衡——真实到让人共鸣，专业到让人想点。1:1淘宝主图规范。不要价格、不要过度促销感。
```

**预期点击率逻辑**：养猫用户在搜索"吸尘器"时，搜索结果页20张主图都是白底+产品——只有这一张是熟悉的"沙发猫毛"场景。场景认同→点击率预期高于白底图1-2个百分点。

### 变体B·吸力数据信任型

**设计思路**：既然M200的用户都在评论区说"吸力不衰减"，那就把这个用户证言变成可见的数据对比图。不是"15KPa"这种冷冰冰的参数，而是"新机vs使用半年后吸力对比"这种用户语言的可视化。

**生成提示词**：

```
基于上传的M200吸尘器白底产品图，用I2I生成一张淘宝主图（1:1）。

保留项：
- 保留吸尘器产品外观（深灰磨砂+银色金属杆+白色集尘杯+灰色红边刷头）
- 保留产品居中在画面左侧1/3处的构图
- 保留白底

编辑项：
- 画面右侧2/3区域：一个简洁的数据对比柱状图
  柱A（左侧，深灰色）：标注"新机吸力 100%"
  柱B（右侧，品牌色橙色#E8731A）：标注"使用半年后 96%"
  柱B高度大约是柱A的96%——视觉上几乎一样高
- 柱状图上方：白色粗体"吸力不衰减，半年用下来和新的一样"
- 柱状图下方小字：白色细体"真实用户追踪测试·3000转无刷电机"
- 左下角极小的品牌角标"云启电器"
- 整体画面的白底均匀提亮到纯白#FFFFFF

风格：家电+科技感、干净、数据可视化。像一张"拆解给你看"的产品说明图。1:1淘宝主图规范。不要价格、不要过度促销。
```

**预期点击率逻辑**：家电用户最怕"买回去用几个月就没吸力了"——变体B直接把这个担忧变成可视化证据（96%几乎不衰减）。参数信任→点击率预期高于纯参数图。

### 现主图·对照组

```

点击率：3.2%
```

## 任务模式配置提示词

```
帮我创建主图AB测试自动轮换任务：

【路由】doubao-creative-design → ecommerce-design

【持久化产品信息】（一次配置，长期复用）
- 店铺：云启电器专营店·天猫
- 主力测试SKU：轻量无线吸尘器M200
- 现主图：白底+产品+"15KPa大吸力·45分钟续航"（已用6个月，点击率3.2%）
- 现主图参考：已上传M200白底产品图
- 核心用户：25-35岁女性，养猫/养狗，租房/小户型
- 需要避免：价格（天猫主图规则）、竞品名、虚假认证

【触发】每周一早8:00

【每周执行流程】

Step 1 · 生成主图变体（每周一出两张新方向）

调用image_gen或image_edit生成两张新主图变体（方向轮换，避免重复上周方向）：

本周方向一（三周轮换一次）：
- 场景痛点型（猫毛/养宠）：产品在真实场景+痛点文案
- 数据信任型（吸力/续航）：白底+数据可视化图表+信任文案
- 轻量场景型（单手举高/窗帘）：产品在举高使用场景+轻量文案

本周方向二（同上逻辑，和方向一不同）：
- 自动避开上周已测试过的方向

每张主图规范：
- 1:1淘宝主图
- 产品外观基于M200白底参考图锁定（不重绘产品）
- 天猫主图规则：无价格、无虚假认证、文字清晰可读
- 品牌角标"云启电器"在左下角

Step 2 · 制定轮换计划

本周7天分配：
- 周一全天：现主图（对照组，收集基准数据）
- 周二-周四全天：变体A（3天，确保足够样本量）
- 周五-周日全天：变体B（3天）

→ 推送飞书"云启·运营群"：
  "本周M200主图AB测试计划已生成：
   周一·对照组（现主图）
   周二-四·变体A「猫毛场景」
   周五-日·变体B「吸力数据图」
   请确认后我帮你替换主图"

Step 3 · 提醒替换

每天推送一条提醒到飞书：
- 周一早8:00："今天M200主图用对照组（现主图），请确认已替换"
- 周二早8:00："今天M200主图切换为变体A「猫毛场景」，请确认已替换"
- ……

（注：如果任务模式支持浏览器操作→可改为"自动通过天猫后台替换"。当前先做"提醒手动替换"版。）

Step 4 · 周日生成测试周报

每周日晚8:00：
1. 从后台读取本周各版主图的曝光/点击/转化/加购数据
   （如果自动读取不可用→推送"请手动输入本周数据"并提供填写模板）
2. 生成分析周报：

【周报结构】
- 数据总览表：三版主图×四个指标（曝光/点击率/转化率/加购率）
- 核心发现：点击率最高的是哪个、转化率最高的是哪个、差异是否显著
- 统计判断：如果某版连续两周显著领先→建议"将该版设为默认主图，结束测试"
- 下周建议：保留本轮测试中最好的方向，替换表现差的方向
- 历史记录：附上本周之前的所有测试结论（避免重复测已验证失败的方案）

3. 推送飞书："M200主图AB测试第{N}周周报"

【需要记住的】（跨周上下文持久化）
- 已测试方向的历史记录——某方向如果上周已经测过且表现差→本周不重复
- 当前主力SKU的产品信息和用户画像
- 历史测试的统计结论（"场景型主图在M200上始终优于参数型"之类的长期发现）
- 品牌视觉参数（品牌色#E8731A橙色不能变成其他橙）

【异常处理】
- 如果某周数据量不足（曝光<10000/版）→自动延长本周为两周测试期
- 如果某版主图点击率明显异常（低于1%或高于10%）→标记异常并提醒排查"可能是图片加载失败或数据统计错误"
- 如果连续三周某个方向都输给对照组→不再生成这个方向
- 如果产品图参考图在上下文中丢失→提醒我重新上传M200白底图
```

**测试关注点**：

| 关注项 | 描述 | 风险 |
|--------|------|------|
| 主图创意差异化 | 每周生成的两张变体是否和现主图有明显差异（不是换了个字体的同一张图） | 🔴 |
| 产品外观保真 | 变体A/B中的吸尘器外观是否与M200白底参考图一致 | 🔴 |
| 方向轮换 | 每周的两个测试方向是否避免了上周已测过的方向 | 🟡 |
| 中文文字渲染 | 主图上的卖点文案是否清晰可读、无乱码 | 🔴 |
| 数据读取 | 能否从后台读取上周的点击率等数据（或手动输入模板的可用性） | 🟡 |
| 周报统计判断 | "差异显著"的判断是否合理（不能单看数字大小，要有统计意识） | 🟡 |
| 长期记忆 | 是否记住了之前测过的失败方案→不再重复 | 🟢 |
| 天猫规则合规 | 生成的主图是否出现了价格、虚假认证等违规内容 | 🟡 |
