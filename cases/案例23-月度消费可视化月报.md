## 造假背景

小陈，26岁，北京朝阳区互联网公司程序员。月薪1万8，北漂三年。租在通州，通勤一小时。每个月感觉自己没花什么钱但月底一看——卡里没了。他试过记账App，坚持了三个月放弃了。不是不想管，是每次对账太麻烦——微信账单、支付宝账单、银行卡流水，三边数据格式不一样，手动整理到Excel再画图，三个小时过去了。

最烦的是对比——这个月外卖是不是比上个月多了？双十一消费占了多少？不知道。Excel里的图是死的，看不了细节。

任务模式破局：每月1号早上8点——小陈把上月三边消费数据粘贴过来→豆包自动分类、算占比、环比对比、标出异常→渲染一张交互消费报告（hover看每一笔明细、点图例筛选分类、切换月对比）。小陈喝咖啡翻三分钟，这个月的财务状况全清楚了。

## 场景一：首次设置=月度消费报告

提示词：
```
帮我创建一个每月定时任务，每月1号早上8点自动执行。

我是个人做消费管理。每月我会把上月三笔消费数据粘贴给你——帮我自动生成一份交互消费报告。

先把6月数据记住，给我出一张报告看看效果：
餐饮：外卖1260+食堂480+聚餐650+咖啡320=2710
交通：地铁200+打车380+共享单车35=615
购物：优衣库450+小米耳机299+日用品180=929
住房：房租3200+水电285+网费99=3584
娱乐：Steam198+电影120+健身房199=517
人情：同事婚礼红包500+父母保健品320=820
其他：快递60+理发78=138
总计：9313

报告要求：
- 一张环形图看大类占比
- 一张柱状图看每个大类里的小类明细
- hover每个柱子/扇区能看到具体金额
- 自动标记异常：哪项占比超过上月50%就标红
- 最后写一段总结：这个月总支出多少，比上月涨了/跌了多少，最大的开销是什么，哪些分类变化最大
```

验证点：环形图占比正确（住房39%最大、餐饮29%第二）、柱状图外卖1260在餐饮里最高、有5月对比数据的自动算环比

## 场景二：次月追加+趋势对比

提示词：
```
7月数据来了，帮我追加到报告里，同时生成一张月度趋势折线图：

7月：
餐饮：外卖980+食堂510+聚餐1200+咖啡280 = 2970
交通：地铁200+打车560+共享单车40 = 800
购物：京东618买的机械键盘799+日用品155 = 954
住房：房租3200+水电305+网费99 = 3604
娱乐：健身房199+朋友KTV200 = 399
人情：0
其他：理发78+干洗120 = 198
总计：8925

新增要求：
- 在报告顶部加一张月度总支出折线图（5月7789→6月9313→7月8925）
- 每个大类下面加一张迷你折线图看三个月的趋势
- 自动标注：餐饮类连续两个月上涨、交通类7月涨幅过40%
```

验证点：折线图数据正确、交通7月800比6月615涨了30%应标注、餐饮连续涨应标注、7月总支出比6月回调

## 场景三：部分更新

提示词：
```
刚发现7月数据里漏了一笔——给女朋友买了生日礼物¥680。帮我更新7月报告，所有图表和总结自动刷新。
```

验证点：7月总计自动更新为8925+680=9605、购物柱状图追加"礼物¥680"、总结中的最大开销重新计算、趋势折线图7月数据更新
## 测试关注点

| 关注项 | 描述 | 优先级 |
|--------|------|--------|
| 多级分类聚合 | 小类→大类聚合是否正确（外卖+食堂+聚餐+咖啡=餐饮2710） | P0 |
| 环形图/柱状图数据正确 | 占比计算、金额标注、hover信息 | P0 |
| 环比对比 | 各类跟5月比涨跌幅是否正确计算 | P0 |
| 异常自动标注 | 交通涨幅过40%、外卖涨了30%以上应标红 | P0 |
| 趋势折线图 | 追加7月后三月的总支出折线是否正确 | P0 |
| 部分数据更新 | 追加一笔购物后所有图表自动刷新 | P1 |
| 跨月持久化 | 关闭后下月1号任务自动执行、历史数据都在 | P0 |
| 总结文案质量 | 不是"您的本月支出为9313元"——要像朋友在帮你分析钱花哪了 | P1 |

## 传播素材

30秒脚本：小陈月底对着微信/支付宝账单截图在Excel里手动加加减减→手机推送：豆包自动生成的交互消费报告——环形图、柱状图、趋势折线、每个柱子hover看明细→他点开"餐饮"往下挖：外卖1260、咖啡320、聚餐650→自动标注："餐饮连续两个月上涨，比5月涨了15%，外卖是主因"→小陈心想：好，下个月少点外卖。

传播一句话："每个月对账单的噩梦，AI三分钟帮你理清楚——花了多少、花在哪、比上个月涨了还是跌了。"

## 模式分析

能力模式七变体：任务模式调度visualization Skill，每月1号自动生成交互消费报告。和现案例库中案例16-18（creative-video）同模式、不同Skill——展示visualization在任务模式中的价值。

独特价值：
1. 人群极度广泛——每个人都要管钱但没人愿意做图表。比宠物医院、婚礼策划的受众更广
2. 月度节奏天然适合任务模式——每月1号自动触发，是任务模式"定时调度"最直观的展示
3. 交互探索是核心——hover看明细、点图例筛选分类、切换月度对比——Excel截图做不到
4. 从发现异常到行为改变的闭环——"哦原来外卖涨了这么多，下个月少点——这是真正有用的AI"

入库建议：能力模式七新增案例。因受众极广且月度节奏天然适合传播，建议优先实测。
