# 豆包任务模式案例 · 婚礼策划全流程节点提醒

> 设计日期：2026-07-08 | 能力模式六 | 端侧：双端 | 待实测

## 过筛

| 筛 | 判断 |
|----|------|
| 第0筛（生产力） | 婚礼策划师经营活动 |
| 第一筛（对话能做吗） | 单场单次问对话能做。但6-8场同时跑、每场30+节点——没人每天早上打开对话框手动问 |
| 第二筛（App能做吗） | 有项目管理App，但婚礼节点有强时间关联（婚期变动→所有节点联动），通用工具不会识别链式依赖 |
| 第三筛（大众在乎吗） | 忘了一个蛋糕订单，新娘在婚礼上哭了——传播力天然拉满 |
| 第四筛（数据可获取吗） | 安全区。策划师口述+提案文档 |

## 造假背景

小鹿，鹿屿婚礼工作室独立策划师，从业四年。手上同时跟6-8场婚礼。每接一场用Notion写策划提案——所有节点、供应商联系方式全在里面。但每天早上要在6-8份提案间翻来翻去，手抄今天要干什么到便利贴上。

**翻车经历**：去年一场婚礼，蛋糕店老板是熟人，口头约了婚礼前一周再确认。小鹿记在脑子里，但那周另一场撞期忙忘了。婚礼当天下午新娘打电话：蛋糕呢？——蛋糕店压根没接到确认通知。最后三倍价钱临时做，新娘哭了。

**任务模式破局**：把策划提案里的节点全部喂给任务模式→每天早上自动一份清单。不用翻文档，不用手抄便利贴。说一句XX做完了，后续节点自动更新。

 ## 场景一：一次性录入

 提示词：

 ```
 帮我创建一个每天定时任务，管理我手上所有婚礼的节点。我是婚礼策划师。

 先记下第一场：

 陈昊 & 林雨薇 | 婚期：2026-07-25 | 场地：栖岸花园酒店

 节点清单：
 - 化妆师试妆：7/7截止（已超期！联系人Nina，备选CC/139xxxx5678）
 - 花艺配色方案确认：7/9截止（未完成，需发给山野花舍）
 - 请柬寄出：7/10截止（印刻Studio出终稿后送印寄出）
 - 蛋糕款式确认：7/15截止（小野蛋糕，老板是朋友——必须提前3天提醒我！）
 - 最终宾客人数确认：7/18截止
 - 座位表完成：7/20截止
 - 婚车预订：7/20截止（目标：白奔驰S×1+奥迪A6×3）
 - 酒店彩排：7/23周四晚7点（栖岸花园，约主持人+新人+双方父母）
 - 花艺进场布置：7/24周五下午（山野花舍来搭，全天盯场地）
 - 婚礼当天：7/25

 以后每天早上8:30自动执行：检查所有婚礼的节点，按以下格式输出：

 今天必须处理（超期+今天到期+明天到期）：
 ——每项标注：节点名 / 截止日 / 超期几天或还有几天 / 关联联系人或备注

 接下来7天到期：
 ——同上格式

 接下来14天到期：
 ——同上格式

 对所有"今天必须处理"的项目，自动拟一段发给对应联系人的微信——口语化、像策划师在说话、不是模板。比如催Nina试妆："Nina姐～陈昊雨薇25号那场，化妆试妆我们该约了，超两天了，新人有点着急。你看这周哪天方便？"

 更新方式：我说"XX做完了"，你自动打钩。婚期变动时所有节点联动调整。
 ```

 验证点：明天早上8:30推送应包含——化妆师试妆（已超期🔴+催Nina的话术）、花艺配色（明天到期🟠）

 ### 场景三：日常更新节点

 提示词："陈昊婚礼的花艺配色完成了，今天发给山野花舍确认好了。还有王磊婚礼的迎宾水牌印刻回复了——明天出。"

 预期：花艺配色自动打钩下次不再出现；迎宾水牌状态更新为"明天出"但明天仍需提醒确认收到。只更新提到的节点，不动其他。


 ## 测试关注点

 | 关注项 | 描述 | 优先级 |
 |--------|------|--------|
 | 多场并行排序 | 两场婚礼节点是否按紧急程度统一排序而非分场次堆叠 | P0 |
 | 催促进度话术质量 | 口语化、自然、带上下文——不是"尊敬的XX您好" | P0 |
 | 节点更新联动 | 只更新提到的节点，不动其他。花艺配色完成≠花艺进场消失 | P0 |
 | 风险节点标注 | 口头约定、容易忘的节点是否在日报中被特别高亮 | P1 |
 | 婚期变更联动 | 改婚期→所有截止日按新日期自动推算 | P1 |
 | 任务持久化 | 关闭后第二天早上任务是否自动执行 | P0 |


 ## 传播素材

 30秒脚本：小鹿在6个Notion页面间切来切去→闪回去年蛋糕没到的下午新娘哭了→早上8:30豆包推送日报每条附微信话术→小鹿一键复制→陈昊婚礼当天一切顺利。"6场婚礼、180个节点。AI每天早上告诉你今天该干什么。"

 传播一句话："婚礼策划师最怕的不是难缠的新人——是忘了哪个供应商还没确认。AI每天早上告诉她：今天该催谁、该确认什么、连微信话术都写好了。"


 ## 模式分析

 纯能力模式六案例。和宠物医院（案例20）互补——宠物医院是长周期低频，婚礼是短周期高频密集。独特价值在于紧迫感天然更强：宠物疫苗忘打可以补，婚礼蛋糕忘订是不可逆的事故。

 入库建议：可入库作为能力模式六代表案例，和案例15（房产中介客户跟进）、案例20（宠物医院）形成"客户跟进 vs 个人服务 vs 项目节点管理"三维覆盖。
