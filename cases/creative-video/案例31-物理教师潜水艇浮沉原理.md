## 一句话案例和痛点

**一句话**：物理老师告诉豆包"我要做一个潜水艇浮沉原理的 60 秒教学视频"，豆包先诊断当前产物的三类错误（分镜缺失、事实不锁、优化方向跑偏），再分镜、逐镜写结构化提示词、转视频、配旁白，输出一段可直接用于课堂的科普短片。

**普通人痛点**：浮力和重力是"看不见的力"，学生靠文字和静态图理解不了水舱进排水→重量变化→浮沉的因果关系。老师即使用了 AI 生视频，出来的东西要么箭头画反了、要么水舱机制搞错了、要么全程推拉镜头没有节奏——因为达人把整条链路塞进了一条 prompt，没有分镜、没有锁定物理事实、没有逐镜验证。

## 达人"好儿"原版问题诊断

本节是案例设计者对达人原视频的诊断，**不喂给豆包**——仅用于验证"豆包是否能在第三轮独立诊断出同类问题"。

**原版做法**：
- 一条 prompt 包含全部 15 秒内容："生成一个潜水艇浮沉原理教学动画视频，0-3 秒出现三艘潜水艇……12-15 秒排水上浮……"
- 没有先分镜、没有锁定物理事实、没有逐镜写提示词

**三类核心问题**：

1. **分镜缺失 → 信息过载**
   - 15 秒塞了 5 个状态切换（下潜→悬停→上浮+总结），每个状态只有 3 秒
   - 学生来不及理解"进水增重"就看到"排水上浮"了
   - 正确做法：分 6-7 镜，每镜聚焦一个因果关系

2. **物理事实未锁定 → 可能出错**
   - 没有明确要求"箭头方向必须正确"——重力箭头必须向下、浮力箭头必须向上、进水箭头必须指向水舱内
   - 没有锁定"水舱水位变化"作为画面核心证据——水位上升=下潜、水位下降=上浮
   - 正确做法：每镜的「绝对锁定」必须引用物理原理，箭头方向和水舱水位是最高优先级

3. **优化方向跑偏 → 打补丁不改结构**
   - 达人觉得画面不够好，加的是"水母、海龟、珊瑚"——这些是装饰，对理解浮沉原理没有任何帮助
   - 真正需要优化的是：箭头是否清晰、水舱剖面是否可见、时间分配是否够学生理解因果关系
   - 正确做法：先诊断结构问题，再调细节——不是先加装饰

## 造假背景与附件

**人物**：王老师，初中物理老师，35 岁。教了 10 年书，最头疼的就是"浮力"这一章——学生靠背公式应付考试，但问他"潜水艇怎么上浮的"，支支吾吾说不对因果关系。试过用 AI 生视频，要么箭头画反、要么水舱结构不像潜水艇、要么全程推拉没有节奏。

**附件**：潜水艇浮沉原理（初中物理·浮力章节）
```
潜水艇的浮沉原理基于阿基米德原理和重力变化：

潜水艇在水中受到两个力：
- 重力（向下），等于潜水艇自身重量+水舱中水的重量
- 浮力（向上），等于潜水艇排开水的重量

潜水艇通过调节水舱中的水量来改变自身总重量，从而控制浮沉：

下潜：打开水舱进水阀 → 海水进入水舱 → 水舱水位上升 → 潜水艇总重量增加 → 重力 > 浮力 → 潜水艇下潜
悬停：关闭水舱阀 → 保持当前水量 → 重力 = 浮力 → 潜水艇悬停在某一深度
上浮：用高压空气将水舱中的水排出 → 水舱水位下降 → 潜水艇总重量减小 → 重力 < 浮力 → 潜水艇上浮

关键结构：水舱（ballast tank）位于潜艇中部，有进水口和排水口，是浮沉控制的核心部件。
```

## 模式A · 七轮共创对话脚本（诊断→分镜→出图→转视频→成片）

### 第一轮：给素材，说目标，不教方法

王老师：
```
我是初中物理老师，想做一个潜水艇浮沉原理的教学视频，上课用。
之前试过让AI直接生成，出来的东西不对——箭头方向有时反的、水舱结构不清楚、而且15秒塞了太多内容学生根本跟不上。
这是浮沉原理的说明：
（粘贴潜水艇浮沉原理全文）
你觉得这个该怎么处理？先别急着出图，帮我理理思路。
```

**豆包应该做的**：理解内容类型（物理原理→教学动画）、受众（初中生）、当前失败原因。确认 16:9 横版、科普教学风格。识别出浮沉原理包含"下潜/悬停/上浮"三个核心状态，需要拆镜展开因果关系。

### 第二轮：补充约束，锁定底线

王老师：
```
对了，视频要投影到教室白板上，16:9 横版。
风格要干净清晰——不是那种花里胡哨的科普动画。学生要能在画面里一眼看到：
1. 潜水艇的剖面结构（水舱、进水口、排水口）
2. 重力箭头和浮力箭头的方向和大小变化
3. 水舱水位的变化——这是因果链的核心证据

最重要的是：重力箭头必须始终向下、浮力箭头必须始终向上、水舱进水时箭头指向水舱内、排水时箭头指向外。这些方向错了整个原理就废了。
```

**豆包应该做的**：确认比例和风格。提取「绝对锁定」：
- 重力箭头始终向下（红色，长度变化表示大小变化）
- 浮力箭头始终向上（蓝色，在水中不变）
- 水舱进水箭头→指向水舱内，水位上升
- 水舱排水箭头→指向外，水位下降
- 潜水艇剖面结构：水舱在中部、两端为密封舱

### 第三轮：让豆包先诊断后分镜

王老师：
```
先帮我分析一下——之前别人做这个主题（网上能搜到的那些），最常见的三个错误是什么？
然后根据浮沉原理，你觉得应该拆成几个镜头？每镜讲清哪一个因果关系？
不用写提示词，先给分镜框架。
```

**豆包应该做的**：
1. **诊断三类典型错误**（此部分用于验证豆包是否具备独立诊断能力）：
   - 错误一：不分镜——把下潜/悬停/上浮塞进一条 prompt → 每个状态时间太短，学生跟不上
   - 错误二：不锁物理事实——不指定箭头方向、不要求水舱剖面 → AI 随机画，可能全错
   - 错误三：优化方向偏——画面不好看时加珊瑚水母，而不是检查箭头/水舱/时间分配
2. **给出分镜框架**（6 镜）：
   - 镜1=问题引入：未注水时重力=浮力，潜艇静止在水面。问号：怎么下潜？
   - 镜2=下潜原理：进水阀打开→水舱水位上升→重力>浮力→潜艇下潜（箭头大小对比）
   - 镜3=悬停原理：水舱保持当前水量→重力=浮力→潜艇悬停下不动
   - 镜4=上浮原理：高压空气排水→水舱水位下降→重力<浮力→潜艇上浮（箭头大小对比）
   - 镜5=完整循环总结：下潜→悬停→上浮三个状态的并列对比图
   - 镜6=收尾字幕："改变自身重量，控制浮沉"

### 第四轮：逐镜写静态图提示词

王老师：
```
分镜可以。现在给每一镜写完整的生图提示词。
每一镜都要包含：画幅、绝对锁定的物理事实、创意母题、构图、光影材质、文字策略、禁止项。
写得越详细越好。现在只写提示词，不执行。
```

**豆包应该写的**（以镜2"下潜原理"为例）：
```
【任务与比例】16:9 横版物理教学插图
【绝对锁定】
  - 重力箭头（红色，向下）必须明显大于浮力箭头（蓝色，向上）
  - 水舱进水箭头（白色或浅蓝）指向水舱内部
  - 水舱水位线必须高于上一镜（镜1）的水位线——这是"增重"的视觉证据
  - 潜艇剖面结构：圆柱形艇身，中部为水舱（用不同颜色标示），两端为密封舱
  - 潜水艇整体应呈轻微下倾姿态（艇首略低于艇尾），表示正在下潜
【创意母题】"进水=变重=下沉——一个因果关系"
【主体、场景与构图】
  - 画面中央：潜水艇剖面图占据 60% 画面
  - 潜艇左侧：进水口（蓝色箭头指向水舱内），海水正在灌入水舱
  - 水舱区域：水位线正在上升，当前水位约占水舱 70%，可用半透明蓝色标示
  - 潜艇上方：蓝色浮力箭头（较短），标注"浮力"
  - 潜艇下方：红色重力箭头（较长，约 1.5 倍浮力箭头长度），标注"重力"
  - 潜艇整体有微弱的下倾角和向下的运动线（虚线）
  - 背景：浅蓝到深蓝的渐变海水，干净无杂物
【材质、光影与色彩】
  - 潜艇外壳：哑光深灰色金属质感，剖面切口为浅灰
  - 水舱内水体：半透明蓝色，水位线清晰
  - 重力箭头：红色实心粗箭头
  - 浮力箭头：蓝色实心粗箭头（比红色短）
  - 背景光线从上方自然衰减
  - 不要任何装饰性元素——珊瑚、鱼类、气泡均禁止（画面干净才能突出物理逻辑）
【文字/后期策略】
  - 箭头旁文字标注"重力↑""浮力↑"用极简白字，字号统一
  - 水舱旁标注"水舱"
  - 不出现任何额外文字或公式——这一镜只讲"进水→变重→下沉"
【针对本任务的禁止项】
  - 禁止重力箭头和浮力箭头方向搞反
  - 禁止不画水舱剖面——潜水艇如果只是外观图，学生看不到因果关系
  - 禁止加珊瑚、水母、海龟、气泡等装饰——画面必须干净
  - 禁止潜艇画成黄色卡通玩具——必须是真实潜水艇的工程美学
```

### 第六轮：静图转视频 + 逐镜镜头指令

王老师：
```
图都看过了，物理事实没问题。现在转视频。

注意：
1. 这是一段"物理推理"视频，不是海洋风光片——镜头应该克制。镜2（下潜）和镜4（上浮）可以做微弱的跟随运动（潜艇向下或向上时镜头轻微track），但镜3（悬停）必须完全静止
2. 箭头在视频里不能一闪而过——每镜的重力/浮力箭头至少停留 3 秒，让学生有时间对比大小
3. 镜1到镜2的过渡：水舱水位线从"空"变为"正在进水"，这是最重要的因果转场，用叠化而不用硬切
4. 每镜 6-8 秒，总片长控制在 50-60 秒
5. 镜与镜之间不要加华丽转场——淡入淡出即可，避免分散注意力

先写视频提示词，不执行。
```

**豆包应该写的**（以镜2为例）：
```
【镜2 · 下潜原理 | 时长 7 秒】
【绝对锁定】
  - 重力箭头始终保持向下，浮力箭头始终保持向上
  - 水舱水位持续上升（从 30% → 70%），不可中断、不可倒退
  - 潜艇下倾姿态持续，运动方向为斜向下
【镜头运动类型】微跟随——镜头以潜艇垂直速度的 1/3 速度向下移动，制造下潜感但保持可读性
【动态时间线】
  0.0-1.0s：镜1叠化到镜2，水舱水位线从镜1的空舱状态开始变化
  1.0-2.5s：进水箭头亮起，水舱水位从 30% 上升到 50%，重力箭头逐渐变长
  2.5-3.0s：重力箭头停止增长，明显长于浮力箭头（约 1.5 倍），两个箭头同时闪烁一次——确认"重力>浮力"
  3.0-5.0s：潜艇开始向下移动，舱体保持轻微下倾，水舱水位继续上升到 70%
  5.0-7.0s：潜艇稳定下潜，水舱水位稳定在 70%。镜头继续微跟随。最后 0.5 秒叠化到镜3
【转场】叠化至镜3（悬停），叠化点选在潜艇轮—上—镜2的水舱水位—与镜3的稳定水位形成视觉衔接
【音效建议】（仅供后期参考）水流灌入水舱的闷响；重力箭头出现时的低沉音效
【针对本任务的禁止项】禁止箭头方向翻转；禁止镜头推拉（只做微跟随）；禁止水舱水位出现倒退动画
```

## 视觉创作专项

本案例为物理教学视频，区别于课文科普视频（案例30）的几个关键差异：
- **核心是"力的可视化"而非"生物形态"**：案例30的致命项是海参画错，本案例的致命项是箭头方向画反
- **画面必须干净，拒绝装饰**：任何不服务于"理解因果关系"的视觉元素都是干扰。达人原版加珊瑚水母正好示范了错误方向
- **因果关系通过箭头+水位双通道呈现**：箭头=力的方向，水位=重量的变化。两者必须一致且同步
- **镜头克制是本案例的信号**：推拉摇移会分散学生对箭头对比的注意力——悬停镜必须完全静止
- **旁白用"讲解语气"而非"朗读语气"**：物理原理需要停顿给学生消化时间，不等于拖沓
- **第三轮的诊断环节是本案例的区分点**：达人原版不会诊断、只会加装饰。豆包如果能独立诊断出"箭头方向没锁""分镜太密""装饰是干扰"，说明方法论生效了
