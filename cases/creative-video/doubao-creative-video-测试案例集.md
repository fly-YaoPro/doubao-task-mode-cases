## 案例一：餐饮店主每日菜单口播短视频（UGC路线）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **能力模式** | 能力模式五：多步链条+视频嵌入（区别于案例13/14的"图片→视频"，本案例是"产品信息→口播视频"） |
| **目标人群** | 餐饮店主、面包房、熟食店、小吃摊（全国约900万餐饮商户） |
| **核心痛点** | 每天需要发短视频引流，但录一条口播要备货、打光、念稿、重录三四遍。录两天就放弃了。不用真人出镜的图文流量又差 |
| **任务模式流程** | 每天上午 → 店主输入今日主推菜品+卖点 → 走UGC路线生成15秒口播短视频 → 推手机 |
| **一句话传播** | "别人还在想今天拍什么，AI已经把口播视频录好了——不嘴瓢、不怯场、比你自己录的还稳" |
| **可量化利益** | 每天节省40-60分钟拍摄+剪辑时间；日更不断更，抖音流量权重持续累积 |

**叙事弧（30秒短视频脚本结构）：**
- 0-5s 痛苦：老板拿着手机架，对着面碗拍了七八遍，嘴瓢、忘词、顾客进来打断
- 5-10s 展示：老板在豆包里输入"今日主推卤肉饭，卤汁熬6小时，肥而不腻"
- 10-20s AI生成：屏幕显示豆包自动生成的口播视频——一个清爽的虚拟主播对着镜头推荐卤肉饭，产品图精准对应
- 20-25s 对比：左边是老板自己录的（光线暗、嘴瓢），右边是AI生成的（干净、专业）
- 25-30s 结尾："你也可以"

**过五道硬筛：**

| 筛子 | 判断 | 理由 |
|------|------|------|
| 第0筛：生产力定位 | ✅ | 餐饮经营=搞钱场景 |
| 第一筛：普通对话能独立做吗 | ✅ 不能 | 写脚本+口播+视觉+剪辑四步链，对话串不起来 |
| 第二筛：现有App能做吗 | ✅ 不能 | 没有App能输入菜单自动生成真人感口播视频 |
| 第三筛：大众真的在乎吗 | ✅ | 短视频内容生产是餐饮老板前三焦虑 |
| 第四筛：信息可获取吗 | ✅ 安全区 | 菜单信息用户自提供 |
| 第五筛：信息源质量 | ✅ | 不涉及外部信息 |

### 二、测试背景信息（造假用）

**店铺信息：**
- 店铺名：老魏家面馆
- 位置：杭州西湖区文三路
- 经营品类：手工面、卤肉饭、酸辣粉
- 开了5年的社区老店，大众点评4.5分
- 抖音账号刚开一个月，粉丝200多，每天自己用手机录口播

**今日主推产品：**
```
产品名：秘制卤肉饭
价格：28元/份
核心卖点：
- 五花肉卤汁慢熬6小时，肥而不腻、入口即化
- 卤料配方是爷爷传下来的，13味香料
- 米饭用五常大米，粒粒分明
- 配溏心卤蛋+爽口酸菜，解腻
- 每天限量80份，中午12点前基本卖完
```

**需要准备的产品图**（如果要在豆包里实际测试，提前拍/找类似的图）：
1. 卤肉饭成品俯拍图（白瓷碗，卤肉铺满，溏心蛋半个，撒葱花）
2. 卤肉饭侧面特写（筷子夹起一块五花肉，看得见油脂光泽）

**风格参考：**
- 接地气、像街坊邻居推荐，不要太精致
- 口播语气：热情、实诚、带点杭州口音的普通话感觉
- 场景：面馆厨房/用餐区背景，暖黄色灯光

### 三、测试提示词

**提示词A（基础版——直接说需求）：**
```
帮我做一个产品口播视频。
产品是"秘制卤肉饭"，28元一份。
核心卖点：五花肉卤汁慢熬6小时，肥而不腻；爷爷传下来的13味香料配方；配溏心卤蛋和爽口酸菜；每天限量80份。
风格要接地气，像街坊推荐，不要太广告感的。15秒就行。
```

**提示词B（进阶版——带具体约束）：**
```
帮老魏家面馆做一条15秒的产品口播短视频。

产品信息：
- 秘制卤肉饭，28元
- 五花肉卤汁慢熬6小时
- 13味香料是爷爷传下来的配方
- 配溏心卤蛋+酸菜
- 每天限量80份

要求：
- 不要那种精致高级的广告腔，要像隔壁大姐跟你分享好吃的
- 要有"我吃了三年了"那种真实推荐感
- 竖屏9:16，适合发抖音
- 口播脚本里要有具体的口感描述，不要只说"很好吃"
```

**提示词C（任务模式模拟版——模拟定时触发场景）：**
```
我是老魏家面馆的老板。以后每天上午10点，我会告诉你今天的推荐菜。
今天先试试：今日主推"秘制卤肉饭"，28元一份，五花肉卤了6个小时，配溏心蛋和酸菜，每天限量80份。
帮我生成一条15秒的口播短视频，竖屏，接地气的风格，像熟人推荐。
```

### 四、预期Skill路由

```
用户prompt → 识别为"产品口播"+"UGC" → 触发 ugc-talking-video-ref.md
↓
产品门控：要求上传产品图 / 确认用文字brief生成
↓
参数澄清：时长15s、比例9:16、口播风格、脚本确认
↓
主播抽卡：从5个模板中随机匹配（默认T2甜美邻家或T4中性盐系较适合餐饮）
↓
产品一致性锁：卤肉饭的颜色、配料、碗具绑定
↓
多卖点+安利腔脚本生成：4个安利节点（为什么需要→凭什么有效→具体感受→适合谁）
↓
轻量分镜表输出 → 视频prompt → 视频工具调用
↓
notify_hunman 展示视频
```

### 五、测试关注点（重点验证什么）

1. **产品信息提取能力**：能从"五花肉卤6小时""13味香料""溏心蛋""限量80份"中自动组织卖点栈、区分核心卖点和辅助卖点
2. **口播脚本的真人感**：脚本是否避免了AI腔（"绝绝子""yyds"等），是否像真人推荐而非产品说明书
3. **产品一致性**：如果上传了卤肉饭照片，视频中的菜品外观是否与原图一致（颜色、配料、碗具）
4. **主播模板匹配**：抽到的主播气质是否贴合"接地气餐饮推荐"场景
5. **分镜表的安利结构**：是否按 痛点暴击→产品揭秘→卖点证明→体感证据→CTA 的结构组织
6. **硬门控是否生效**：是否先输出分镜表再调视频工具（不倒置）

## 案例二：节日营销视频自动排期生产（Marketing路线）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **能力模式** | 新建能力模式："定时日历监控+创意视频自动生产"（组合模式二的定时监控 + 模式五的视频生成） |
| **目标人群** | 实体零售店主、社区服务商、餐饮老板、本地生活商家（全国约8000万个体户） |
| **核心痛点** | 节气/节日要做营销内容，但(1)记不住那么多节点 (2)等想起来只剩一天来不及做 (3)临时拼凑的图+文字效果=发了等于没发。一年错过十几个营销节点，加起来少赚几万块 |
| **任务模式流程** | 设置行业和店铺类型 → 每个节日/节气前5天自动提醒 → 用户确认营销诉求 → 走Marketing路线生成15秒促销短视频 → 推手机 |
| **一句话传播** | "别人的店还在群里问'端午发什么'，AI提前一周把营销视频做好了" |
| **可量化利益** | 全年覆盖12+营销节点，每个节点提前5天自动备好素材，不再错失流量窗口 |

**叙事弧（30秒）：**
- 0-5s：花店老板翻日历发现明天就是七夕，手忙脚乱翻照片P图
- 5-12s：另一个水果店老板的手机收到豆包推送——"小暑节气还有5天，已为您生成营销视频"，展示视频效果
- 12-20s：对比——花店老板发的是一张图配几十个字；水果店老板发的是15秒精致视频
- 20-25s：数据——水果店那条视频播放量2万，当天到店客户多了40%
- 25-30s："你也可以"

**过五道硬筛：**

| 筛子 | 判断 | 理由 |
|------|------|------|
| 第0筛 | ✅ | 经营/赚钱场景 |
| 第一筛 | ✅ | 日历监控+营销策略+文案+视频四步链 |
| 第二筛 | ✅ | 无App能自动排期+生成 |
| 第三筛 | ✅ | 内容营销是实体商家第二大痛点（仅次客流） |
| 第四筛 | ✅ | 节日/节气公开信息 |
| 第五筛 | ✅ | 节气日期可查官方 |

```
调性是东方美学、留白、自然感的。竖屏9:16。
```

### 四、预期Skill路由

```
用户prompt → 识别为"产品营销广告"+"无分镜需求" → 触发 product-marketing-ad-video-no-storyboard-ref.md
↓
产品门控：要求上传产品图/确认用文字brief
↓
品类识别：茶饮/食品 → 推食品/饮品调性路线
↓
传播主线判断：氛围种草（不是品鉴证明、不是促销转化、不是功能卖点）
↓
创意视觉解法匹配：材质联想（水、雾、竹、光）+ 负空间留白
↓
调性路线：诗性旁白路线 → 2-3句主体旁白+1句slogan收尾
↓
音效系统：低频、慢节奏BGM，自然环境音（竹叶沙沙、水流、冰块碰撞）
↓
15s分镜：0-2s蓄势→2-5s产品揭秘→5-10s卖点证明→10-13s情绪价值→13-15s packshot
↓
Seedance prompt编译 → 视频工具调用 → notify_hunman展示
```

### 五、测试关注点

1. **传播主线的判断准确性**：是否能在"品鉴证明""功能卖点""促销转化""氛围种草"之间选对路线
2. **诗性旁白的克制程度**：旁白密度是否恰当（2-3句而非5-6句），是否避免了"说明书腔"
3. **创意视觉解法的落地**：材质联想、负空间、光影策略是否在分镜里具体化（而非停留在"要有高级感"）
4. **品牌调性一致性**：如果提供了品牌调性词（东方美学/留白），是否贯彻到分镜、旁白、配色的所有维度
5. **slogan收尾质量**：最后一句是否为可记忆的、贴合品牌主张的收束（而非"快来买吧"）
6. **产品一致性**：如果上传了产品图，视频中的瓶型、标签、颜色是否准确

## 案例三：小微企业宣传视频自动生成（Corporate路线）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **能力模式** | 新建能力模式："企业公开信息→商务视频自动生产"（Corporate路线首次进入任务模式案例库） |
| **目标人群** | 小微企业主、个体创业者、工作室创始人（全国1.7亿市场主体，做过企业宣传片的不到1%） |
| **核心痛点** | 找制作公司一个宣传片5-20万；自己拍不知道说什么、怎么剪。官网介绍全是文字，客户看了没感觉。参加展会/见投资人/投标，连一个像样的企业介绍视频都拿不出来 |
| **任务模式流程** | 上传logo+官网+业务介绍 → 自动搜索工商/新闻公开信息 → 走Corporate路线生成15秒企业宣传短视频 → 推手机 |
| **一句话传播** | "别人花5万做企业宣传片，AI一晚上帮你生成好了——能拿去投标、能发朋友圈、能放官网首页" |
| **可量化利益** | 省5-20万制作费；从有想法到拿到成品缩到30分钟以内 |

**叙事弧（30秒）：**
- 0-5s：一个做工业设计的创业者被客户问"你们公司有没有介绍视频？"，尴尬地说"还在做"
- 5-12s：他在豆包里上传了公司logo和官网链接，描述了业务。豆包自动搜索了工商信息和行业新闻
- 12-22s：AI生成了15秒企业宣传片——干净、专业，有logo露出、有业务场景、有核心竞争力
- 22-25s：他把视频发给了那个客户，客户当场回复"挺专业的，约个时间详聊"
- 25-30s：结尾

**过五道硬筛：**

| 筛子 | 判断 | 理由 |
|------|------|------|
| 第0筛 | ✅ | 企业经营/获客场景 |
| 第一筛 | ✅ | 搜索+资料整理+叙事策略+分镜+视频，五步链 |
| 第二筛 | ✅ | 无App能自动完成企业宣传片 |
| 第三筛 | ✅ | 企业视频是B2B获客刚需 |
| 第四筛 | 可探讨区 | 工商/官网公开信息可搜；内部素材需用户提供 |
| 第五筛 | ✅ | 优先官网/工商等一手中文源 |

### 二、测试背景信息（造假用）

**企业信息（虚构但完整）：**

```
公司全称：杭州云帆智能科技有限公司
品牌名：云帆智能 / CloudSail AI
成立时间：2021年3月
所在地：杭州余杭区未来科技城
行业：企业级AI数据服务
员工规模：40+人
```

### 四、预期Skill路由

```
用户prompt → 识别为"企业宣传/商务视频" → 触发 corporate-business-video-ref.md
↓
搜索优先：如果给了公司名/官网，先调用search_tool搜索公开信息
↓
source_summary生成：
  - company_identity
  - official_sources（工商信息、官网、高新技术企业公示）
  - business_facts（区分"已核实/用户提供/待确认"）
  - claim_risks（8000万条、99.2%准确率——标注为"用户提供，待确认"）
  - brand_assets_needed（logo、品牌色）
↓
四类必填信息收集：
  1. 企业信息（搜索+用户补充）
  2. 宣传文案（主宣传语+卖点+禁用词）
  3. 品牌资产（logo/IP/VI/色值/图片素材）
  4. 旁白文案（完整旁白脚本）
↓
策略字段确定：
  - business_objective：建立信任/获客
  - target_audience：企业客户/投资人
  - communication_mainline：专业能力
  - narrative_motif：能力到信任
  - proof_chain：认证+客户+数据量
↓
用户确认摘要 → 确认后进入分镜
↓
10-12个分镜（15秒，Corporate硬性要求）：
  场景张力→企业身份→能力证明→人物/客户→价值结果→品牌收束
↓
旁白逐镜分配 → prompt编译 → 视频工具调用 → notify_hunman展示
```

### 五、测试关注点

1. **公开搜索的触发和准确性**：skill是否在获得公司名后主动搜索？搜索结果是否区分了"已核实/用户提供/待确认"
2. **四类必填信息的收集逻辑**：是一次输完摘要还是逐个追问？是否在信息不全时阻断生成
3. **叙事母题的选择**：是否选了"能力到信任"（最匹配），还是"技术到人"、"案例到结果"
4. **proof_chain的可视化**：认证、客户、数据量的"证据"在分镜里如何呈现——是堆图标还是有叙事推进
5. **事实核查**：8000万条、99.2%准确率——skill是否标注为"用户提供未核实"，还是直接当作事实写入
6. **10-12分镜硬性要求**：Corporate路线要求最少10个分镜，是否达标
7. **旁白的业务现实感**：是否至少安排了2个"现实业务瞬间"（而非全程抽象视觉）
8. **长视频拆段判断**：如果用户要求30秒/60秒，skill是否先询问"多条独立15秒"还是"一个长片分段拼"
