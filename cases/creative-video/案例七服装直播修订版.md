# 案例七（修订）：服装直播团队·每日直播高光切片自动生成

> 修订日期：2026-06-30
> 修订原因：原案例为食品直播（唐点），改为服装直播（沫竹Mozoo）以匹配用户准备的实测素材
> 适用素材：一段服装直播全程录像（预期时长1-3小时，主播讲解+试穿+互动）


## 过筛

| 筛 | 判断 |
|----|------|
| 第0筛（生产力） | ✅ 服装直播带货是经营活动 |
| 第一筛（对话能做吗） | 单次能做一个切片，但每天直播3小时，手动回看找高光+逐条剪辑=每天多耗2-3小时 |
| 第二筛（App能做吗） | 无App能自动识别"试穿转身""面料特写""穿搭对比"等服装直播独有高光时刻 |
| 第三筛（大众在乎吗） | ✅ 抖音服装直播日均超10万场，切片视频是直播间引流+橱窗带货的核心素材 |
| PC-1 | ✅ 直播回放文件在电脑端管理，剪辑软件在PC操作 |
| PC-2 | ✅ 3小时回放→提取多条切片→多SKU管理=高信息密度 |
| PC-3 | ✅ 需在回放浏览、视频工具、投放后台之间切换 |

**对话模式 vs 任务模式差距**：无法替代。对话不会每天自动看3小时回放找高光，也不会记住"上周试穿效果最好的那件针织衫这周应该优先切片"。


## 造假背景（虚构，用于测试提示词逻辑）

**直播间**："沫竹Mozoo"淘宝+抖音双平台女装直播间。主播小鹿一个人播2-3小时（通常下午2-5点），每场展示15-20款衣服，每款做：挂拍展示→面料特写→自己试穿→正面/侧面/背面转一圈→搭配建议→报尺码+价格→引导下单。

**困境**：一场直播下来，有太多可以做切片的瞬间了——试穿转身的飘逸感、面料对光特写的质感、两种穿法对比的实用感。但小鹿每天下播后还要打包发货——根本没有时间再花2小时回看自己3小时的直播来找这些片段。不剪切片=直播间没投流素材=仅靠自然流量卖了两个月一直没突破日均2000的观看量。

**任务模式破局**：每天下播后自动读取回放→按服装直播特有的高光规则提取5-8个片段→每个片段调用doubao-creative-video生成15秒切片→自动叠加价格+尺码+购买入口→命名归档→推送到投放素材库。小鹿第二天醒来直接选几条发抖音就行。


## 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个服装直播每日高光切片自动生成任务：

【任务触发】每天下午5:30（直播结束后30分钟）

【任务流程】
1. 读取今日直播回放视频（默认路径"D:\沫竹Mozoo\直播回放\{日期}\\"）
2. 根据以下规则识别5-8个高光片段：

   服装直播高光识别优先级（按此权重排序）：

   优先级一·试穿效果高光（权重最高，取3-4个）
   - 主播换好衣服后第一次完整出镜展示的镜头（面向镜头站定→转一圈→背面→转回来）
   - 识别特征：人物从画面左侧/右侧走入→站定→展开双臂→缓慢转圈→再次面朝镜头
   - 取转圈过程中裙摆/裤脚/衣袖飘动最自然的连续5-8秒
   - 如果转圈后有"走近镜头"展示细节的动作→额外加分

   优先级二·面料质感高光（取1-2个）
   - 主播用手揉搓/拉伸/对光展示面料的特写镜头
   - 识别特征：手部入画→抓住衣服某处→揉搓/拉伸/举起对光→镜头放大
   - 取面料纹理最清晰、光影最好的3-5秒
   - 针织/亚麻/天丝等材质优先（牛仔/化纤次之）

   优先级三·穿搭对比高光（取1个）
   - 主播展示"同一件衣服的两种穿法"或"搭配前vs搭配后"
   - 识别特征：主播说"这个还可以……""如果你有XX也可以配……"等对比句式
   - 取两个穿法的并排对比瞬间

   优先级四·价格/权益宣布高光（取1个）
   - 主播明确报出价格+福利的瞬间
   - 识别特征："今天直播间价格是……""现在拍下还送……""仅限今天……"
   - 取说话内容最完整的10-15秒

   优先级五·互动反应高光（取0-1个，备选）
   - 主播回应弹幕/回答观众问题的有趣瞬间
   - 识别特征：主播说"有姐妹问……""XX码还有吗……"然后回答
   - 取问答完整的一个来回

3. 对每个高光片段，调用doubao-creative-video做以下处理：
   
   【保留项】原直播画面、主播人物形象（不换脸不改身材）、产品外观（颜色/版型/面料不漂移）
   
   【增强项】
   - 提亮画面（直播灯光通常偏暗/偏黄→修正为自然白光感）
   - 去噪+加锐（直播画质一般偏软→边缘清晰但不塑料感）
   - 保持9:16竖版不变
   
   【叠加项】
   - 产品名称+价格标签：在画面左下角，半透明黑底+白字，如"亚麻衬衫 ¥269"
   - 尺码标签：在画面左下角价格旁边，小字，如"S-L码"
   - 品牌水印：画面右上角，小字半透明，"沫竹Mozoo"
   - 购买引导条：画面底部，窄条横幅，品牌色#4A3728底+白字"点击下方小黄车购买"
   
   【不需要的】
   - 不要加BGM（保留直播原声，观众对真实感敏感）
   - 不要加AI配音/旁白（原声才有信任感）
   - 不要逐句字幕（画面已经很复杂了，避免文字叠文字）
   - 不要美颜过度（直播滤镜已经够重了）

4. 每条切片15秒，9:16竖版

5. 按"产品名_高光类型_时长"命名（如"亚麻衬衫米白_试穿效果_0630_15s.mp4"）

6. 保存到"E:\沫竹Mozoo\投放素材库\直播切片\{月份}\"

7. 生成今日切片清单（表格），推送到飞书群"沫竹·内容运营"：
   | 文件名 | 产品 | 高光类型 | 建议发布时间 | 备注 |
   |--------|------|---------|-------------|------|
   | 亚麻衬衫米白_试穿效果_0630 | 亚麻衬衫 MZ2401 | 试穿效果 | 今晚8点 | 转圈飘逸感很好 |
   | 阔腿裤_面料质感_0630 | 天丝阔腿裤 MZ2412 | 面料质感 | 明早9点 | 对光能看到面料纹理 |
   ……

【需要记住的模板（每次复用）】
- 品牌：沫竹Mozoo，日系简约女装
- 品牌色：深棕#4A3728、米白#FAF5F0
- 切片风格：真实直播感、不过度加工、保留原声、去滤镜化
- 价格标签模板：半透明黑底+白字+圆角、字号适中不抢画面
- 尺码标签模板：紧挨价格标签右侧，字体小一号
- 高光识别权重：试穿效果 > 面料质感 > 穿搭对比 > 价格宣布 > 互动反应
- 时长规则：每条15秒，如果原始高光素材不足15秒→保留原素材长度不做拉伸
- 产品命名规则：产品品类+颜色/款号（从直播中自动提取）

【本周运营策略】（需每周更新）
- 本周主推款：MZ2408 A字连衣裙（库存深、利润高、夏季应季）
- 本周新品：MZ2425 亚麻短裤（本周三开始卖的，需要多切片引流）
- 主推款切片额外加分（优先从切片中推这2款）

【异常处理】
- 如果回放文件不存在→推送提醒"今日直播回放未找到，请检查录制系统"
- 如果提取到的高光片段不足3个→只处理已找到的，并在清单中标注"本场高光较少，建议优化直播内容节奏"
- 如果某个片段画面噪声过大/主播不在画面中/画面几乎静止→跳过，不计入总量
- 如果某个SKU本周已被切片5次以上→降低该SKU高光识别权重（避免同一个产品切片太多用户看腻）
```

**测试关注点**：

| 关注项 | 描述 | 风险 |
|--------|------|------|
| 试穿高光识别 | 能否准确找到"转圈展示"的时间点而非随便一个站定镜头 | 🔴 |
| 面料特写识别 | 能否区分"手摸面料特写"和"手在镜头前晃过" | 🟡 |
| 产品名称自动提取 | 从主播口头说的"这件是……"中正确提取SKU名和款号 | 🔴 |
| 价格提取 | 从"今天直播间价格是……"中提取正确价格数字 | 🔴 |
| 视频增强自然度 | 提亮+去噪后的画面是否自然（直播原画面通常偏黄偏暗） | 🟡 |
| 标签不遮挡 | 价格/尺码/水印是否遮住了产品主体或主播的脸 | 🟡 |
| 保留原声 | 是否遵守了"不加BGM不换旁白"的规则 | 🟢 |
| 同SKU频控 | 同一产品本周切片是否控制在5条以内 | 🟢 |


## 测试用素材准备说明

你要去down的服装直播回放，预期有这样的内容结构（帮你判断素材是否合适）：

**时长**：1-3小时均可（太短的话高光点太少，太长的话处理时间太久）

**需要包含的典型镜头**：
1. ✅ 主播至少展示过5件以上不同的衣服（否则切片产品太单一）
2. ✅ 每件衣服至少有"试穿→转一圈"的展示动作（这是切片的核心素材）
3. ✅ 至少有2-3次面料揉搓/对光的特写（质感高光来源）
4. ✅ 有价格报出/优惠说明的片段（价格标签需要这个信息）
5. ✅ 画面不要大面积黑屏/卡顿/花屏（AI高光识别会误判）

**什么类型的直播回放不太适合**：
- ❌ 全程坐播、衣服只举在身前比划不试穿的（没有试穿高光）
- ❌ 节奏太慢、5分钟讲一件衣服的（3小时只有十几件→高光密度太低）
- ❌ 全程用手机前置摄像头竖屏自拍的（画面抖动严重、产品看不清）
- ❌ 原视频已经被加了大量贴纸/字幕/滤镜的（AI再叠加标签会花）

**优选**：找那种"主播站着播、一件衣服展示3-5分钟、会试穿会转圈会展示面料"的服装带货直播回放。播主不用是大V——普通腰部主播的直播节奏反而更适合切片，因为大主播直播间画面太复杂（遮挡物多、弹幕多、镜头切来切去）。
