## 案例十五：财经自媒体作者·每日开盘前市场热点简报

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 财经内容创作——生产力/经营活动 |
| 第一筛 | 对话能做一次热点解读，但每天早上手动检索+梳理+写稿=不可能每天坚持 |
| 第二筛 | 无App能记住内容风格偏好+每天自动跑热点分析+输出固定结构 |
| 第三筛 | ✅ 财经自媒体（公众号/雪球/抖音）从业者数十万，每天需要"今天写什么"的选题和素材 |

**对话模式 vs 任务模式**：对话=每天早上手动搜新闻→手动整理→手动写稿。任务模式=早8:00自动出热点简报→人只需要确认和改写。

### 造假背景

**人物**：小刀，前财经记者，现在做自己的财经公众号"刀哥看市"，5万粉丝。每天发一篇市场解读，风格是"说人话、讲逻辑、不装神弄鬼"。

**困境**：每天早上醒来第一件事就是翻华尔街见闻、财联社、金十数据——看昨晚外盘怎么样、今天A股有什么大事、政策面有什么新变化。然后花1-2小时梳理成一篇1500字的文章。这件事每天早上都要做，雷打不动。最痛苦的是周末想休息一天——结果周一早上要补周末两天的信息量，直接崩溃。

**造假数据·内容风格偏好**：

```
账号：刀哥看市
定位：A股市场每日解读，说人话版本
风格：口语化、有态度、逻辑链条清晰、不堆数据
固定结构：
  1. 一句话核心判断（今天市场最值得关注的一件事）
  2. 昨天发生了什么（A股+外盘+商品三句话）
  3. 今天盯什么（政策/事件/数据）
  4. 刀哥怎么看（300字观点，有论据有边界）
字数：1200-1500字
发布时间：每天早上8:30
```

### 场景测试

**预判路由**：doubao-market-hotspot → 对话框正文（7段叙事链）

**为什么匹配**：skill明确支持"市场整体与宏观事件解读"，输出结构恰好是7段叙事链——核心观点→增量信息→为什么重要→如何影响市场→各方观点→验证/证伪信号→信息来源。这个结构可以直接作为公众号文章的素材骨架。

**提示词**：

```
帮我创建每日开盘前市场热点自动简报：

【路由】doubao-market-hotspot → 对话框正文

【持久化配置】
- 市场范围：A股为主+隔夜外盘（美股/港股）+重要大宗商品（原油/铜/黄金）
- 时间窗：近24小时（交易日覆盖昨日收盘到今早开盘前的所有增量）
- 分析模式：全市场默认模式

【内容风格模板】（每次复用）
- 账号：刀哥看市
- 定位：说人话的市场解读
- 风格：口语化但不轻浮、有态度但讲逻辑
- 读者画像：有基本投资常识的散户，不需要解释什么叫PE什么叫MACD
- 固定结构（对应skill的7段叙事链）：

  【核心判断】（对应"核心观点"）
  用一句话说今天最值得关注的一件事。例如：
  "昨晚美联储放鸽，今天A股大概率高开。但别高兴太早——高开之后能不能撑住，取决于北向资金是不是真的进场。"

  【隔夜发生了什么】（对应"有哪些关键增量信息"）
  三句话讲清：A股昨收+外盘隔夜+大宗早盘。只写数字和事实，不加评论。

  【这些事为什么重要】（对应"这些信息为什么重要"+ "如何影响市场"）
  解释增量信息背后的真正变化和预期差。例如：
  "美联储这次不是'鸽'了利率，而是'鸽'了对通胀的表述——从'仍然偏高'改成了'取得进一步进展'。这意味着9月降息概率从60%跳到75%。"

  【今天盯什么】（对应"验证/证伪信号"）
  今天盘中需要盯的3个信号。例如：
  "盯北向资金开盘15分钟的净流向（如果净流入超20亿→外资在真金白银买）；盯A股券商板块（券商先动=情绪真的回暖）"

  【刀哥怎么看】（对应"核心观点"的展开）
  300字观点。有论据、有边界、有"如果XX则XX"的条件推演。不写"明天必涨"这种垃圾话。

- 字数：1200-1500字
- 不要：emoji、过度排比、AI套话（"值得关注的是""综上所述""毋庸置疑"）
- 要：具体数字、具体日期、具体信号、条件推演（"如果XX则XX，否则XX"）

【触发】每个交易日早上8:00

【执行流程】
1. 调用doubao-market-hotspot，获取昨日收盘到今早的增量信息
2. 按上述内容风格模板，将skill的7段叙事链改写为"刀哥看市"的口语化版本
3. 推送飞书"刀哥·内容素材"：标题建议+完整文章素材
4. 同时保存到"E:\刀哥看市\市场简报\{日期}.md"

【需要记住的】
- 昨天简报中的核心判断——今早如果被验证/证伪了，要在开头提一句
- 上周提到过的"待观察信号"——如果本周出现了，要点出来

【异常处理】
- 如果隔夜外盘数据不可得→标注"外盘数据暂未更新"
- 如果当日无增量信息（极其罕见的新闻真空）→推送"今日无重大增量，延续昨日判断"
- 非交易日→跳过
```

**测试关注点**：

| 关注项 | 描述 | 风险 |
|--------|------|------|
| 事实准确性 | skill要求"关键事实必须有可打开来源"，简报中的数字/日期/事件是否可追溯 | 🔴 |
| 口语化改写 | 从skill的7段正式叙事→"刀哥"口语风格的转化是否自然 | 🟡 |
| 跨日逻辑连贯 | 昨天的判断今天是否被追踪（验证/证伪） | 🟡 |
| 字数控制 | 命中的1200-1500字是否稳定（不150字敷衍也不3000字啰嗦） | 🟢 |
| 合规 | 有无"必涨""肯定""绝对"等违规词 | 🔴 |


## 案例十六：独立投资人·财报季持仓股业绩自动扫描

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 投资管理——财报分析是职业刚需 |
| 第一筛 | 对话能做一次财报分析，但无法自动监控"持仓股的财报什么时候发布"+自动跑分析 |
| 第二筛 | 无App能记住持仓股财报日历+自动触发分析 |
| 第三筛 | ✅ 财报季是A股最重要的信息节点——错过一份财报的分析窗口=错过最佳调仓时机 |

**对话模式 vs 任务模式**：对话=每次手动搜"XX股发了财报没"→手动输入→手动分析。任务模式=自动盯财报日历→公告一出自动触发分析→推送摘要+完整报告。

### 造假背景

**人物**：老徐（同案例十三/十四），管理500万自有资金。最怕财报季——不是因为看不懂财报，而是因为持仓3只股+观察池10只股=13份财报需要在发布当天或次日看完并形成判断。每年的2-3月和8-9月是噩梦——13份财报集中在4-6周内发布，每天都有一到两份新财报要读。

他需要一个"财报哨兵"——不是替他读财报，而是告诉他"XX股发了财报→核心数据是什么→和三季度前的预测有多大差距→值得不值得花时间精读"。

### 场景测试

**预判路由**：doubao-earnings-analysis → 报告模式

**为什么匹配**：skill的"报告模式"专门针对"公司+报告期但无具体问题"的全局性财报分析——覆盖收入/利润/毛利率/现金流/费用率等核心科目，产出从"发生了什么"到"为什么公司不能改变它"的深度因果链。老徐需要的第一层扫描=skill报告模式的前半段（核心数据+关键变化+亮点/风险）。

**提示词**：

```
帮我创建财报季持仓股业绩自动扫描任务：

【路由】doubao-earnings-analysis → 报告模式

【持久化持仓+观察池】（财报季自动盯）
持仓：
- 中科创达 300496.SZ | AI agent+智能汽车 | 关键指标：智能汽车业务增速
- 亿航智能 EH.US | eVTOL | 关键指标：交付量和适航证进展
- 立讯精密 002475.SZ | 消费电子+汽车电子 | 关键指标：毛利率趋势

观察池（财报日自动纳入分析）：
- 德赛西威 002920.SZ | 智能座舱 | 关键指标：新定点项目数
- 科大讯飞 002230.SZ | AI | 关键指标：星火大模型收入
- 汇川技术 300124.SZ | 工控 | 关键指标：新能源业务增速

【触发条件】
1. 每个交易日早上8:00，扫描持仓+观察池是否有公司在前一交易日收盘后发布财报
2. 如果命中→自动触发财报分析
3. 如果财报在盘中发布→当天收盘后补触发

【执行流程（对每份命中的财报）】

Step 1 · 调用doubao-earnings-analysis报告模式
产出分析正文（固定结构）：
- 收入：出货量/ASP/分业务的真实变化——不是"收入增长10%"，而是"收入增长10%来自手机业务出货量+15%但被汽车业务单价-8%对冲"
- 毛利率：如果是下降→是原材料涨了、产品降了价、还是出货结构变了？
- 核心亮点（≤3个）：必须是可持续的，不包含"政府补贴""资产处置"等一次性项目
- 核心风险（≤3个）：必须具体到"如果XX指标下季度继续恶化→意味着XX"
- 和三季度前预测的对比：哪个数字差最大？那个差异说明了什么？

Step 2 · 生成扫描摘要（不展开深度分析，帮老徐判断"值得看吗"）

```
📊 中科创达 2026Q2 财报扫描

一句话结论：营收+18%略超预期，但毛利率下滑1.2pp至34.3%（不如表面数字好看——汽车电子毛利率只有28%，拖累了整体）

和预测的差异：
- 营收¥12.3亿 vs 预测¥11.9亿（+3.2%，OK）
- 毛利率34.3% vs 预测35.6%（-1.3pp，⚠ 需关注）
- 扣非净利¥2.1亿 vs 预测¥1.95亿（+7.7%）

亮点：
✅ AI agent业务首次披露单独收入¥1.2亿（占10%）——增速+67%，远超预期。这可能是市场还没定价的新增量
✅ 海外客户新增2家德系车企——打开了欧洲市场想象空间

风险：
⚠ 汽车电子毛利率仅28%，同比-4pp——管理层解释是"新品导入期"。但如果是结构性下降（竞争加剧），不是导入期问题
⚠ 研发费用率同比+2pp至18%——如果AI agent收入增速跟不上研发投入，利润率会持续承压

结论：值得精读。核心矛盾在"汽车电子毛利率-4pp是结构性的还是阶段性的"——这个问题不搞清楚不能做仓位决策。

👉 回复"精读"生成完整飞书文档报告
```

Step 3 · 推送飞书"老徐·财报季"
→ 每份财报一条推送，带"精读"快捷指令
→ 如果用户回复"精读"→触发完整报告模式（生成飞书文档）


【需要记住的】
- 持仓+观察池的每只股的"关键指标"——财报分析时重点回答这些
- 每只股上一份财报的核心数据——方便环比（"和上个Q比发生了什么变化"）
- 财报日历——预测发布日期（基于历史规律和交易所公告）

【异常处理】
- 如果财报数据不足（如某些美股字段不可得）→标注"数据不全"，可分析的科目正常输出
- 如果财报是预披露/业绩快报而非正式年报→标注"基于快报数据，正式年报可能有调整"
- 如果发布时间异常（如延迟发布）→额外标注"延迟发布⚠ 关注是否有未披露事项"
```

**测试关注点**：

| 关注项 | 描述 | 风险 |
|--------|------|------|
| 财报日历触发 | 能否准确知道持仓股什么时候发布财报 | 🔴 |
| 收入函数识别 | skill能否正确识别不同公司的收入函数（中科创达=软件授权+项目，亿航=交付量×ASP） | 🔴 |
| 解释深度 | 是否推到"为什么公司不能改变这个变化"的层面 | 🔴 |
| 数据来源绑定 | skill要求每个进入正文的数字可追溯到来源——扫描摘要虽然简短，但数字是否可信 | 🟡 |
| 关键指标聚焦 | 是否针对用户预设的"关注问题"重点分析了关键指标 | 🟡 |
| 和三季度前对比 | 如果上下文中有之前财报数据，是否做了环比反差 | 🟢 |
| 合规 | 无评级/目标价/买卖建议 | 🔴 |
