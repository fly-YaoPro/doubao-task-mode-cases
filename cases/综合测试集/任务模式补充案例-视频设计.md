## 案例七：电商直播团队·每日直播高光切片自动生成

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛（生产力） | ✅ 直播带货是经营活动 |
| 第一筛（对话能做吗） | 单次能做一个视频，但每天直播2-4小时，手动找高光时刻+剪辑+导出=每天多耗2小时 |
| 第二筛（App能做吗） | 无App能自动识别直播高光+每日自动剪辑 |
| 第三筛（大众在乎吗） | ✅ 抖音电商直播日均50万场，直播间切片视频是投流素材核心来源 |
| PC-1 | ✅ 视频文件管理/剪辑天然在电脑端 |
| PC-2 | ✅ 长直播→多条短视频的素材管理密度高 |
| PC-3 | ✅ 需在直播回放文件、剪辑参数、投放后台之间切换 |

**对话模式 vs 任务模式差距**：无法替代（对话不会每天自己看回放找高光）

### 造假背景

**直播间**："唐点·广式点心旗舰店"（虚构），每天下午2-5点在抖音直播。主播小夏一个人撑3小时，边蒸边讲——虾饺蒸熟开盖的那个瞬间、流沙包掰开流心的画面、凤爪脱骨的镜头，都是天然的高光素材。但是每天下播后还要花2小时剪切片视频发抖音——不剪就没有投流素材，不投流就没有新流量。

**困境**：这件事形成了一个死循环——直播越做越好，切片需求越多，但剪切片的人只有小夏自己。她试过AI剪辑工具，但那些工具只按"声音突然变大"来判定高光——结果剪出来的全是锅碗瓢盆摔的片段。

**任务模式破局**：每天下播后自动：读取直播回放→按产品出现的时间点（蒸笼开盖/流沙包掰开/特写镜头）提取3-5个高光片段→每个片段调用doubao-creative-video生成15秒short→自动配好字幕+商品链接→推送到投放素材库。

### 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个每日直播切片自动生成任务：

【任务触发】每天下午5:30（直播结束后30分钟）

【任务流程】
1. 读取今日直播回放文件（默认路径"E:\唐点\直播回放\{日期}\\"）
2. 根据以下规则识别3-5个高光片段：
   - 产品展示高光：蒸笼开盖冒蒸汽持续3秒以上的镜头
   - 质感高光：流沙包掰开/虾饺夹起对光/凤爪脱骨的近景镜头
   - 反应高光：主播试吃后给出好评反应（点头/眼睛睁大/说"这个真不错"）
   - 价格高光：主播宣布"今天直播间价格是XX"的时刻
3. 对每个高光片段，取前后各5秒=共10-15秒素材
4. 对每个素材片段，调用doubao-creative-video的Marketing路线：
   - 基于原始画面做视频增强（提亮、加锐、去噪）
   - 添加中文字幕（关键卖点大字+价格标签）
   - 添加品牌尾帧（唐点logo+抖音店铺入口）
   - 比例9:16竖版
5. 按"产品名_高光类型_日期"命名（如"虾饺_质感高光_0630.mp4"）
6. 保存到"E:\唐点\投放素材库\切片\{月份}\"
7. 生成今日切片清单（文件名+对应产品+建议投放时段），推到我飞书

【需要记住的模板】
- 品牌：唐点·广式点心
- 品牌色：朱砂红#C43A30、米白#FAF5F0
- 视频风格：真实直播感、不追求棚拍质感、字幕用粗圆体（零食风）
- 尾帧模板：品牌logo+抖音店铺二维码+"直播间同款·点击购买"
- 高光判断权重：产品展示>试吃反应>价格宣布（不要只挑声音大的片段）

【异常处理】
- 如果直播回放文件不存在→推送提醒"今日直播回放未找到，请确认录制是否正常"
- 如果整场直播找不到3个高光片段→只处理找到的片段，并在清单里标注"本场高光较少"
- 如果某个片段画面全黑或严重过曝→跳过该片段
```

**测试关注点**：

| 关注项 | 描述 |
|--------|------|
| 高光识别准确性 | 是否按产品展示>试吃反应>价格宣布的权重正确筛选 |
| 视频增强效果 | 直播画面（通常有点灰/暗）增强后是否自然 |
| 字幕叠加 | 中文卖点字幕是否清晰+位置不遮产品 |
| 尾帧一致性 | 每条视频的尾帧是否统一 |
| 命名归档正确 | 文件名是否符合规则、归档文件夹层级是否正确 |

### 场景发散

- **服装直播**：换装展示→自动切片每套look
- **美妆直播**：上妆前后对比→自动切片效果片段
- **3C直播**：产品功能演示→自动切片关键操作片段
- **农产品直播**：切开/掰开/试吃→自动切片质感高光
- **珠宝直播**：细节对光→自动切片火彩/透明度片段


## 案例八：电商卖家·大促期间每日倒计时视频矩阵

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 电商经营活动 |
| 第一筛 | 对话能做一条，但大促需要每天一条且视觉连贯→对话记不住昨天的风格 |
| 第二筛 | 无App能跨天记住视频风格参数自动产出系列 |
| 第三筛 | ✅ 每年618/双11/年货节，所有电商卖家都需要预热内容 |
| PC-1 | ✅ 视频审核+投放后台是PC场景 |
| PC-2 | ✅ 多条视频+多平台+排期表=高信息密度 |

### 造假背景

**卖家**："沫竹Mozoo"，同上日系女装品牌。618大促预热期（5月28日-6月17日共21天），需要每天发布一条倒计时视频到抖音+小红书+视频号。

**困境**：运营小陈去年618手动做了21条倒计时视频，每天一条。做到第10天的时候已经完全分辨不出"今天这条和第8天的有什么区别"——视频模板化之后，用户也看麻了，后面几天的完播率断崖式下降。

她想要的不是21条一模一样的视频，而是"21天里视觉上有变化、但不跑偏品牌调性、最后3天明显加重紧张感"的倒计时视频矩阵——这需要一个人花整整3天来逐条设计，但她只有一个人。

### 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个618大促倒计时视频矩阵任务：

【任务周期】2026年5月28日 - 6月17日（21天）

【任务触发】每天早上7:00自动执行

【品牌信息】（整个周期记住）
- 品牌：沫竹Mozoo，日系简约女装
- 品牌色：暖米色#FAF5F0、深棕#4A3728、低饱和鼠尾草绿#9CAF88
- 视频风格：温暖、安静、有质感。不是喊"便宜了便宜了"的促销风，而是"你喜欢的衣服，现在值得买了"
- 比例：9:16竖版，15秒
- 路线：doubao-creative-video的Marketing路线，有旁白但语速慢、治愈感

【倒计时阶段视觉策略】
第一阶段 · 5月28日-6月7日（11天）：种草预热期
- 画面：产品场景图+自然光影+轻旁白
- 旁白风格："还有X天。不急，先看看你喜欢哪件"
- 视觉关键词：呼吸感、大留白、慢节奏
- 每天展示不同产品线（衬衫/裤子/裙子/开衫轮换）

第二阶段 · 6月8日-6月14日（7天）：利益释放期
- 画面：产品+价格标签+满减信息
- 旁白风格："再等X天。全店7折，满599再减50"
- 视觉关键词：信息卡片化、利益点清晰但不轰炸

第三阶段 · 6月15日-6月17日（3天）：临门一脚
- 画面：品牌标志性单品+大号倒计时数字+紧迫感光影
- 旁白风格："还剩X天。你收藏的那件，别让人抢了"
- 视觉关键词：倒计时数字占画面1/3、暖调金棕色光、微动态

【每日任务内容】
1. 计算当前日期距离6月18日的剩余天数
2. 判断当前属于哪个阶段→匹配对应的视觉策略
3. 调用doubao-creative-video的Marketing路线生成当日倒计时视频
   - 如果阶段一：从产品图库随机选当日展示产品线
   - 如果阶段二：叠加价格利益信息
   - 如果阶段三：取品牌最畅销3款产品+大号倒计时数字
4. 自动生成配套发布文案（抖音版+小红书版+视频号版，不同语气）
5. 视频+文案打包推送到飞书群"沫竹·618大促"，并附"一键复制文案"
6. 保存视频到"E:\沫竹Mozoo\618\倒计时视频\"

【需要记住的】
- 品牌调性+色板（整个21天复用）
- 每日视频的风格渐变策略（不要21天一模一样）
- 发布文案的三平台差异化
- 前一天用了哪个产品线，次日轮换下一个

【异常处理】
- 如果到了当天早上8点我还未审核前一天的视频→跳过，继续生成当天的
- 如果某个产品线的素材图缺失→自动替换为品牌logo+氛围图
```

**测试关注点**：

| 关注项 | 描述 |
|--------|------|
| 视觉渐变 | 三个阶段之间的视觉过渡是否自然（不是硬切） |
| 跨天一致性 | 品牌色/logo/整体调性21天是否保持 |
| 产品轮换 | 产品线是否按规律轮换（不重复不漏） |
| 倒计时正确 | 天数计算是否正确（跨月+最后一天=1天） |
| 三平台文案 | 抖音/小红书/视频号文案是否确实不同 |
| 紧迫感递进 | 最后3天是否能感知到视觉紧张感加剧 |


## 案例九：跨境电商·多语言产品视频自动本地化

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 跨境电商经营 |
| 第一筛 | 对话能做一条翻译版视频，但每周处理多语种×多SKU=无法手动 |
| 第二筛 | 无App能记住产品信息跨语种自动生成视频 |
| 第三筛 | ✅ 跨境电商卖家核心痛点：多语种内容成本 |
| PC-1 | ✅ 海外社媒管理工具+视频处理在PC端 |
| PC-3 | ✅ 需在多语种文案和多平台之间切换 |

### 造假背景

**卖家**："赛音Saiyin"，同上蓝牙耳机工厂。2026年开始做跨境电商，开了Shopee（东南亚）、Amazon（美国）、速卖通（全球）。国内版的带货视频做了中英双语字幕凑合用，结果东南亚退货率15%——后来才发现，印尼和泰国的消费者根本看不懂中英文字幕，只是觉得视频里的耳机"看起来不错"就买了，拿到手发现不知道怎么用。

**困境**：一条视频做成印尼语、泰语、越南语、英语四个版本——找翻译+重新配音+重新剪字幕，一条视频的本地化成本比原片还贵。不做多语种就没办法在各地区投放，做了又根本投不起。

### 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个多语种产品视频自动本地化任务：

【任务触发】每周一上午10:00（处理上周新增的产品视频）

【品牌/产品信息】（永久记住）
- 品牌：Saiyin（赛音）
- 产品：T10 真无线降噪蓝牙耳机
- 外观：白色圆柱形充电仓，哑光磨砂，耳机入耳式短柄
- 核心卖点：ANC主动降噪/30小时续航/蓝牙5.4/IPX5防水
- 中文原版视频路径："E:\Saiyin\产品视频\中文原版\"

【任务流程】
1. 扫描"E:\Saiyin\产品视频\中文原版\"，找到上周新增的视频
2. 对于每条新视频，提取其中的：
   - 口播台词/旁白文本
   - 中文字幕文本
   - 产品卖点关键词
3. 调用doubao-creative-video，用原视频作为参考素材，生成以下语种版本：
   - 印尼语版（目标市场：印尼，Shopee/TikTok Shop）
   - 泰语版（目标市场：泰国，Shopee/Lazada）
   - 英语版（目标市场：美国/菲律宾/马来西亚，Amazon/TikTok）
4. 每个语种版本要求：
   - 保留原视频画面结构和产品外观
   - 口播/旁白翻译为对应语种（非机翻感，需要口语化）
   - 字幕替换为对应语种（保留原字幕的字体风格方向）
   - 品牌名/产品型号/参数数值保持原文不翻译
   - 15秒、9:16竖版
5. 按"产品_语种_版本号"命名（如"T10_ID_001.mp4"）
6. 分别保存到"E:\Saiyin\产品视频\{语种}\"
7. 生成本周处理报告（原视频数×语种数=总产出数），推飞书

【需要记住的模板】
- 品牌名"赛音/Saiyin"在所有语种中保持原样
- 产品型号"T10"保持原样
- 参数数值（30小时、ANC、蓝牙5.4、IPX5）保持原文
- 印尼语版：口语化、亲切、贴近TikTok风格
- 泰语版：有礼貌但不生硬
- 英语版：自然Tech Review风格

【异常处理】
- 如果原视频中口播文本无法提取→只用字幕文本做翻译
- 如果某个语种翻译质量明显差→标记为"需人工复核"不自动入库
```

**测试关注点**：

| 关注项 | 描述 |
|--------|------|
| 产品外观保真 | 多语种版中产品外观是否与原视频一致 |
| 翻译口语化 | 各语种是否像真人说话而非机翻 |
| 品牌名/参数保留 | "Saiyin""T10""30小时"等是否保持原文 |
| 字幕同步 | 新语种字幕的时间轴是否与原字幕对齐 |
| 批量处理 | 多条视频×4语种的处理稳定性和成功率 |


## 案例十：电商运营·每日店铺主图自动AB测试轮换

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 电商运营优化 |
| 第一筛 | 对话能做一次AB图，但无法每天自动读取后台数据做决策+自动换图 |
| 第二筛 | 无App能自动读店铺后台+自动生成新主图+自动换图 |
| 第三筛 | ✅ 主图点击率=流量×转化率的第一道门 |
| PC-1 | ✅ 店铺后台在PC端操作 |

### 造假背景

**运营**：小林（同案例四），天猫家电代理商。她负责的吸尘器店铺有一款主力SKU"轻量无线吸尘器M200"，月销2000+台。主图用了半年了——白底+产品+一行字"15KPa大吸力·45分钟续航"。

她一直怀疑这张主图还能优化——要不要试试"手持吸尘器吸沙发上的猫毛"场景图？要不要换个颜色？要不要把"15KPa"改成"比有线还猛"的感性文案？但这些想法只是想法——因为她没有时间一张张做图、一张张替换、一周后看数据判断哪张好。

### 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个店铺主图AB测试自动轮换任务：

【任务周期】每周为一个测试周期，持续运行

【任务触发】每周一早上8:00

【任务流程】
1. 读取上周天猫后台的主图点击率数据，识别当前主力SKU（销量Top3）
2. 对每个主力SKU，调用doubao-creative-design生成两张新主图变体：
   
   变体A·场景痛点型（区别于现主图的白底产品照）
   - 展示产品在真实使用场景中（如吸尘器吸沙发猫毛）
   - 中文卖点大字：感性+具体利益，如"养猫家庭的救星·每天吸出一只猫"
   - 比例1:1，淘宝主图规范
   
   变体B·数据信任型
   - 白底+产品+数据可视化图表（如吸力对比/续航对比柱状图）
   - 中文卖点：理性+权威感，如"真实用户测试·吸力保留率96%"
   - 比例1:1
   
3. 制定本周轮换计划：
   - 周一：现主图（对照组）
   - 周二-周四：变体A
   - 周五-周日：变体B
4. 按计划自动替换主图（通过天猫后台或提醒我手动替换）
5. 每周日晚上生成测试周报：
   - 三张主图各自的点击率、转化率、加购率
   - 统计显著性判断（是真实差异还是随机波动）
   - 建议："变体A点击率高12%且统计显著→建议设为默认主图"
6. 推送到飞书群"电商运营·每日早会"

【需要记住的模板】
- 当前主力SKU列表+产品信息
- 品牌色+排版风格
- 历史测试记录（避免重复测试已失败的方案）
- 天猫主图规范（1:1、无价格、主卖点文字居中偏上）

【异常处理】
- 如果上周数据不足以判断显著性→自动延长测试周期为两周
- 如果后台数据无法自动读取→推送提醒"请手动提供上周主图数据"
- 如果变体生成质量不佳→不自动替换，标记为"待人工审核"
```

**测试关注点**：

| 关注项 | 描述 |
|--------|------|
| 数据读取 | 从天猫后台自动读取的点击率数据是否准确 |
| 主图创意质量 | 生成的变体是否确实和原主图有明显差异（不是微调版） |
| 替换机制 | 是否能成功替换店铺主图（或至少生成可直接上传的图+提醒） |
| 周报分析 | 统计显著性判断是否合理（不只比数字大小） |
| 长期记忆 | 是否记住了之前测过的失败方案→不再生成同类方案 |


## 案例十一：品牌方·每周社媒视觉内容日历自动生成

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 品牌营销运营 |
| 第一筛 | 对话能做一张海报，但每周做7天×3平台的视觉日历+多产品轮换，对话记不住上周发了什么 |
| 第二筛 | 无App能跨平台跨周管理视觉内容日历 |
| 第三筛 | ✅ 所有消费品牌都需要社媒内容 |
| PC-1 | ✅ 设计排版+日历管理在PC端 |
| PC-2 | ✅ 多平台×多天=信息密度高 |

### 造假背景

**品牌**："唤醒WakeUp"（接案例二），冷萃咖啡液品牌。创始人小陈只有一个市场人员（兼客服兼发货）——就是他自己。

**困境**：每个周日晚上是他最恐惧的时刻——需要排下周7天的社媒内容。小红书3条+朋友圈5条+公众号1篇=9个内容单元，每个都需要配图。他用Canva自己做，周日晚上从9点做到凌晨2点——做完最后一张图的时候，经常已经不知道第一张图长什么样了，整周内容的视觉连贯性全靠"差不多就行"。

### 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个每周社媒视觉内容日历自动生成任务：

【任务触发】每周日下午4:00

【品牌信息】（永久记住）
- 品牌：唤醒WakeUp
- 品牌色：活力橙#FF6B35、米白#FAFAF5、深灰#2D2D2D
- 品牌调性：温暖、年轻、有能量但不浮躁。像"早上第一杯咖啡"的感觉
- 产品：冷萃咖啡液10条装（1条=1杯）
- 核心卖点：10倍浓缩、0糖0脂、3秒即溶

【内容日历模板】
每周固定内容结构：

周一 · 产品种草（小红书3:4竖版）
- 主题：周末喝了什么/本周咖啡预备
	
周三 · 生活方式（小红书3:4竖版）
- 主题：办公室/居家喝咖啡的治愈时刻
	
周五 · 福利互动（小红书1:1正方+朋友圈1:1）
- 主题：周末咖啡局/抽奖/互动话题

每天 · 朋友圈日常（1:1正方，共5条）
- 周一：产品日常+"这周的咖啡已备好"
- 周二：使用场景+"开会前15秒搞定一杯"
- 周三：咖啡知识+"冷萃和热萃到底什么区别"
- 周四：用户晒图+"被返图种草了自己卖的东西"
- 周五：周末预告+"周末想带哪条出门"

周日 · 公众号封面+内页头图（16:9横版）
- 主题：本周咖啡灵感/冷萃知识/新品剧透

【任务流程】
1. 读取上周发布的内容清单→本周自动轮换产品角度（不重复上周的卖点角度）
2. 调用doubao-creative-design，为每个内容单元生成配图：
   - 小红书配图：生活化场景+品牌色点缀+手写感轻文字
   - 朋友圈配图：更随意、更"像随手拍的"、轻修图感
   - 公众号封面：品牌设计感、大留白、强文字对比
3. 所有配图保持跨平台的品牌色+logo统一
4. 打包为"下周内容日历"文件夹（按日期+平台命名）
5. 推送飞书："下周视觉日历已生成，共9张图。建议本周内容角度：{本周角度}。上周已用角度：{上周角度}"
6. 同步推送到手机——小陈可以直接从手机保存发

【需要记住的】
- 品牌色板+排版风格
- 上周发布的内容角度（本周轮换避免重复）
- 各平台的视觉规范差异
- 固定栏目结构（周一/周三/周五/周日）

【异常处理】
- 如果下周有节日（618/双11/元旦）→调整内容结构优先做节日内容
- 如果某张图生成质量不佳→标记为"需人工调图"，但不阻塞整个日历输出
```

**测试关注点**：

| 关注项 | 描述 |
|--------|------|
| 跨平台视觉一致性 | 小红书/朋友圈/公众号的图是否共享品牌色板 |
| 角度轮换 | 本周卖点角度是否与上周不同 |
| 平台风格差异 | 小红书图≠朋友圈图≠公众号图（确实三种风格） |
| 日历可用性 | 生成的9张图能否直接发布（不需要二次设计） |
| 节日识别 | 是否自动识别了下周节日并调整内容 |


## 案例十二：设计工作室·客户多方案对比展示页自动生成

### 过筛

| 筛 | 判断 |
|----|------|
| 第0筛 | ✅ 设计服务经营 |
| 第一筛 | 对话能做一次排版，但每个客户每次提案都手动排版=每周重复劳动 |
| 第二筛 | 无App能记住客户品牌色+提案模板 |
| 第三筛 | ✅ 任何设计工作室/freelancer都有提案需求 |
| PC-1 | ✅ 设计提案排版在电脑端大屏操作 |
| PC-2 | ✅ 多方案×多屏=信息密度高 |

### 造假背景

**工作室**："野格设计事务所"（虚构），3人小型品牌设计工作室。接logo设计、包装设计、品牌VI。每个月要做4-6次客户提案。

**困境**：每次提案，设计师阿野需要把2-3个方案排成对比展示页——左边方案A、右边方案B、中间标注差异点。这个排版工作每次一模一样（只是图不一样），但每次都要花费1-2小时手动排版。最崩溃的是——提案前一刻客户说"能不能把方案B的配色也试一试方案A的排版"，然后所有排版要重做。

### 场景测试

**提示词（任务模式配置）**：

```
帮我创建一个客户多方案对比展示页自动生成任务：

【任务触发】当我在飞书群"野格·设计提案"中说"提案对比：{客户名}"

【任务流程】
1. 读取指定客户文件夹下的方案素材图
2. 识别方案数量（通常2-3个）
3. 调用doubao-creative-design，基于该客户的品牌信息生成提案展示页：

   展示页结构（多屏信息图风格，按屏次生成）：
   
   屏一·封面
   - 客户品牌名+项目类型（Logo设计/包装设计/VI）
   - 设计事务所logo在底部
   - 设计日期
   - 风格：极简专业、大面积留白、品牌色
   
   屏二·方案对比总览
   - 3个方案并排展示（每个占画面1/3宽度）
   - 每个方案下方标注：方案名+1句设计理念
   - 风格：白底+清晰对比网格
   
   屏三·方案A详解
   - 方案A大图居中+设计理念展开
   - 适用场景建议（标牌/包装/App图标等小尺寸mockup）
   
   屏四·方案B详解
   - 同上结构，但配色/排版方向与屏三区分
   
   屏五（如有方案C）·方案C详解
   - 同上
   
   末屏·下一步
   - 反馈收集方式（在线文档链接/群内讨论/下次会议时间）
   - 设计事务所联系方式

4. 打包输出PDF/长图格式
5. 推送到客户提案群或保存到客户文件夹

【需要记住的模板】
- 每个客户的品牌色/偏好排版风格（不同客户可以不同）
- 设计事务所的标准提案模板结构
- 字体/字号/间距标准

【异常处理】
- 如果客户文件夹为空→提醒"请先上传方案素材图"
- 如果只有1个方案→不生成对比页，改为单方案深度展示
```

**测试关注点**：

| 关注项 | 描述 |
|--------|------|
| 客户信息记忆 | 不同客户的品牌色/风格是否准确调取 |
| 方案布局清晰度 | 多方案对比是否一目了然 |
| 专业感 | 整体提案页的设计质量是否达标（比设计师手动排版更好看或至少持平） |
| 输出格式 | PDF/长图的输出是否可直接发给客户 |
| 个性化 | 不同客户的提案页是否有差异化（不是换个logo的同一模板） |
