### 案例1：连锁餐饮·每周新菜海报+外卖平台装修图自动更新

**Skill**：doubao-creative-design → media-creative.md

**为什么匹配**：media-creative明确覆盖"朋友圈/公众号"赛道。外卖平台店铺装修图本质是"朋友圈风格的产品海报+信息图"，在media-creative的能力边界内。

**造假背景**："唐点"广式点心连锁（虚构，12家门店）。市场部1人——小杨。每周三出新菜（2-3款），需要做：美团/饿了么店铺头图更新+新菜海报（发朋友圈+门店屏幕）+外卖商品图更新。一套流程每周耗4小时，周三成了小杨的"通宵做图夜"。

**任务模式 vs 对话模式**：对话=每周手动输入3款新品→各出3张图=9次操作。任务模式=每周三早9点自动读取新品表→一次出9张→按平台归档→推群。

**提示词**：

```
创建每周新菜自动出图任务：

【路由】doubao-creative-design → media-creative

【持久化品牌信息】
- 品牌：唐点·广式点心
- 品牌色：朱砂红#C43A30、米白#FAF5F0
- 风格：传统手艺+现代质感。不网红、不老气

【触发】每周三早9:00

【流程】
1. 从飞书表格"唐点·每周新品"读取本周新品（菜品名+照片+一句话卖点+价格）
2. 对每个新品生成三张图：

   图A·朋友圈新菜海报（3:4竖版）
   - 食物特写+蒸汽/光泽+品牌色点缀
   - 标题：菜品名+手写感字体
   - 左下角小字："本周新品·现已上线"
   
   图B·外卖平台头图（16:9横版）
   - 美团/饿了么店铺头图规范
   - 菜品居中偏左，右侧留白放logo
   
   图C·外卖商品图（1:1正方）
   - 白底+菜品+底部品牌标签

3. 按"菜名_图类型_日期"命名→保存到"唐点·营销素材/新品/"
4. 推飞书"唐点·门店运营"：9张图+一键保存指引

【异常】如果本周无新品→推送"本周无新品，跳过"
```

**测试关注点**：食物质感保真（虾饺的皮/流沙包的流心/凤爪的胶质）、三图品牌一致性、文字在缩小到外卖商品图尺寸后的可读性

### 案例2：房产中介·每日新房源短视频自动生成

**Skill**：doubao-creative-video → Marketing路线

**为什么匹配**：Marketing路线支持"产品展示大片"。房源视频=把房子当"产品"，展示空间+光线+动线——在Marketing的"产品主导、广告级镜头"能力边界内。

**造假背景**："安家找房"（虚构），北京朝阳区一家10人房产中介。每个经纪人每天新增2-3套房源，每套需要拍视频发朋友圈+抖音+贝壳。现在全靠经纪人自己拍——手抖、横屏竖拍、光线暗、配乐土。但请专业摄影一套3000，一个月新增30套=9万，根本投不起。

**任务模式 vs 对话模式**：对话=每套房手动写brief→出片→下载→发布。任务模式=每天早上自动读新增房源表→每套从0生成15秒展示短视频→经纪人只需要发。

**提示词**：

```
创建每日新房源短视频自动生成任务：

【路由】doubao-creative-video → Marketing

【持久化品牌信息】
- 品牌：安家找房
- 视频风格：清爽明亮、有空间感、像专业房产摄影师拍的
- 15秒结构：0-3s客厅全景→3-7s主卧→7-11s厨房/卫生间→11-15s户型卖点+价格标签+经纪人联系方式

【触发】每天早上9:00

【流程】
1. 从飞书表格"安家·新增房源"读取昨日新增房源
2. 对每套房源调用Marketing路线生成15秒竖版视频：
   - 取房源基本信息（户型/面积/楼层/朝向/核心卖点/价格）
   - 取房源照片作为product_ref
   - 旁白：3-5句中文女声，温柔专业。"XX小区，XX平米，朝南，阳光从早上9点照到下午4点。主卧带飘窗，厨房刚刚翻新过。358万。明天下午可以看房。"
   - slogan："安家找房·真实房源·每天更新"
3. 叠加价格标签+经纪人二维码在最后3秒
4. 命名"小区_户型_日期.mp4"→保存+推送

【异常】如果房源照片质量太差→标记"照片不足，建议重拍"
```

**测试关注点**：空间感是否正确（不会把60平拍成100平）、价格标签准确性、房源照片作为product_ref的产品一致性

### 案例3：短视频编剧·每周短剧剧本+分镜自动生成

**Skill**：doubao-creative-drama → planner→scriptwriter→storyboard

**为什么匹配**：drama skill专门做"短剧/剧情视频全流程"，从剧本到分镜到资产到关键帧到视频prompt。本案例只用到scriptwriter+storyboard阶段。

**造假背景**："快看短剧"（虚构），抖音10万粉短剧号。博主阿杰一个人写剧本+拍摄+剪辑+发布。每周更新2集，每集1分钟。最痛苦的是剧本创作——每周要想新剧情、写台词、画分镜，脑子被榨干。他需要的不是AI替他拍，而是AI帮他把"一个故事点子"扩展成"可直接拍摄的剧本+分镜表"。

**任务模式 vs 对话模式**：对话=每次手动输入故事点子→等AI写剧本。任务模式=每周一自动基于上周剧情续写本周2集剧本+分镜。

**提示词**：

```
创建每周短剧剧本自动续写任务：

【路由】doubao-creative-drama → scriptwriter → storyboard

【持久化设定】
- 账号：快看短剧
- 题材：都市情感+轻悬疑
- 风格：快节奏、每15秒一个反转
- 主角：林小棠（27岁女摄影师）+ 陆时寒（30岁建筑设计师）
- 当前剧情锚点：[每周更新]
- 每周产出：2集剧本+分镜表（不生成视频）

【触发】每周一早9:00

【流程】
1. 读取上周已发布剧本的最后500字→作为本周创作起点
2. 调用scriptwriter阶段生成本周2集剧本：
   - 每集60秒，9:16竖屏
   - 每集至少3个情绪转折
   - 对话口语化，不用书面语
3. 调用storyboard阶段为每集出分镜表：
   - 时间|画面|台词|镜头运动
   - 每集10-15个镜头
4. 推送到飞书，等待阿杰确认后再进入assets/frame阶段

【异常】如果上周剧本尚未确认→推送提醒，不续写
```

**测试关注点**：剧情连贯性（跨周续写是否和新剧情衔接）、台词口语化程度、分镜密度

### 案例4：电商运营·大促活动页自动生成+每日数据更新

**Skill**：doubao-app-builder → app_builder_agent

**为什么匹配**：app-builder的"网页应用生成"原生支持数据看板+活动页面。大促活动页=品牌色+倒计时+商品卡片+实时数据=一个需要每日更新的网页应用。

**造假背景**："云启电器"（虚构），天猫家电代理。每次大促做一个活动页——但要手动更新每日的"已抢XX件""距离结束XX小时""实时销量排行"。运营小林每天手动改HTML里的数字，烦到崩溃。

**任务模式 vs 对话模式**：对话=生成一次页面→每次更新重新找AI。任务模式=一次生成→每天早上自动更新实时数据→页面自动刷新。

**提示词**：

```
创建大促活动页自动生成+每日更新任务：

【路由】doubao-app-builder

【持久化品牌信息】
- 品牌：云启电器
- 品牌色：深蓝#1a1a2e、亮橙#e85d04
- 活动：夏清仓·全场5折

【触发】活动开始前3天生成→活动期间每天早上8:00更新数据

【生成要求】
- 单页活动H5：倒计时模块+3款主推商品卡片+实时销量排行榜+优惠券领取按钮
- 移动端优先，9:16竖屏适配
- 顶部大号倒计时"距结束还有 X天X时X分"
- 中间3款主推商品：产品图+原价（划线）+活动价+进度条"已抢XX%"
- 底部实时销量Top5榜单
- 暖色调大促风格，不暗沉

【每日更新】早上8:00自动更新：倒计时、已抢百分比、销量排行榜

【异常】如果活动已结束→自动替换为"活动已结束"静态页
```

**测试关注点**：倒计时准确性、数据更新稳定性、移动端适配

### 案例5：基金经理·每日持仓组合可视化快报

**Skill**：doubao-visualization → html type="renderer"

**为什么匹配**：visualization skill明确支持"数据图表显著增益：趋势、占比、排名、多指标对比"。持仓组合可视化=净值曲线+行业分布饼图+个股权重柱状图+风险指标——完全在白名单内。

**造假背景**：老徐（接昨天案例十三/十四），管理500万自有资金。每天收盘后需要一个"今日持仓全景"——今天赚了多少、哪个板块贡献最大、哪只票拖后腿、仓位偏离了基准多少。以前用Excel手动更新，天天复制粘贴。现在想每天早上8点和下午3点半自动生成一份可视化快报。

**任务模式 vs 对话模式**：对话=每天手动问"帮我画个持仓饼图"。任务模式=一天两次自动出图→搭配文字摘要。

**提示词**：

```
创建每日持仓可视化快报任务：

【路由】doubao-visualization → html renderer

【触发】每个交易日 早8:30（开盘前快照）+ 下午3:30（收盘后快照）

【内容模块】（每次生成一个renderer，包含以下4个模块）
1. 今日收益卡片：总资产+日收益+日收益率+累计收益——大字，醒目
2. 行业分布饼图：ECharts环形图，按行业分（AI应用/低空经济/消费电子/现金）
3. 个股权重柱状图：中科创达/亿航智能/立讯精密——柱状标注仓位%+今日涨跌
4. 风险指标条：最大回撤/夏普比率/波动率——三个指标横向排列

【设计约束】
- 深色主题、数据看板风格、PC宽屏
- 不超300行，超了拆模块
- CDN用jsDelivr
- 不写死高度，内容自然撑开

【数据来源】飞书表格"老徐·持仓数据"，每日更新行情和仓位

【异常】如果数据源未更新→推送提醒，用昨日数据+标注"数据截至{X日}"
```

**测试关注点**：数据准确性（从飞书表格取数）、ECharts渲染稳定性、深色主题可读性

### 案例6：品牌公关·每日品牌舆情自动监控

**Skill**：doubao-sentiment-tracker → 五步舆情工作流

**为什么匹配**：sentiment-tracker是专门做"舆情监控+品牌声量追踪"的skill。每日自动跑=天然适合任务模式——品牌不可能每天手动搜自己的舆情。

**造假背景**："沫竹Mozoo"（接昨天案例），日系女装品牌。最近在小红书被一个3万粉博主吐槽"面料和描述不符"——帖子点赞2000+，但品牌的社交媒体运营过了三天才发现，已经发酵了。老板阿琳说"以后再也不能这么被动"。

**任务模式 vs 对话模式**：对话=每次手动搜"沫竹Mozoo 评价"。任务模式=每天早上自动全网扫描→按正负向归类→推送摘要+原帖链接。

**提示词**：

```
创建每日品牌舆情自动监控任务：

【路由】doubao-sentiment-tracker → 五步法

【持久化监控对象】
- 品牌：沫竹Mozoo
- 关键词：沫竹、Mozoo、沫竹女装
- 重点平台：小红书、微博、抖音、知乎
- 时间窗：最近48小时

【触发】每天早上8:30

【流程】
1. 全网检索品牌关键词+相关话题
2. 重点平台深度浏览
3. 筛选：互动量>50或粉丝>1000的帖子
4. 强制溯源：每条必须可追溯到原帖链接
5. 输出：按正负向归类+详细列表（带链接）

【推送格式】
📊 {日期} 沫竹Mozoo 品牌舆情日报

正向舆情：
- [原帖链接] 用户晒单·面料惊喜 | 👍203 | 正向
- ...

负向舆情（需关注）：
⚠ [原帖链接] 面料和描述不符 | 👍2100+ | 负向 | 建议回复
- ...

本月趋势：正向占比X%，负向占比Y%
昨日新增：正向+3条，负向+1条

【异常】如果昨天无新增舆情→推送"昨日无新增品牌舆情"
```

**测试关注点**：原帖链接完整性（sentiment-tracker硬红线）、正负向归类准确性、平台覆盖率

### 案例8：私募销售·每日客户晨报自动生成

**任务模式 vs 对话模式**：对话=每次手动问"今天市场有什么"。任务模式=自动出7段叙事链晨报→人只审核→一键发客户。

**提示词**：

```
创建每日客户晨报任务：

【持久化风格】
- 品牌：磐石资产
- 客户画像：高净值个人，有基本投资常识
- 风格：专业但不冰冷、说人话、有判断但不夸张

【触发】交易日早7:30

4. 同步保存到客户推送文件夹

【异常】如果外盘数据未更新→标注，优先用A股信息
```

**测试关注点**：事实来源可追溯、7段叙事链完整性、"磐石观点"是否基于skill分析而非自己编

### 案例9：基金投顾·每周持仓股技术面扫描

**Skill**：doubao-daily-stock → 对话框日报

**造假背景**：小周，某券商基金投顾，服务50个客户。每周要给客户做一次"持仓回顾+下周展望"——5只推荐标的的技术面、消息面、机构评级变化。以前每周五下午手动查5只票=3小时。想把"每只票的日报"自动生成→她只需要做组合层面的总结。

**任务模式 vs 对话模式**：对话=每周手动查5次。任务模式=每周五收盘后自动跑5只→打包→她只写组合总结。

**提示词**：

```
创建每周持仓技术扫描任务：

【路由】doubao-daily-stock

【持久化推荐池】
- 中科创达 300496 | 关注：AI agent业务+汽车电子
- 亿航智能 EH | 关注：适航证+交付量
- 立讯精密 002475 | 关注：苹果订单毛利率
- 德赛西威 002920 | 关注：新定点项目
- 科大讯飞 002230 | 关注：星火大模型收入
```
