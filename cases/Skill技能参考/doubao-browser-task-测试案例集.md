# browser-task Skill 测试案例集

> 生成时间：2026-06-29
> Skill本质：真实浏览器GUI环境下的兜底执行器——专门解决"只能在网页上鼠标键盘操作才能完成"的任务
> 与其他skill的关系：sentiment-tracker依赖browser-task浏览社交媒体、finance-sector依赖browser-task爬取行情
> 白名单站点：淘宝/天猫、微博、小红书（有专属子Skill，走优化流程）+ 其他站点（走通用GUI流程）

## 案例一：小红书竞品笔记采集（核心·白名单站点）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **触发方式** | 命中白名单"小红书"→进入references/xiaohongshu-skill.md→走站内搜索→采集→去重→输出 |
| **目标人群** | 市场/运营/产品——需要了解小红书上关于某个话题/产品/品牌的真实用户内容 |
| **核心价值** | 手动翻小红书=搜索结果不全+无法批量导出+容易重复。AI自动翻页、去重、结构化输出 |

### 二、测试提示词

```
帮我采集小红书上关于"豆包任务模式"的笔记。

要求：
- 搜索关键词"豆包任务模式"
- 采集最近一个月的笔记
- 每条记录：标题、作者、点赞数、收藏数、评论数、发布时间、笔记链接
- 优先高互动量的笔记
- 至少采集15条
- 内容不要重复
```

### 三、预期Skill路由

```
用户prompt → 命中白名单"小红书" → 进入 xiaohongshu-skill.md
↓
读取 xiaohongshu/collect.md（采集子流程）
↓
Prompt 拆解：主任务=采集笔记、次任务=筛选互动量、约束=最近一个月+15条+去重
↓
浏览器操作：打开小红书→搜索"豆包任务模式"→按时间筛选→滚动加载→逐条提取
↓
去重：URL去重+内容相似度去重
↓
输出：结构化Markdown报告，含每条笔记的完整字段
```

### 四、测试关注点

| # | 关注点 | 通过标准 | 常见错误 |
|---|--------|---------|---------|
| 1 | **触发分发** | 正确分发到xiaohongshu-skill而非走通用GUI流程 | 走通用流程，没有使用小红书专用的采集优化 |
| 2 | **采集数量** | ≥15条，能自动翻页加载更多 | 只拿了首页几条就说"够了" |
| 3 | **信息完整性** | 每条含标题+作者+互动数+时间+链接 | 缺少链接或互动数据 |
| 4 | **去重** | 同URL不重复出现，相似内容合并 | 同一篇笔记出现两次 |
| 5 | **时间筛选** | 按用户要求的"最近一个月"筛选 | 混入了半年前的内容 |
| 6 | **输出格式** | Markdown结构化表格/列表 | 散乱文字堆砌 |


## 案例二：淘宝跨店比价+选靠谱款（白名单+决策辅助）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **触发方式** | 命中白名单"淘宝/天猫"→进入taobao-skill.md |
| **核心价值** | 在淘宝搜一个品类→帮用户对比价格/销量/店铺信誉→筛选"靠谱的款"→生成购买清单+备选清单+待确认问题 |

### 二、测试提示词

```
帮我看看淘宝上"机械键盘 无线 87键"这个品类。

要：价格300-600元之间、销量高、旗舰店优先。
帮我找5款靠谱的，列一下每款的价格、店铺、销量、评分、是否有运费险。
另外如果有特别便宜的但又不太靠谱的，也列出来提醒我避开。
```

### 三、预期输出结构（taobao-skill的三段式交付）

```
🛒 推荐购买清单（5款）
| 商品名 | 价格 | 店铺 | 销量 | 评分 | 运费险 | 推荐理由 | 链接 |

📋 备选清单（2-3款）
（条件基本符合但有minor瑕疵的）

❓ 待确认问题
- "轴体类型"没有统一标注，需要你确认喜欢红轴/青轴/茶轴？
- 有一款标注"87键"但评论区有人说是84键，需确认
```

### 四、测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **价格筛选** | 严格在300-600区间 | 混入299或601的产品 |
| **旗舰店识别** | 正确标注是否为官方旗舰店 | 把普通店铺标为旗舰店 |
| **三段式输出** | 推荐清单+备选清单+待确认问题 | 只给了推荐清单 |
| **避坑提醒** | 标注了"便宜但可能不靠谱"的选项 | 没有风险提示 |


## 案例三：微博搜索话题并采集高互动帖子（白名单+批量）

### 二、测试提示词

```
帮我在微博上搜"豆包桌面端"这个话题，采集最近一周的高互动帖子。

要求：
- 至少10条
- 优先互动量高的（转发+评论+点赞多的）
- 每条记录包含：发帖人、内容摘要、转发/评论/点赞数、原帖链接
- 区分原创和转发
```

### 三、测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **微博域名链接** | 链接必须是weibo.com/[uid]/[id]格式 |
| **互动量筛选** | 优先展示高互动帖而非简单按时间排列 |
| **原创/转发区分** | 标注"原创"或"转发"，转发贴不重复计算 |


## 案例四：批量点赞/收藏小红书笔记（白名单·交互操作）

### 一、案例设计

> 测"登录态操作"——这是browser-task区别于搜索工具的独特价值。需要用户已登录小红书。

### 二、测试提示词

```
帮我在小红书上搜索"豆包任务模式"，然后给前10条笔记都点赞+收藏。
先展示搜索结果让我确认，确认后再批量操作。
操作完后告诉我完成情况（哪些成功、哪些失败、为什么）。
```

### 三、测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **先展示后操作** | 用户确认后才执行，不是搜完就直接点赞 |
| **操作验证** | 点赞/收藏后检查是否生效（刷新确认） |
| **失败处理** | 某条失败时说明原因并继续执行下一条 |
| **完成报告** | 结束时给"成功X条/失败Y条/原因" |


## 案例五：通用网站表单批量填写（非白名单·通用GUI流程）

### 一、案例设计

> 测非白名单站点的通用流程——没有专属子Skill优化的兜底方案

### 二、测试提示词

```
帮我把这个Google表单填5份。
表单链接：https://docs.google.com/forms/d/e/xxx（换成实际测试表单）
填写数据如下：
姓名：张三、李四、王五、赵六、钱七
部门：技术部、产品部、设计部、市场部、运营部
满意度：5、4、3、4、5
每份填完后提交，然后再填下一份。
```

### 三、测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **通用流程** | 走通用GUI流程而非硬套白名单子Skill | 对非白名单站点用了淘宝的小红书的专属规则 |
| **逐份提交** | 填一份→提交→确认成功→填下一份 | 连续填5份一起提交 |
| **表单元素定位** | 正确识别input/radio/select/button | 找不到提交按钮或填错字段 |


## 案例六：任务模式——定时采集+价格监控

### 一、案例设计

> browser-task作为任务模式的执行器——每天定时采集、对比、推送变化

### 二、测试提示词

```
以后每天早上9点，帮我在淘宝搜"iPhone 15 Pro Max 256GB"，
记录前5个搜索结果的价格和店铺。
如果某款价格比前一天降了超过50元，提醒我。
今天先跑一次看看效果。
```

### 三、测试关注点：跨天对比是否能正确记住昨天的价格并计算出变化
