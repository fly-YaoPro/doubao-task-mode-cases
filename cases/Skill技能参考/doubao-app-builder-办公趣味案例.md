# doubao-app-builder 办公场景高吸引力案例

> 设计原则：办公刚需 × 趣味交互 × 明显高于Excel/在线文档的体验 × 截图分享欲望

---

## 四个办公趣味案例

| # | 案例 | 实用维度 | 趣味维度 | 吸引力来源 |
|---|------|---------|---------|-----------|
| 9 | 会议成本计算器 | 让所有人感知"开会很贵" | 实时跳动的金额+"够买XX杯奶茶了"的换算 | 戳中全民痛点——无效会议 |

## 案例九：会议成本计算器

### 为什么选这个

无效会议是打工人第一痛点。一个实时显示"这场会已经花了公司多少钱"的计时器——既是实用工具，也是行为改变的触发器，还能成为团队内部传播的meme。

### 测试提示词

```
帮我做一个会议成本计算器。

【核心功能】
- 顶部：大号实时计时器（从"会议开始"按钮点击起计时），数字用等宽大字体，像倒计时炸弹
- 计时器下方：实时跳动的费用数字——"本场会议已花费 ¥XXX"
- 费用 = 参会人员时薪总和 × 会议时长（按分钟折算）
- 每个参会人预设时薪：总监级 ¥500/h、经理级 ¥300/h、普通员工 ¥150/h、实习生 ¥50/h
- 费用数字跳动时有颜色变化：¥0-500绿色、¥500-2000黄色、¥2000+红色

【参会人管理】
- 可以添加/删除参会人（姓名+级别下拉选择）
- 预设快速选择："全员大会"=全公司、"技术评审"=技术部8人、"产品周会"=产品部6人
- 添加新参会人时，人数和总费用的变化有微小弹跳动画

【趣味换算——会议成本的冲击力】
- 费用数字下方实时显示等价换算（选最有冲击力的）：
  ¥500 → "够买 25 杯奶茶了"
  ¥1200 → "够吃一顿人均150的火锅了（8人）"
  ¥3000 → "够买一张回家的高铁票了（北京→上海）"
  ¥8000 → "够给团队每人买一本好书"
- 换算文案每30秒自动切换一条，用小字浅灰展示

【会议结束时】
- 点击"结束会议"→弹出结算面板：
  - "本场会议耗时 47 分钟，花费 ¥2,350"
  - "够买 117 杯奶茶 | 够15人吃火锅 | 够2张高铁票"
  - "今天你开了 3 场会，共花费 ¥6,820"
  - 一个"😱"表情大号居中

【视觉要求——非常重要】
- 深色背景（#111827），数字用白色/绿色/黄/红四级变色
- 计时器用大号等宽（72px mono），费用数字（48px bold）
- 参会人用圆形头像占位符+名字+时薪小标签
- 整体像飞机驾驶舱的仪表感+黑色科技风
- 移动端竖屏时计时器和费用在上方，参会人列表在下方
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **计时逻辑** | 开始→计时→暂停→继续→结束，流程完整 | 暂停后恢复时间跳变 |
| **费用计算** | 实时按分钟折算，颜色随金额变化 | 计算精度错误（如用整数小时而非分钟） |
| **趣味换算** | 有奶茶/火锅/高铁票等接地气换算 | 只有数字没有换算 |
| **结束面板** | 结算弹窗有总结+换算+今日累计 | 关闭后什么都看不到 |
| **快速选择** | 预设"全员大会"自动填充18人 | 预设不生效 |


## 案例十：团队心情/能量签到板

### 为什么选这个

远程办公后，管理者最大的焦虑是"不知道团队状态"。一个每天花5秒就能完成的心情签到——团队成员点一下、管理者看一眼，比任何周会都更实时地感知团队状态。关键是做得有趣，让人愿意每天点。

### 测试提示词

```
帮我做一个团队每日心情签到板。

【签到方式——像素风角色选择】
- 每个团队成员有一个固定的像素风小角色（8-bit风格，20×20px）
- 每天签到时，点击自己的角色→弹出心情选择面板：
  ⚡ 充满能量（金黄色）
  🙂 正常（绿色）
  😴 有点累（蓝色）
  🔥 压力山大（红色）
  🤒 不舒服请病假（灰色）
- 选择后角色颜色变成对应心情色，角色上方浮出一个气泡表情
- 已经签到的人角色会微微跳动一下，没签到的人是黑白的

【团队视图——像素风小镇】
- 所有团队成员的角色排列在一个像素风的小场景里（草地+天空背景）
- 按部门分成几个"房子"区域：技术部小木屋、产品部树屋、设计部画室...
- 每个部门区域上方显示"签到率 X/8"
- 鼠标hover到某个角色→显示名字+今日心情+连续签到天数

【管理者视图】
- 右下角小按钮切换到"本周趋势"视图
- 本周7天的心情热力图（每人一行，7列，绿色=好、黄色=一般、红色=差）
- 自动标注"需要关注"的成员（连续3天红色/灰色）
- 顶部一行摘要："今日签到 14/18 | 能量满满 8人 | 需要关注 2人"

【趣味细节】
- 连续签到7天的人角色出现小皇冠
- 周一签到时角色会"打个哈欠"（一个小动画）
- 周五签到时角色旁边出现"🎉"粒子
- 整个团队都签到完时，场景上方出现一行字"全员就位，今天加油！"

【视觉要求——非常重要】
- 像素风不能粗糙，要像Stardew Valley那种"精致的复古"感
- 背景色柔和：天空用渐变浅蓝+几朵像素白云+草地用像素绿
- 角色配色鲜艳，黑白未签到和彩色已签到形成强烈对比
- 心情选择面板是弹出气泡式（不是弹窗遮罩），圆角+微阴影
- 整体像一个小游戏，不像是"企业SaaS工具"
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **签到交互** | 点击角色→选心情→角色变色+气泡 | 签到后无视觉反馈 |
| **像素风质量** | 精致复古像素，不是粗糙马赛克 | 像素太大/太小看不出是人 |
| **热力图** | 7天×18人热力矩阵正确显示 | 颜色映射错误 |
| **连续签到** | 7天皇冠、周一哈欠、周五礼花 | 动画不触发 |
| **需要关注** | 连续3天红色/灰色自动标注 | 没有检测逻辑 |


## 案例十一：项目上线倒计时+里程碑庆祝

### 为什么选这个

每个团队都有"那个大日子"——产品上线、活动开始、客户交付。一个挂在团队群里的倒计时页面，不仅是实用工具，更是团队的情绪锚点。每完成一个里程碑触发庆祝动画——这是仪式感。

### 测试提示词

```
帮我做一个项目上线倒计时页面。

项目：豆包桌面端 V2.0 正式发布
目标日期：2026年8月15日 10:00

【倒计时主屏】
- 大号倒计时数字：DD天 HH时 MM分 SS秒（等宽字体，深色背景）
- 数字每秒更新，秒数变化时有微小的数字翻页动画
- 倒计时下方一行字："距离豆包桌面端 V2.0 正式发布还有"
- 背景色随剩余天数变化：>30天=冷静蓝、30-7天=专注紫、<7天=紧张橙、<24小时=红色脉冲
- <24小时时，数字显示到秒，并且加一个缓慢的红色脉冲动画

【里程碑追踪】
- 倒计时下方是7个里程碑，横向排列
  1. 需求冻结 ✅ 6月15日 已完成
  2. 设计定稿 ✅ 6月28日 已完成
  3. 前端开发完成 ✅ 7月12日 已完成
  4. 后端开发完成 ● 7月20日 进行中
  5. 联调测试 ○ 8月1日 待开始
  6. 内部验收 ○ 8月8日 待开始
  7. 正式上线 ○ 8月15日 待开始
- 已完成=绿色勾+完成日期、进行中=蓝色圆点+进度条、待开始=灰色空心圆
- 进度条上有一个小圆点在当前里程碑上跳动

【里程碑完成时的庆祝】
- 当管理员标记某个里程碑为"已完成"时：
  → 全屏飘落彩带/纸屑动画（2秒）
  → 该里程碑卡片做一个"弹跳+放大"动画
  → 顶部出现一行大字"🎉 联调测试完成！还剩3个里程碑！"
  → 3秒后动画消退

【团队留言墙】
- 倒计时下方有一个简易留言区
- 团队成员可以留一句话（不需要登录，输入名字+内容即可）
- 留言像便利贴一样排列，不同颜色
- 最新留言在顶部

【视觉要求——非常重要】
- 深色背景+大号倒计时为主体，像火箭发射倒计时
- 里程碑用时间轴样式：一条横线贯穿，圆点在线上，上方卡片
- 完成状态绿色+对勾、进行中蓝色+呼吸动画、待开始灰色
- 庆祝动画不过度（2秒彩带就够），不要一直飘
- 整体科技感+一点期待感，不是冷冰冰的"项目管理系统"
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **倒计时** | 数字每秒更新，翻页动画 | 不更新或跳秒 |
| **背景色变化** | 随剩余天数从蓝→紫→橙→红 | 不变色或颜色不对 |
| **里程碑状态** | 三种状态正确显示 | 进度条动画不流畅 |
| **庆祝动画** | 标记完成→彩带+弹跳+大字 | 动画不触发或无法标记 |
| **留言墙** | 输入名字+内容→便利贴展示 | 留言不显示或布局错乱 |


## 案例十二：办公室零食/下午茶投票系统

### 为什么选这个

"下午茶点什么"是每个办公室每周都要面对的灵魂拷问。微信群投票只能单选、看不到历史、有人刷票。一个AI生成的有趣投票器，能投票、能看历史、能追踪预算——高频使用+社交属性+实用。

### 测试提示词

```
帮我做一个办公室下午茶投票系统。

【本周投票】
- 4个候选选项（预设本周可投的品类，管理员可修改）：
  🧋 奶茶类（喜茶、奈雪、霸王茶姬...）
  🍰 甜品蛋糕（好利来、鲍师傅、泸溪河...）
  🍗 炸鸡小吃（麦当劳、肯德基、正新鸡排...）
  🥗 健康轻食（沙拉、水果捞、酸奶碗...）
- 每个选项一张卡片，左侧大图emoji+品类名，右侧投票按钮+当前票数
- 点击投票→票数+1+按钮变灰"已投"+轻微弹跳动画
- 每个品类下方有一行小字："上次点的是 喜茶·多肉葡萄（6月15日）"——避免重复

【本周结果公布】
- 到指定时间（如每周三下午3点截止），自动显示结果
- 得票最高的品类卡片放大+金色边框+顶部"🏆 本周下午茶：炸鸡小吃！"
- 第二第三名缩小显示在下方
- 如果票数相同→显示"平局！请老板决定"（趣味处理）

【历史记录】
- 底部折叠区"📜 下午茶历史"
- 展开后显示过往每周的选择：
  6月第3周：🧋 奶茶（14票）
  6月第2周：🍗 炸鸡（11票）
  6月第1周：🍰 甜品（9票）
  5月第4周：🥗 健康轻食（7票）
- 每一行右侧有一个小emoji标签："连吃三周奶茶了！""这周终于健康了"

【预算追踪】
- 顶部小卡片："本月下午茶预算 ¥2,000"
- 本月已花费 ¥1,280（64%），进度条橙黄色
- 平均每次 ¥320，还能再点2次
- 如果本月快超预算→进度条变红+"⚠️ 只剩 ¥180 了，这周省着点"

【视觉要求——非常重要】
- 轻松活泼的餐饮App风格，暖色调（橙色/暖黄为主色）
- 投票卡片大而清晰，留白充足
- 票数变化时有弹跳动画
- 历史记录折叠展开有平滑过渡
- 整体像"饿了么/美团"的投票精选页，不是灰色表单
```

### 测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **投票交互** | 每人一票、票数实时更新、已投按钮变灰 | 可以重复投票 |
| **历史重复提醒** | 上次点过的品类显示上次记录 | 历史数据不显示 |
| **截止公布** | 到时间自动显示获胜者+金色边框 | 无法模拟截止时间 |
| **预算追踪** | 花销从历史自动汇总，进度条正确 | 预算计算错误 |
| **趣味文案** | "连吃三周了""终于健康了"自动生成 | 文案硬编码不变化 |


## 四个案例吸引力排序

**案例九（会议成本）> 案例十二（下午茶投票）> 案例十一（上线倒计时）> 案例十（心情签到）**

- 案例九戳中全民痛点——没有打工人不恨无效会议。计时+跳动的费用+奶茶换算——这个截图会被转发到无数工作群
- 案例十二高频——每周都用，粘性最高，而且是"AI帮我做了个工具来点奶茶"这种轻松话题，传播门槛低
- 案例十一仪式感——上线/交付/活动倒计时是所有团队的通用刚需
- 案例十有温度——远程管理真实痛点，像素风设计讨喜，但使用频次是四个中最不确定的
