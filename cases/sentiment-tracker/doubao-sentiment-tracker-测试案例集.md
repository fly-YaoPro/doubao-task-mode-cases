# doubao-sentiment-tracker Skill 测试案例集

> 生成时间：2026-06-29
> Skill本质：PC端独占的五步舆情搜集与溯源工作流
> 核心红线：每条舆情必须有原始帖子直链、必须覆盖所有核心平台、先归类后详情、必须文档交付
> 任务模式价值：定时周期性舆情监控（每周/每月）+ 跨时间段舆情变化对比

---

## 案例一：品牌周度舆情监控报告（核心案例）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **技能触发** | "帮我搜一下""了解评价""追踪舆情""用户反馈收集"等——参见评估集10条应触发Query |
| **目标人群** | 品牌经理、市场运营、产品负责人——每周需要了解"大众怎么讨论我们品牌" |
| **核心痛点** | 手动搜舆情=打开微博搜一圈→知乎搜一圈→B站搜一圈→小红书搜一圈→手动复制粘贴到Excel→整理正负面→写结论。一期下来3-4小时。关键是手动搜容易漏平台、漏来源 |
| **任务模式价值** | "以后每周一早上，帮我扫描上周关于XX品牌的全网舆情，生成一份带原始帖子链接的报告" |
| **一句话传播** | "别人手动翻3个平台做舆情周报，AI帮你全网搜完、标好正负面、附上每个帖子的原始链接——一篇报告自动生成" |
| **可量化利益** | 每周省3-4小时手动搜集整理时间；原始链接可点击验证，不用担心"这数据哪来的" |

**叙事弧（30秒）：**
- 0-5s 痛苦：市场经理在微博搜索框输入品牌名→滚动翻页→复制帖子→打开知乎→再搜一遍→粘贴到Excel
- 5-15s 展示：她在豆包PC端说"帮我搜一下上周关于豆包任务模式的舆情"。AI自动打开微博、知乎，浏览帖子，查找原始链接
- 15-25s 结果：几分钟后生成一份文档——正面评价归类、负面评价归类、整体判断，每个帖子点进去就是原始链接
- 25-30s 结尾："下周一，她只要说'帮我做上周舆情'，报告自动生成"

### 二、测试背景信息（造假用）

**"豆包任务模式"上周舆情（模拟数据，用于验证输出格式）：**

用户在微博、知乎、即刻上发布了以下典型帖子（AI需搜索并溯源）：

| 平台 | 发帖人 | 内容倾向 | 大致内容 |
|------|--------|---------|---------|
| 微博 | @数字游民小李 | 正面 | "豆包任务模式太好用了，每天帮我自动搜补贴政策，省了不知道多少时间" |
| 微博 | @产品经理老王 | 正面 | "用了任务模式一周，最大的感受是不再忘事了，到点自动提醒" |
| 微博 | @科技吐槽君 | 负面 | "任务模式功能还太基础，不如Claude的scheduled task灵活" |
| 知乎 | @AI产品分析 | 中性偏正 | "深度用了两周，总结一下优缺点（长文分析）" |
| 即刻 | @独立开发阿明 | 中肯 | "比我预期的好用，但需要更多定制化配置" |

### 三、测试提示词

**提示词A（基础版——单平台明确搜索）：**
```
帮我在微博上搜一下"豆包任务模式"的相关讨论。
要最近一周的，帮我整理正面评价和负面评价分别有哪些。
每个帖子都要有原始链接。
```

**提示词B（进阶版——多平台舆情搜集）：**
```
帮我做一份舆情调研：看看最近一个月大家怎么讨论"豆包任务模式"的。
重点看微博和知乎，如果小红书、B站上有相关内容也帮我找一下。
我要看到：
- 正面评价有哪些（分几类）
- 负面评价有哪些（分几类）
- 每个帖子都要有原始链接能点进去看
- 整体舆情是正面为主还是负面为主
```

**提示词C（任务模式场景——设定周期性监控）：**
```
以后每周一早上，帮我做一份豆包任务模式的舆情周报。
扫描微博、知乎、即刻上的相关讨论，整理正负面评价。
这次先按这个范围做一份。
```

### 四、预期Skill路由

```
用户prompt → 触发识别："搜一下""讨论""评价"+"豆包任务模式"（产品舆情调研）
↓
第一步：全网搜索 → general_search → 获取广泛信息
↓
第二步：平台深挖 → 打开微博s.weibo.com → 搜索"豆包任务模式"
  → 打开知乎 → 搜索"豆包任务模式"
  → 打开即刻 → 搜索"豆包任务模式"
  → 每个平台浏览搜索结果、滚动加载、查看帖子详情
↓
第三步：内容筛选 → 按互动量/内容质量/时间筛选有价值的帖子
↓
第四步：强制溯源 → 逐条点击发布时间进入微博详情页
  → 复制 https://weibo.com/[uid]/[postid] 格式链接
  → 确认链接可访问、非搜索结果页、非用户主页
  → 如果帖子已删除 → 查找二手来源并标注原因
↓
第五步：输出 → 使用豆包文档输出
  → 先归类总结（正面/负面/中性，各分几类，核心观点）
  → 后详细列表（每条含：发帖人/时间/摘要/原帖链接/互动数据/来源类型）
↓
最终交付：文档 → 用户可打开查看
```

### 五、测试关注点

| # | 关注点 | 通过标准 | 潜在风险 |
|---|--------|---------|---------|
| 1 | **触发准确性** | 用户说"舆情""评价""讨论""反馈"时触发本skill，说"帮我写""帮我分析摘要"时不触发 | 可能把纯文档写作请求误触发为本skill |
| 2 | **平台覆盖完整性** | 即使用户只提"微博"，也必须额外覆盖其他核心平台（至少知乎） | skill跳过知乎，只做了微博 |
| 3 | **原帖链接质量** | 每个链接必须是https://weibo.com/[uid]/[postid]格式，不是s.weibo.com搜索结果页、不是用户主页 | 链接指向搜索结果页而非具体帖子 |
| 4 | **先归类后详情** | 输出必须先有正负向归类总结，再展开具体列表，顺序不可颠倒 | 直接输出列表，没有先给全貌 |
| 5 | **文档交付** | 默认使用豆包文档输出，除非用户明确要求"直接输出Markdown" | 直接在聊天区输出Markdown而非生成文档 |
| 6 | **移动端限制** | 手机端不触发skill，提示"电脑版或网页端效果更佳" | 手机端也触发了本skill |
| 7 | **二手来源标注** | 找不到原始帖子时必须标注"二手来源（原因：xxx）" | 找不到原帖就跳过不输出 |
| 8 | **互动数据准确性** | 转发/评论/点赞数与帖子页面一致 | 互动数据错误（如把转发数当评论数） |

## 案例二：新品发布48小时舆情追踪

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **技能触发** | 事件舆情追踪 |
| **目标人群** | 产品Launch团队、PR公关——发布后48小时内需要密集关注舆论反应 |
| **核心痛点** | 新品发布后的舆论窗口期极短——24小时内要了解"大家喜欢什么/讨厌什么/最大的槽点是什么"。手动刷平台效率太低，且容易错过关键帖子。 |
| **任务模式价值** | "发布后2小时、6小时、24小时、48小时各扫描一次，自动分时段对比舆论变化" |
| **一句话传播** | "新品发布会结束后，AI帮你盯着社交媒体——2小时一报、48小时内持续跟踪，哪个平台的负面情绪在升温第一时间知道" |

### 二、测试提示词

```
豆包刚发布了桌面端任务模式的新版本（支持PC端操作和vibecoding），今天是第一天上线的第二天。

帮我追踪从昨天发布到今天这48小时内，微博、知乎上的用户反馈：
1. 正面评价主要集中在哪些点？
2. 负面吐槽/bug反馈有哪些？
3. 和昨天刚发布时比，今天的舆论风向有没有变化？
4. 有没有人提到竞品对比（比如和Claude Code对比）？

每个帖子都要有原始链接。
```

### 三、测试关注点（补充案例一未覆盖的）

| 关注点 | 说明 |
|--------|------|
| **分时段对比** | 能否区分"发布第一天"和"第二天"的舆情变化趋势 |
| **竞品提及捕获** | 用户提到了"和Claude Code对比"，是否在搜索结果中主动标注竞品相关信息 |
| **Bug反馈敏感度** | 负面帖子中提到的bug/功能缺陷是否被单独分类（如"功能不完整的吐槽"vs"性能问题的反馈"） |
| **时效性优先** | 48小时内的帖子是否优先于更早的帖子 |

## 测试执行建议

### 现实限制说明

⚠️ **注意**：本skill需要**真实浏览器访问社交平台**（微博需要登录才能看完整搜索结果），且社交媒体搜索结果的**真实性不可控**（帖子随时可能被删除/设为私密/限流）。实际测试结果以skill在真实网络环境下的表现为准。

### 推荐测试步骤

**Step 1**：先用简单的品牌/产品名，测试skill触发的准确性
```
帮我搜一下微博上关于"豆包"的最近讨论
```

**Step 2**：验证输出格式是否满足"先归类总结、后详细列表、每条有原帖链接"
- 如果没有原帖链接 → 不合格，要求返工
- 如果只有微博没有知乎 → 不合格，要求补上核心平台
- 如果是搜索页链接而非具体帖子链接 → 不合格

**Step 3**：测试"找不到原帖"时的降级处理
- 故意提一个很少人讨论的小众话题
- 验证skill是否标注"二手来源"而不是跳过不输出

**Step 4**：测试移动端限制（用手机端豆包触发）
- 预期skill不触发，提示"电脑版效果更佳"

## 与已有能力模式的差异

| 维度 | 已有模式 | sentiment-tracker |
|------|---------|-------------------|
| 数据类型 | 结构化数据（GMV、价格、公告） | 非结构化舆情（帖子、评论、观点） |
| 输出形式 | 推送消息/看板/图表 | **结构化文档**（归类+列表+链接） |
| 操作深度 | 监控+抓取 | 搜索+筛选+**溯源**+归类+整理 |
| PC端独占 | 部分模式App端也可 | **仅PC端**（需要浏览器自动化） |
| 核心约束 | 无强制格式 | **强制原帖链接+先归类后详情** |
