# browser-task 办公采购场景测试案例

> 设计原则：所有案例必须"真的需要浏览器操作才能完成"——不能是API能做的、不能是搜索就能解决的
> 核心价值：采购流程中最耗时的"跨平台比价""招标信息采集""供应商资质核查"——全是手工翻网页的体力活

---

## 三个采购场景案例

| # | 案例 | 采购环节 | 核心能力 |
|---|------|---------|---------|
| 1 | 办公耗材多平台比价 | 询价比价 | 跨站点搜索→价格/起批/运费提取→对比表 |
| 2 | 政府/企业招标公告自动采集 | 寻源拓客 | 定时监控政府采购网→新公告提取→结构化汇总 |
| 3 | 供应商资质批量核查 | 供应商管理 | 天眼查/企查查批量查询→关键信息提取→风险评估 |

---

## 案例一：办公耗材多平台比价

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **采购场景** | 公司行政每月采购办公耗材——A4纸、墨盒、文件夹、笔——需要对比京东企业购、天猫、1688三个平台的价格和起批量 |
| **核心痛点** | 三个平台各搜一遍、手工记录价格、1688还要看起批量和运费——每月采购前花2小时做这件事 |
| **browser-task不可替代性** | 比价不是"搜一下"——1688的商品名称和天猫不一样、起批量规则不同、运费模板需要点到详情页才看得到。这些只有真实的浏览器操作才能完成 |

### 二、测试提示词

```
帮我做一个办公耗材的三平台比价。

采购清单（需要对比京东企业购、天猫、1688）：

1. A4复印纸 70g 500张×5包/箱
2. 惠普88A硒鼓（CC388A）
3. 得力文件夹 A4 蓝色 背宽55mm

要求：
- 京东企业购：看企业价和普通价的差异
- 天猫：看月销量和评分
- 1688：看起批量和运费（很多标了低价但3箱起批+运费50）
- 每条提取：商品名、价格、店铺名、起批量/运费、月销量、店铺类型（自营/旗舰店/普通店）
- 最后生成一个对比表，标出"推荐"和"避坑"
- 1688上如果有超低价但起批量巨大的，标注"适合大量采购"
```

### 三、测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **跨平台搜索** | 同一个商品名在三平台能正确搜索并找到对应商品 |
| **价格口径一致** | 标注清楚"单价/箱价/含税/不含税"的不同口径 |
| **1688起批量** | 正确提取起批量+运费模板（点详情页查看） |
| **企业价识别** | 京东企业购的"企业专享价"是否正确提取 |
| **避坑标注** | 超低价但起批量巨大的正确标注 |


## 案例二：政府/企业招标公告自动采集+汇总

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **采购场景** | 公司是做办公家具/IT设备的，需要每天盯政府/国企的采购招标公告。招标公告挂在政府采购网、公共资源交易中心、目标国企官网上——每天手动去三四个网站翻公告 |
| **核心痛点** | 招标公告有时效性——从发布到投标截止可能只有5-7天。漏看一天就可能错过一个标。而且不同网站的公告格式不一样，手工整理很繁琐 |
| **browser-task不可替代性** | 政府采购网没有API、没有RSS、没有邮件订阅。你想知道今天有什么新公告——只能打开浏览器去翻。这就是browser-task的完美场景 |

### 二、测试背景（造假用）

```
公司：杭州云帆办公家具有限公司
主营：办公桌椅、文件柜、办公屏风
目标客户：政府机关、国企、学校、医院
投标范围：浙江省+杭州市两级

需要监控的网站：
1. 浙江政府采购网 https://zfcg.czt.zj.gov.cn/
2. 杭州市公共资源交易中心 http://hzctc.hangzhou.gov.cn/
3. 国家电网电子商务平台 https://ecp.sgcc.com.cn/（国网浙江分公司）
```

### 三、测试提示词

```
帮我每天定时采集办公家具类的招标公告。

监控以下网站（浙江省内）：
- 浙江政府采购网
- 杭州市公共资源交易中心  
- 国网电子商务平台（浙江分公司）

搜索关键词：办公家具、办公桌椅、文件柜、办公屏风、办公工位

要求：
- 每天早上9点自动扫描一次
- 只采今天新发布的公告
- 每条提取：标题、采购单位、预算金额、投标截止日期、公告链接
- 预算超过50万的单独标黄提醒
- 如果某个公告的截止日期距今不足5天，标红"紧急"
- 生成一个汇总表格，按截止日期排序
```

### 四、测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **新公告识别** | 正确区分"今天新发布的"和"历史公告" |
| **关键词匹配** | 在标题和正文中搜索关键词，不只是标题匹配 |
| **金额提取** | 从公告正文中正确提取预算金额（格式不统一，有的是"预算金额：XX万元"，有的是"采购预算XX元"） |
| **紧急标注** | 截止日期<5天的正确标红 |
| **跨站汇总** | 三个来源的公告汇总在一张表里，按截止日期排序 |
| **重复公告** | 同一招标在不同网站可能都有公示，识别去重 |


## 案例三：供应商资质批量核查

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **采购场景** | 公司准备引入3家新供应商，供应商发来了营业执照和公司名称，采购需要核实：公司是否真实存在、有没有经营异常、有没有司法风险。这些信息分散在天眼查/企查查/国家企业信用信息公示系统里，一家一家手工查很慢 |
| **核心痛点** | 查供应商资质不是"搜一下公司名"——需要打开企查查看工商信息、打开国家企业信用系统看异常名录、打开中国裁判文书网看有没有官司。三个网站、每家公司查一轮，3家供应商就是9次操作 |
| **browser-task不可替代性** | 天眼查/企查查有反爬机制，无法用API。国家企业信用公示系统需要输入验证码。只能真实浏览器操作 |

### 二、测试背景（造假用）

```
待核查供应商：
1. 杭州新锐办公设备有限公司
2. 浙江博雅家具制造有限公司
3. 宁波天一办公用品有限公司

核查维度：
- 工商信息：注册资本、成立日期、经营范围
- 风险信息：经营异常、行政处罚、失信被执行人
- 司法信息：是否有买卖合同纠纷（供应商最怕这个）
- 资质认证：是否有ISO9001/ISO14001等
```

### 三、测试提示词

```
帮我核查3家备选供应商的资质。

供应商名单：
1. 杭州新锐办公设备有限公司
2. 浙江博雅家具制造有限公司
3. 宁波天一办公用品有限公司

核查平台：
- 国家企业信用信息公示系统（官方）——查工商基础信息+经营异常
- 天眼查——查司法风险、行政处罚
- 中国裁判文书网——查是否有买卖合同纠纷

要求：
- 每家生成一份"供应商资质核查卡"：
  ✅ 基础工商：公司全称、注册资本、成立日期、经营范围
  ⚠️ 风险标记：经营异常？行政处罚？失信？如果有→标红
  ⚖️ 司法风险：买卖合同纠纷案件数（如有→标红）
  📋 资质认证：ISO认证、行业许可证（如有→标绿）
  
- 最后给一个三家的对比总结："推荐/可考虑/建议排除"
```

### 四、测试关注点

| 关注点 | 通过标准 | 风险点 |
|--------|---------|--------|
| **跨平台查询** | 在3个平台都完成查询，不是只查了天眼查 | 只查了一个平台就说"已完成" |
| **风险标记准确** | 经营异常、失信、合同纠纷都正确标出 | 没发现明显的风险项 |
| **信息真实性** | 信息来自页面实际内容，不是编造 | 用记忆/常识填充了查不到的信息 |
| **验证码处理** | 国家企业信用系统弹出验证码时，暂停并提示用户介入 | 卡在验证码页面不动或直接跳过 |
| **对比结论** | 给明确的"推荐/可考虑/排除"建议+理由 | 只有信息罗列没有判断 |


## 三个案例的价值总结

| 案例 | 每次手工耗时 | browser-task耗时 | 年节省 |
|------|------------|-----------------|--------|
| 办公耗材比价 | 1.5-2小时（三平台各搜一遍+手工整理） | 约5分钟 | 每月一次=年省20小时 |
| 招标公告采集 | 每天30分钟（三个网站各翻一遍） | 约2分钟 | 每天一次=年省150小时 |
| 供应商核查 | 每家30分钟×3家=1.5小时 | 约10分钟 | 每季度引入新供应商时用 |
