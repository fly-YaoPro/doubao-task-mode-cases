# VibeCoding 案例：一句话生成个人数据追踪看板

> VibeCoding 定位：用户说需求 → 豆包生成完整可运行的本地应用 → 用户直接使用
> 与 app-builder 区别：app-builder 生成云端网页（公网URL），vibecoding 生成**本地直接跑**的应用（HTML双击打开或命令行启动），可读写本地文件、无部署成本
> 核心叙事："你描述，AI生成。不用写一行代码，就能拥有一个属于自己的工具。"

---

| 维度 | 内容 |

## 测试案例：每日目标打卡追踪器

### 为什么选这个

- 视觉效果天然好：进度圆环、连续打卡火焰动画、周统计热力图
- 实用：几乎每个人都需要，但没人会为这个专门下App
- 数据存本地：localStorage 无需后端，隐私安全
- PC端独占优势：生成的HTML在电脑上打开更自然（不是手机App场景）

### 案例场景

**用户身份**：一个想做日常习惯追踪的自由职业者——每天需要记录心情评分、专注时长、运动分钟、喝水杯数，想看到本周趋势和连续打卡天数。

**数据存储**：所有数据存 localStorage（浏览器本地存储），无需任何后端服务。用户换浏览器或清缓存会丢失，但这是明确告知的 tradeoff。

## 核心对话序列（VibeCoding 特色：多轮迭代生成）

### 【轮次1】一句话需求 → AI生成初版

**用户说**：
```
帮我做一个每日打卡追踪工具。

我想每天记录4个指标：
- 心情评分（1-5分，用星星图标）
- 今天工作了几个小时（专注时长）
- 运动了几分钟
- 喝了几杯水

要有输入面板和本周统计。
要好看，深色主题。
能做吗？
```

**预期豆包行为**：不解释"能"，直接开始生成。

```
✨ 正在生成你的打卡追踪器...

✅ 已生成文件：E:\Users\你的名字\Desktop\每日打卡追踪器.html

包含功能：
  📝 今日输入面板：心情⭐、专注⏱、运动🏃、喝水🥤
  💾 数据自动保存（存浏览器本地，刷新不丢失）
  📊 本周统计栏（每个指标的本周平均值）
  🎨 深色主题（#1a1a2e底色 + 渐变色卡片）

双击桌面上的文件即可打开使用。
```

**VibeCoding关键点**：
- 豆包生成了完整HTML文件（包含CSS/JS），直接写入桌面
- 不需要用户打开VS Code、不需要npm install、不需要配置。双击就用
- 数据用localStorage持久化

**验证重点**：
- 文件是否真的生成到了正确路径
- 双击是否正常打开
- 4个输入控件是否都能正常交互
- 输入数据后刷新页面，数据是否还在（localStorage生效）
- 深色主题是否真的好看（不是十年前Bootstrap默认白底）

---

### 【轮次2】不满意 → 加视觉要求迭代

**用户说**：
```
出来了但太素了。帮我把视觉效果升级一下：

- 顶部加一个连续打卡天数的大数字，达到7天加火焰动画🔥
- 心情评分不要5个下拉框，换成5个可点击的星星，点第3颗前面3颗都亮
- 专注时长换成可拖动的圆形进度条（0-12小时）
- 运动分钟和喝水杯数改成+/-按钮，点一下数字变
- 整体配色再高级一点，不要太"程序员审美"
```

**预期豆包行为**：

```
✅ 已更新：E:\Users\你的名字\Desktop\每日打卡追踪器.html

升级内容：
  🔥 连续打卡：顶部大数字显示，≥7天触发火焰粒子动画
  ⭐ 星级评分：5颗星可点击交互，选中时星星放大+金色发光
  ⏱ 专注时长：SVG圆环拖动条，拖动时圆环弧长实时变化
  ➕ 计数器：运动/喝水用圆形+/-按钮，按一下数字跳变
  🎨 配色升级：深紫黑底色 + 渐变紫卡片 + 金色强调

请刷新浏览器查看效果。
```

**VibeCoding关键点**：
- 这是vibecoding的核心体验——不是"AI写对代码"，而是"用户可以像跟设计师沟通一样迭代视觉"
- 每轮生成一个完整HTML，用户刷新浏览器就能看到新版本
- localStorage数据在重新打开文件后不丢失（同域名）

**验证重点**：
- 星星点击：点第3颗时是否前3颗都亮、后2颗灭
- 圆环拖动：拖动时数字是否实时更新、圆环弧长是否正确
- 火焰动画：连续7天打卡时是否触发、动画否流畅
- 每次刷新文件（重新生成），旧数据是否还在

---

### 【轮次3】加数据可视化

**用户说**：
```
很不错了。帮我加一个统计页面：

- 本周7天心情评分折线图
- 本周专注时长柱状图
- 本月运动分钟热力图（类似GitHub的Contributions）
- 底部显示"总打卡天数""最长连续打卡""本月平均心情"三个数据卡片

图表要交互的，能hover看具体数值。
```

**预期豆包行为**：

```
✅ 已更新：E:\Users\你的名字\Desktop\每日打卡追踪器.html

新增统计页面（右上角切换按钮切换输入/统计视图）：

  📊 本周心情趋势：折线图（hover显示具体日期和评分）
  📊 本周专注时长：柱状图（hover显示小时数）
  🔥 本月运动热力图：7×4格子，颜色深浅代表运动量
  📇 底部数据卡片：总打卡/最长连续/本月平均心情

  ⚠️ 注意：图表库选择了轻量Canvas自绘（约5KB），
  无需加载外部CDN，文件可离线打开。
```

**VibeCoding 关键点**：
- 豆包需要在"用ECharts CDN（加载慢但图表好看）"和"纯JS自绘（更快但实现复杂）"之间做选择
- 对于本地文件优先选择自绘制：离线可打开、无外部依赖
- 如果要求太复杂的图表（如雷达图），可以建议用CDN

**验证重点**：
- 离线环境（断网）是否仍能正常打开并显示图表
- 热力图7×4格子是否正确展示本月运动数据
- 切换到统计视图再切回输入面板，数据是否一致

---

### 【轮次4】加数据导出/导入

**用户说**：
```
最后一个功能：我想把数据导出备份，或者换电脑后能导进来。
帮我加导出JSON和导入JSON的功能。
```

**预期豆包行为**：

```
✅ 已更新：E:\Users\你的名字\Desktop\每日打卡追踪器.html

新增功能（右下角设置齿轮图标）：
  📤 导出数据：一键下载 JSON 文件到桌面
  📥 导入数据：选择 JSON 文件恢复历史数据
  🗑️ 清空数据：重置所有记录（需二次确认）

备份建议：每周导出一次保存到云盘。
```

**VibeCoding关键点**：
- 数据持久化从"localStorage"升级到"localStorage + JSON文件备份"
- 导入导出是本地的、不经过任何网络——符合隐私需求

## 整体验收清单

| 验证项 | 通过标准 |
|--------|---------|
| 文件生成 | HTML文件正确生成到用户桌面 |
| 零门槛打开 | 双击HTML可直接使用，无需任何依赖（CDN除外） |
| 数据持久化 | 输入数据→关闭浏览器→重新打开HTML→数据还在 |
| 数据不丢失 | 修改HTML（生成新版本）→旧localStorage数据仍然保留 |
| 星星交互 | 点第N颗=前N颗全亮，取消点=全部灭 |
| 圆环拖动 | 拖动圆环时数字实时变化，弧长正确 |
| 火焰动画 | 连续7天打卡时触发，断打卡后熄灭 |
| 统计图表 | 折线图和柱状图正确展示本周数据 |
| 热力图 | 7×4格颜色深浅代表运动量 |
| 导出导入 | 导出JSON→清空数据→导入JSON→数据完全恢复 |
| 离线可用 | 断网后打开HTML所有功能正常（如果有CDN依赖则需注明） |
| 多轮迭代 | 每轮修改后旧数据不丢，功能持续叠加 |

---

## 安全与限制

1. ❌ 生成的可执行文件（.exe/.bat/.ps1）必须让用户确认（HTML是数据文件不是可执行文件，可放心双击）
2. ⚠️ 如果用户要求联网功能（如数据同步云端），必须明确告知"当前是本地版本，数据不上传"
3. ⚠️ localStorage放在浏览器默认路径，清浏览器缓存会丢失——提醒用户定期导出备份

## VibeCoding 与 App Builder 的区分

| 场景 | 用哪个 | 为什么 |
|------|--------|--------|
| "帮我做个打卡工具" | **VibeCoding** | 个人使用、本地数据、工具化、迭代频繁 |
| "帮我做个活动报名页" | App Builder | 多人使用、需要公网链接、部署在云端 |
| "帮我做个销售数据看板" | App Builder | 长期用、多人看、需要URL分享 |
| "帮我写个处理CSV的脚本" | VibeCoding | 本地执行、操作本地文件、一次性任务 |
