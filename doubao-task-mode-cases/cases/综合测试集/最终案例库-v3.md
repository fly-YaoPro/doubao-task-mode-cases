### 案例一：电商·产品图自动处理流水线

**预判路由**：doubao-creative-design → ecommerce-design.md

**为什么匹配**：ecommerce路由明确支持白底主图+场景带入图+材质细节屏。I2I的"保留产品→替换背景"能力是本案例的核心武器。

**阻塞风险**：ecommerce有1轮澄清门控——批量自动化时需验证能否预设参数跳过。

**造假背景**："沫竹Mozoo"日系女装淘宝店，月均上新15款，每款需白底主图+场景图+特写图=4张/款=月均60张。店主阿琳每周四通宵手动做图。

**任务模式差异**：对话模式=每次手动上传1张→描述需求→等出图→下载→命名→归档（×60次/月）。任务模式=拖进文件夹，每周五自动批处理60张，记住品牌模板。

**提示词**：

```
帮我创建一个每周产品图自动处理任务：

【触发】每周五下午2点

【路由】doubao-creative-design → ecommerce-design

【预设参数（跳过澄清）】
- 品牌：沫竹Mozoo，日系简约女装
- 平台：淘宝/天猫，主图1:1白底
- 单次交付：每张原图→出3张（白底主图+场景图+细节特写）
- 风格：暖调、低饱和、呼吸感留白。不用网红滤镜。不用浮夸大标题
- 场景偏好：咖啡馆角落/现代试衣间/窗边自然光，不同款式轮换场景角度

【执行流程】
1. 扫描"E:\沫竹Mozoo\新品图待处理\"，取本周新增的所有产品照片
2. 对每张照片，连续调用image_edit：
   - 图一·白底主图：保留产品颜色/版型/面料→替换背景为纯白(#FFFFFF)→产品居中占85%以上→自然投影→1:1
   - 图二·场景图：保留产品→放入简约现代试衣间场景→窗边自然光→半身镜反射→3:4
   - 图三·细节特写：自动裁切领口/袖口/面料纹理→微距质感→1:1
3. 所有输出按"款号_图类型.png"命名（从文件名提取款号）
4. 保存到"E:\沫竹Mozoo\产品图库\{年月}\"
5. 生成处理报告：总数/成功/失败/异常

【异常处理】
- 如果一张照片里有多件衣服→标记异常跳过
- 如果照片分辨率过低→标记跳过
- 如果某款本周已处理过→跳过不重复
```

**测试关注点**：产品跨图一致性（白底/场景/特写是否同一件衣服）、批量命名准确性、场景不撞图


### 案例二：电商·大促倒计时视频矩阵

**预判路由**：doubao-creative-video → Marketing路线 → product-marketing-ad-video-no-storyboard-ref.md

**为什么匹配**：Marketing路线支持"品牌短片、广告片"。倒计时视频本质是品牌短片+时间信息叠加。

**阻塞风险**：⚠️ Marketing确认门控——每天生成前需输出确认摘要等用户确认。21天=21次确认，不自动。需验证：是否可在任务创建时一次性预设备阶段参数→执行时跳过确认。

**造假背景**："沫竹Mozoo"618预热21天（5.28-6.17），运营小陈去年手动做了21条视频，做到第10天自己都分不清了。她需要21天里视觉有递进、品牌不跑偏、最后3天加重紧迫感的视频矩阵。

**任务模式差异**：对话模式=每天手动写新brief→等确认→出片→下载→发布（×21天）。任务模式=一次配置三阶段策略→每天自动用当天参数跑。

**提示词**：

```
帮我创建618大促21天倒计时视频矩阵：

【路由】doubao-creative-video → Marketing

【预设参数（一次确认，21天复用）】
- 品牌：沫竹Mozoo，日系女装
- 比例：9:16竖版，15秒
- 品牌色：深棕#4A3728、米白#FAF5F0
- 视频调性：温暖安静有质感。不是喊"便宜了"的促销风
- 旁白风格：语速慢，治愈感。每条含3-5句主体旁白+1句slogan
- 产品图：已上传至"E:\沫竹Mozoo\产品图库\"，按产品线轮换（衬衫→裤子→裙子→开衫）

【阶段策略】
第一阶段·种草预热（5.28-6.7，11天）
- 画面：产品场景图+氛围光影+轻旁白
- slogan："还有X天。不急，先看看你喜欢的"
- 每天展示不同产品线，轮换

第二阶段·利益释放（6.8-6.14，7天）
- 画面：产品+价格标签+"全店7折满599减50"信息
- slogan："再等X天。喜欢的衣服现在值得买了"

第三阶段·临门一脚（6.15-6.17，3天）
- 画面：品牌爆款+大号倒计时数字（占画面1/3）+暖调金棕光影
- slogan："还剩X天。你收藏的那件，别让人抢了"

【每日触发】早上7:00
1. 计算当天距离6.18天数
2. 匹配对应阶段策略
3. 调用text_to_video生成当日视频
4. 同步生成三平台发布文案（抖音/小红书/视频号）
5. 保存→推飞书

【异常】如果前一天视频未审核→跳过继续当天
```

**测试关注点**：三阶段视觉过渡是否自然、品牌一致性、天数计算正确性


### 案例三：电商·客服FAQ热点自动生成海报

**预判路由**：doubao-creative-design → media-creative.md

**路由指定**：提示词中明确"生成小红书风格的快问快答海报"，锁定media-creative路由，避免与infographic歧义。

**造假背景**："沫竹Mozoo"客服团队3人，日均200+咨询，30%是重复问题——"会缩水吗""会起球吗""尺码怎么选"。客服每天复制粘贴同样的话，但店里没有任何地方可视化展示这些答案。

**任务模式差异**：对话模式=每次手动输入一个问题→出一张海报。任务模式=每天自动读聊天→统计Top3→出3张海报→推群，已有海报不复用。

**提示词**：

```
帮我创建每日客服FAQ海报自动生成任务：

【路由】doubao-creative-design → media-creative（小红书风格配图）

【预设参数】
- 品牌：沫竹Mozoo，日系女装
- 品牌色：深棕#4A3728、米白#FAF5F0
- 海报风格：小红书种草风——温暖、干净、手写感轻文字、留白30%
- 比例：1:1正方形（适用淘宝详情页+朋友圈）
- 话术风格：亲切专业、像服装顾问、不像淘宝客服

【每日触发】早8:30
1. 读取昨日客服聊天记录 → 统计被问最多的3个问题
2. 检查是否已有该问题的海报在库中 → 有则复用，不重复生成
3. 对每个新问题，调用image_gen生成小红书风格FAQ卡片：
   - 上方2/3：问题大字（如"会缩水吗？"）+简洁回答（3行以内）
   - 下方1/3：产品示意图标+品牌色点缀
4. 3张海报+昨日客服数据摘要（总咨询/Top3问题/平均响应）→打包日报→推飞书

【异常】如果无新高频问题→只推数据日报，跳过出图
```

**测试关注点**：问题提取准确性、FAQ海报复用逻辑、品牌调性一致性


### 案例四：SaaS·版本发布自动公告海报

**预判路由**：doubao-creative-design → media-creative.md（公众号封面赛道）

**为什么匹配**：media-creative的"公众号-产品上新/品牌故事赛道"明确支持"16:9封面+结构化卖点图标+品牌色"，与本案例的"技术产品更新亮点海报"完全对位。

**造假背景**："云策CloudStrategy"企业项目管理SaaS，两周一发版。产品经理小赵每次发版最痛苦的不是code review——而是发版后的配套物料。明明更新了3-5个功能，但要排版成海报+写两套文案（客户群口语版+公众号正式版），每次耗2-3小时。发了30个版本做了30次一模一样的流程。

**任务模式差异**：对话模式=每次手动写文案→描述海报需求→等出图→下载→手动发布。任务模式=自动读发版记录→提取功能→出海报+两套文案→推群。

**提示词**：

```
帮我创建版本发布自动公告任务：

【路由】doubao-creative-design → media-creative（公众号封面）

【预设参数】
- 品牌：云策CloudStrategy，项目管理SaaS
- 品牌色：科技蓝#2563EB、深灰#1E293B
- 海报风格：科技感、干净、参考Notion/Linear发版风格
- 比例：16:9横版（公众号封面）
- 文案风格：客户群版用"这次更新了三件事……"开头（口语），公众号版用小标题+结构化

【触发】每月1号和15号下午4点
1. 从飞书项目/TAPD读取"本次发布"需求列表
2. 提取功能名+一句话描述+面向用户群
3. 调用image_gen生成更新亮点海报：
   - 标题："云策 V{版本号} 更新"
   - 3-5个功能用卡片式Bento布局
   - 品牌色+留白+科技感
4. 生成两套发布文案（客户群口语版+公众号正式版）
5. 海报+文案打包→推飞书群"云策·产品发布"

【异常】如果本次只有Bug修复无新功能→生成简化版"维护更新"海报
```

**测试关注点**：功能提取准确性、文案差异化（口语vs正式）、版本号连续性


### 案例五：HR·新员工入职物料自动生成

**预判路由**：doubao-creative-design → media-creative（欢迎海报+名片图）+ graphic-infographic（入职指南），分两步执行避免路由冲突。

**造假背景**：某100人科技公司行政小雨。每周1-3个新人入职。需要做欢迎海报（发全员群）、名片图（发部门群）、入职指南长图。每周一模一样的事——换人名+照片+部门+日期——但每周手动做。忘过一次：周一新人入职，全员群没有欢迎海报，显得公司不重视。

**任务模式差异**：对话模式=每次手动输入新人信息→一张张出图。任务模式=每周五自动读入职表→为下周新人自动生成全套物料→打包推HR群+创建待办提醒。

**提示词**：

```
帮我创建每周新员工入职物料自动生成任务：

【路由】
Step 1：doubao-creative-design → media-creative（欢迎海报+名片图）
Step 2：doubao-creative-design → infographic（入职指南长图）

【预设参数】
- 公司品牌色：深蓝#1a1a2e、亮橙#e85d04
- 欢迎海报风格：科技公司、年轻活力、9:16竖版
- 名片图风格：简洁信息卡、1:1正方、暖色背景
- 入职指南内容（固定模板）：Wi-Fi密码/打印机位置/食堂推荐/会议室预定/HR联系方式

【触发】每周五下午5点
1. 从飞书表格"员工入职表"读取下周入职的所有新人信息
2. Step 1·调用media-creative，对每个新人生成：
   - 欢迎海报（全员群）：新人照片+姓名+职位+部门+入职日期
   - 名片图（部门群）：姓名+英文名+一句话介绍+飞书二维码
3. Step 2·调用infographic，生成一份入职指南长图（固定内容，仅改日期）
4. 按"新人姓名"分文件夹打包→推飞书群"HR·入职准备"
5. 生成待办："周一早9点发欢迎海报到全员群"

【异常】如果某新人无照片→用姓名首字母生成简洁头像占位
```

**测试关注点**：飞书表格读取准确性、海报个性化、入职指南固定内容正确、多新人独立打包


### 案例六：电商·直播后爆款UGC种草视频自动生成（修正版）

**预判路由**：doubao-creative-video → UGC路线 → ugc-talking-video-ref.md

**原案问题**：初版设计"从3小时直播回放提取高光片段"——video skill没有视频理解/剪辑能力，不可行。

**修正方案**：改为"下播后读销售数据→取今日爆款Top3→为每款从0生成一条15秒UGC口播种草视频"。完全在UGC路线能力边界内。

**造假背景**："沫竹Mozoo"直播间，主播小鹿每天播3小时展示15-20款。下播后需要短视频素材投流——不投就没有新流量。但她没时间剪。

**任务模式差异**：对话模式=手动为每款输入brief→生成→下载→发布。任务模式=自动读今日销售数据→取Top3→各生成一条→命名归档+推送。主播模板自动轮换不撞脸。

**提示词**：

```
帮我创建直播后爆款UGC种草视频自动生成任务：

【路由】doubao-creative-video → UGC

【预设参数】
- 品牌：沫竹Mozoo，日系女装
- 视频：15秒、9:16竖版
- 风格：真实UGC博主推荐感——像朋友种草，不像广告
- 主播模板：5模板轮换，跨日不撞脸
- 产品外观：基于产品图库中对应SKU参考图锁定，不重绘
- 口播风格：第一人称体验式（"我自己穿了一周发现……"），去AI腔
- 平台：抖音+小红书双平台语气

【触发】每天下午5:30（直播结束后）
1. 从飞书表格"沫竹直播间·每日销售"读取今日销量Top3产品
2. 提取每个产品的：款号、名称、颜色、核心卖点、今日直播中主播强调的卖点句
3. 对每个产品调用UGC路线生成一条15秒口播视频：
   - 0-3s：产品入画+痛点hook（"通勤穿什么最头疼……"）
   - 3-7s：产品展示+面料特写
   - 7-12s：试穿效果+具体体感（"这条裤子穿上就舍不得脱"）
   - 12-15s：推荐收束+软CTA
4. 主播模板：从T1-T5中取当日未使用的模板
5. 按"产品名_日期_T{模板号}"命名
6. 保存到"E:\沫竹Mozoo\投放素材库\直播短视频\"
7. 生成切片清单→推飞书

【异常】如果今日销售数据未更新→推送提醒，不执行
```

**测试关注点**：产品一致性（跨视频的外观保真）、主播去重、脚本去AI腔、Top3提取准确


### 案例七：跨境电商·多语种产品视频独立生成（修正版）

**预判路由**：doubao-creative-video → UGC路线 → ugc-talking-video-ref.md

**原案问题**：初版设计"基于已有中文视频，翻译配音+替换字幕"——video skill不能编辑已有视频，不可行。

**修正方案**：改为"基于中文产品brief，分别生成印尼/泰/英三个语种的独立15秒UGC带货视频"。每条是独立生成的视频，但产品外观基于同一参考图锁定。

**造假背景**："赛音Saiyin"蓝牙耳机工厂做跨境电商，Shopee（东南亚）+Amazon（美国）+速卖通。国内视频只有中文字幕，东南亚退货率15%——用户看了视频买回去发现不会用，因为看不懂中英文。

**任务模式差异**：对话模式=每次手动输入一个语种的brief→生成→下载→重复三次。任务模式=每周自动读取本周主推产品→生成三个语种版本→按市场归档。

**提示词**：

```
帮我创建多语种产品视频自动生成任务：

【路由】doubao-creative-video → UGC（每个语种独立调用）

【预设参数】
- 品牌：Saiyin（赛音）
- 产品：T10真无线降噪蓝牙耳机
- 产品外观参考：已上传产品图库，所有视频基于同一参考图锁定外观
- 视频：15秒、9:16竖版
- 产品核心卖点（保持原文）：ANC主动降噪/30小时续航/蓝牙5.4/IPX5防水
- 品牌名"Saiyin"和型号"T10"在所有语种中保持原文不翻译

【触发】每周一上午10:00
1. 读取本周主推产品列表（从飞书表格）
2. 对每个产品，分别生成三个语种版本：

   印尼语版 → TikTok Shop印尼风格
   - 主播：东南亚审美素人
   - 口播：印尼语，口语化亲切
   
   泰语版 → Lazada泰国风格
   - 主播：泰国审美素人
   - 口播：泰语，有礼貌但不生硬
   
   英语版 → Amazon/TikTok Global风格
   - 主播：欧美/Caucasian审美素人
   - 口播：英语，自然Tech Review感

3. 每个语种版的安利脚本从中文卖点翻译但做口语化改写（不机翻）
4. 按"产品_语种_版本号"命名（如"T10_ID_001.mp4"）
5. 分别保存到"E:\Saiyin\产品视频\{语种}\"
6. 生成周报→推飞书

【异常】如果某语种翻译质量明显差→标记为需人工复核
```

**测试关注点**：产品外观跨语种保真、翻译口语化程度、品牌名/参数锁定、主播审美匹配当地市场


### 案例八：电商·主图AB测试自动轮换

**预判路由**：doubao-creative-design → ecommerce-design.md

**为什么匹配**：ecommerce路由的主图系列明确支持场景带入图+卖点解析图。两个变体方向（场景痛点型/数据信任型）完全在路由覆盖内。

**阻塞风险**：⚠️ "自动替换天猫后台主图"不是design skill能力——需任务模式的浏览器操作。"读取后台点击率数据"同理。

**造假背景**：天猫家电代理商运营小林，主力吸尘器主图用了半年——白底+产品+"15KPa大吸力"。她一直想测场景图vs数据图哪个点击率高，但没时间做图+替换+跟踪数据。

**任务模式差异**：对话模式=手动做一张图→手动替换→一周后手动看数据（可能忘了）。任务模式=每周自动生成两张变体→按计划轮换→自动读数据→出分析周报。

**提示词**：

```
帮我创建主图AB测试自动轮换任务：

【路由】doubao-creative-design → ecommerce-design

【预设参数】
- 当前主力SKU：轻量无线吸尘器M200
- 平台：天猫主图，1:1
- 现主图：白底+产品+"15KPa大吸力·45分钟续航"（对照组）

【触发】每周一早8:00
1. 读取上周天猫后台主图点击率数据
2. 对每个主力SKU，调用image_gen生成两张新主图变体：

   变体A·场景痛点型
   - 产品在真实场景中（吸尘器吸沙发猫毛）
   - 卖点：感性+具体利益，如"养猫家庭的救星·每天吸出一只猫"
   
   变体B·数据信任型
   - 白底+产品+数据可视化图表（吸力对比柱状图）
   - 卖点：理性+权威感，如"真实用户测试·吸力保留率96%"

3. 制定轮换计划：
   周一：现主图（对照组）
   周二-周四：变体A
   周五-周日：变体B
4. 提醒手动替换主图（或通过浏览器操作自动替换）
5. 每周日生成测试周报：
   - 三张主图各自点击率/转化率/加购率对比
   - 统计显著性判断
   - 胜出版本建议→设为默认

【异常】如果数据量不足以判断→自动延长测试周期
```

**测试关注点**：主图创意差异化、数据读取准确性、统计显著性判断


### 案例九：品牌·每周社媒视觉内容日历

**预判路由**：doubao-creative-design → media-creative.md

**为什么匹配**：media-creative覆盖小红书(3:4)+朋友圈(1:1)+公众号(16:9)三个平台，每个有独立的视觉指纹和排版规范。9张图分三组建模，避免跨平台视觉混淆。

**造假背景**："唤醒WakeUp"冷萃咖啡液品牌，创始人小陈一个人的团队。每周日晚从9点做到凌晨2点做下周7天的社媒图——小红书3条+朋友圈5条+公众号1篇=9张。做完最后一张已经忘了第一张长啥样。

**任务模式差异**：对话模式=每张图手动描述→等出图→下载→排期。任务模式=每周日自动生9张→按平台分组→打包推手机，记住上周用了什么角度这周轮换。

**提示词**：

```
帮我创建每周社媒视觉内容日历自动生成任务：

【路由】doubao-creative-design → media-creative
（注意：分三组建模——小红书组→朋友圈组→公众号组，各组独立保持视觉一致性）

【预设参数】
- 品牌：唤醒WakeUp，冷萃咖啡液
- 品牌色：活力橙#FF6B35、米白#FAFAF5、深灰#2D2D2D
- 品牌调性：温暖、年轻、有能量但不浮躁
- 内容结构（固定，每周复用）：
  周一·小红书（3:4）——产品种草
  周三·小红书（3:4）——生活方式
  周五·小红书+朋友圈（1:1）——福利互动
  每天·朋友圈（1:1，共5条）——产品日常/使用场景/咖啡知识/用户晒图/周末预告
  周日·公众号封面（16:9）——咖啡灵感/新品

【触发】每周日下午4:00
1. 读取上周内容角度清单→本周自动轮换（不重复角度）
2. 分三组生成：
   小红书组（3张）：调用image_gen，生活化场景+品牌色点缀+手写感轻文字
   朋友圈组（5张）：调用image_gen，更随意更像随手拍、轻修图感
   公众号组（1张）：调用image_gen，品牌设计感、大留白、强文字对比
3. 所有图保持品牌色+logo统一
4. 打包为"下周内容日历"文件夹，按日期+平台命名
5. 推送飞书："下周视觉日历已生成，共9张。本周角度：{本周}。上周角度：{上周}"
6. 同步推手机（可直接从手机保存发）

【异常】如果下周有节日→优先级调整做节日内容
```

**测试关注点**：三平台风格差异、品牌色跨组建一致性、角度轮换正确

**任务模式差异**：对话模式=每次手动排版1-2小时。任务模式=自动读素材→记住客户品牌色→一键生成对比展示页。

```
帮我创建客户多方案对比展示页自动生成任务：

- 结构模板（固定，每次复用）：
  屏1·封面：客户品牌名+项目类型+日期+事务所logo

3. 按结构模板逐屏调用image_gen：
   - 封面屏：客户品牌名+项目类型+事务所logo
   - 对比总览屏：各方案并排+标注
   - 详解屏（每个方案独立一屏）
   - 末屏：下一步信息
4. 打包输出→推送到客户提案群

【异常】如果只有1个方案→改为单方案深度展示（不是对比结构）
```

**测试关注点**：客户信息跨次记忆、方案布局清晰度、多方案差异可辨


### 案例十一：品牌·节日营销视频提前排期

**预判路由**：doubao-creative-video → Marketing路线 → product-marketing-ad-video-no-storyboard-ref.md

**任务模式差异**：不是"每次大促临时做视频"，而是"年初配置全年节日策略→到点自动触发一个提前两周开始的排期流程（出脚本→出预览→出正式→发布）"。

```
- 七夕（8.29）："穿去约会的裙子"主题
- 中秋节（9.25）："团圆日的温暖穿搭"
- 双11（11.11）："全年最低价"促销
- 双12（12.12）："年末犒赏自己"
- 年货节（2027.1月）："新年新衣"

【排期流程（每个节日自动执行）】
T-14天·自动触发
→ 读取节日主题
→ 生成视频脚本+分镜方案（文案三版：抖音/小红书/视频号）
→ 推飞书待审核

T-7天·自动触发（基于T-14审核通过的版本）
→ 生成预览视频（带预览水印）
→ 推飞书终审

T-0天·节日当天早8:00
→ 生成正式版（去除水印）
→ 自动推送到各平台（或生成可直接发布的终版+文案）

```

**测试关注点**：节日识别准确性、脚本贴合品牌调性、三平台文案差异化、排期触发的准时性


### 案例十二：电商·竞品定价周度对比看板

**任务模式差异**：对话模式=每次手动打开竞品页面→手动记录→手动对比。任务模式=每周一早9点自动打开5个竞品店→提取价格→环比上周→生成看板→推群。
