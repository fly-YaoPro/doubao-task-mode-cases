# doubao-finance-sector Skill 测试案例集

> 生成时间：2026-06-29
> Skill本质：双轨量化板块热度分析（信息热×行情热）+ 两阶段交付（文字分析→交互仪表盘）
> 核心差异：不回答"这个板块好不好"，只回答"现在有多热、为什么、接下来盯什么"
> 独特机制：双轨打分（1-5）+ 背驰检测（四象限）+ 10只代表股 + 脚本流水线 + HTML仪表盘
> 任务模式价值：定期追踪板块轮动变化——周度/月度生成更新的热度分析

## 案例一：AI/人工智能板块热度分析（核心案例·高热场景）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **触发条件** | 用户问"XX板块现在热不热""能不能追""为什么涨""怎么看"时触发 |
| **目标人群** | 关注A股板块轮动的投资者、需要快速了解一个板块当前状态的人 |
| **核心痛点** | 散户看板块热度靠"听消息看涨跌"——涨了就觉得热、跌了就觉得冷。但"涨了一周但没放量"和"涨了一天但天量成交"是完全不同的热度质量。一个人无法同时盯10只代表股的价格/成交/估值/催化事件，更无法判断"信息热度"和"行情热度"是否一致 |
| **不可替代性** | 双轨打分+背驰检测——行情热5分但信息只有2分=纯资金驱动，持续性存疑。这种判断需要同时查10只股票的数据+归因催化事件+对比两个维度，人工做一次至少3小时 |
| **一句话传播** | "别人还在看涨跌，AI帮你把板块热度拆成信息热和行情热两条轨道——信息也热、行情也热才是真热；只有一边热就是虚火" |

### 二、测试提示词

```
帮我分析一下AI/人工智能板块现在热不热，能不能追。
重点看看最近一周的行情表现和有什么催化事件。
```

（如果只触发回答了行情但没触发skill流程，补一句：）
```
能不能再详细一点，帮我做个板块热度分析？我想知道热度的持续性怎么样。
```

### 三、预期Skill路由

```
用户prompt → 触发识别：板块/概念热度分析 → 启动skill
↓
第一阶段启动后读取：
  1. data_contract.md（数据源分级）
  2. facts_contract.md（10股+7日字段）
  3. scoring_and_divergence.md（双轨打分+背驰）
↓
建facts.json：
  → general_search 形成归因假设（3-5条）
  → 确定T0（最近已收盘交易日）
  → seed_finance_search 搜索板块指数+10只代表股原始行情
  → 填写meta/sector_checks/stock_checks/facts原始字段
  → derive_facts.py 计算衍生字段
↓
精选催化：为3-5条催化找来源URL（一级/二级）
↓
lint_analysis.py --strict → 修复 → 再跑
↓
读chat_contract.md + chat_example.md → 写正文章稿
↓
lint_analysis.py --strict 草稿+facts → 修到0错误
↓
输出完整分析（6模块+固定结尾）
```

### 四、测试关注点

| # | 关注点 | 通过标准 | 潜在风险 |
|---|--------|---------|---------|
| 1 | **触发准确性** | 问"热不热""能不能追""怎么看"触发skill；问"什么是AI板块""AI行业前景"不触发 | 行业趋势类问题误触发 |
| 2 | **T0确定** | 使用最近已收盘交易日，未收盘不用当日盘中的数据 | 用盘中实时数据冒充T0收盘价 |
| 3 | **10股提取** | 从seed_finance_search获取10只代表股，不是从记忆/新闻中拼凑 | 手动凑10只股名而不查行情 |
| 4 | **双轨打分** | 给出行情热度和信息热度各自的1-5分+理由，综合分=round(行情×0.55+信息×0.45) | 只给一个综合分，或权重计算错误 |
| 5 | **背驰检测** | 比较信息热vs行情热，输出四象限判断（双热/纯资金/题材未启动/双冷） | 跳过了背驰检测这最关键一步 |
| 6 | **四个维度表格** | 模块1必须有价格涨跌/成交量能/代表股表现/估值位置四行表格 | 表格缺失某维度或用10股表现代替板块级数据 |
| 7 | **催化归因** | 模块2每条催化=事实+为什么重要+后续验证，含可访问URL | 催化只有标题没有URL，或用了自媒体报道 |
| 8 | **固定结尾** | 必须以"下一步是否为您生成完整的板块热度仪表盘？如需请回复生成HTML"结尾 | 结尾缺失或措辞不对 |
| 9 | **风险提醒** | 第一行必须输出固定风险提醒文字 | 遗漏或修改了固定风险提醒措辞 |


## 案例二：消费电子板块降温分析（冷清场景）

### 一、案例设计

> 测"冷"的边界——当板块确实没热度时，skill能不能诚实地说"不热"，而不是为了凑内容硬找出几条似有若无的催化

### 二、测试提示词

```
最近消费电子板块好像没什么动静，帮我看看现在到底冷不冷，为什么没人关注了。
```

### 三、测试关注点（在案例一基础上新增）

| 关注点 | 通过标准 | 潜在风险 |
|--------|---------|---------|
| **诚实评分** | 行情和信息热都给出1-2分（冷清/温和），综合分不超过2 | 怕"没内容"而硬拉到3分 |
| **冷的原因分析** | 明确说清"为什么冷"——不是没催化，就是资金在别处/估值不吸引/无新故事 | 只说"没有催化事件"过于表面 |
| **不放水填催化** | 模块2的催化列表可以为空或仅1-2条，不硬塞无关新闻 | 为凑数塞入"行业日常新闻" |


## 案例三：信息热·行情冷背驰检测（题材未启动）

### 一、案例设计

> 测四象限判断中最关键的一个——利好满天飞但股价没动，到底是"还没启动"还是"利好出尽"？

### 二、测试提示词

```
最近低空经济/飞行汽车政策消息很多，但股票好像没怎么涨。
帮我看看这个板块的热度，是不是消息和行情脱节了？
```

### 三、测试关注点（新增）

| 关注点 | 通过标准 | 潜在风险 |
|--------|---------|---------|
| **背驰类型正确** | 输出"信息热·行情冷 → 题材未启动或利好兑现回落" | 没做背驰判断 |
| **持续性判断** | 明确说"等行情跟上才能确认"还是"利好已充分定价，偏风险" | 只描述不判断 |


## 案例四：行情热·信息冷背驰检测（纯资金驱动）

### 一、案例设计

> 测另一种背驰——股价猛涨但找不到靠谱催化，纯买盘驱动的风险提示

### 二、测试提示词

```
最近ST板块莫名其妙涨了好多，但没看到什么利好消息。帮我分析一下这个热度靠谱吗？
```

### 三、测试关注点

| 关注点 | 通过标准 |
|--------|---------|
| **风险提示强度** | "纯筹码/资金驱动，缺少基本面催化的持续性存疑，一旦驱动逻辑反转易回撤" |
| **信息来源标注** | 如果ST板块确实没有可靠催化，要诚实标注"未找到一级/二级催化来源" |


## 案例五：小众新概念板块分析（边界场景）

### 一、案例设计

> 测数据稀缺时的降级行为——一个小众概念可能连10只代表股都凑不齐

### 二、测试提示词

```
帮我分析一下"AI agent"这个概念板块的热度。
A股好像没有专门的AI agent板块指数，但最近讨论挺多的。
```

### 三、测试关注点

| 关注点 | 通过标准 | 潜在风险 |
|--------|---------|---------|
| **数据不足声明** | 如果确实没有对应板块指数代码，诚实说明且用相近板块替代 | 编造一个不存在的板块代码 |
| **代表股提取** | 尽可能找相关标的，不足10只时说明原因 | 找不相关的股票硬凑10只 |
| **评分保守** | 数据不完整时降低评分置信度 | 缺数据也能打4-5分 |


## 案例六：板块轮动监控（任务模式·周期追踪）

### 一、案例设计

| 维度 | 内容 |
|------|------|
| **任务模式价值** | 每周一自动对3-5个板块做热度扫描，横向对比"本周哪个板块最热"，纵向对比"AI板块这周比上周热了还是冷了" |
| **一句话传播** | "以后每周一早上，AI帮你把几个板块的热度全扫一遍——谁在升温、谁在退潮、谁的行情和信息在背驰，一张表看清" |

### 二、测试提示词

```
以后每周一帮我做一次板块热度扫描，重点看这几个：
- AI/人工智能
- 新能源车
- 半导体
- 消费电子

每次帮我横向对比哪个最热、哪个最冷，然后纵向和上周对比变化。
这次先跑一次看看效果。
```

### 三、测试关注点

| 关注点 | 说明 |
|--------|------|
| **横向对比** | 能否在单次报告中对比4个板块的相对热度 |
| **纵向对比** | 如果已有上周数据，能否标出"较上周热度上升/下降" |
| **稳定性** | 每周使用相同的10只代表股口径（不随意更换对比基准） |


## 案例七：HTML仪表盘生成（第二阶段）

### 一、案例设计

> 测两阶段交付的切换——第一阶段文字分析完成后，用户回复"生成HTML"才进入第二阶段

### 二、操作步骤

1. 先完成案例一的第一阶段（文字分析）
2. 看到第一阶段的固定结尾"下一步是否为您生成完整的板块热度仪表盘？如需请回复生成HTML"
3. 回复：`生成HTML`

### 三、测试关注点

| 关注点 | 通过标准 | 潜在风险 |
|--------|---------|---------|
| **阶段切换** | 第一阶段不主动生成HTML，只有用户明确回复后才进入第二阶段 | 第一阶段就直接渲染HTML |
| **payload生成** | 基于facts.json生成payload.json，通过hydrate填充计算字段 | 跳过payload直接在HTML里硬编码 |
| **双轨仪表盘** | HTML中必须展示两条热度轨道（信息热/行情热）的仪表盘可视化 | 只有一个综合分数字 |
| **背驰矩阵** | HTML中必须包含四象限背驰矩阵图表 | 遗漏背驰可视化 |
| **交付方式** | FileBatchUpload → 公网链接 → 附件传给app_builder_agent → 用arch_type=html渲染 | 直接输出HTML在聊天里 |
| **免责声明** | HTML有自己的独立免责声明（不搬运第一阶段的固定风险提醒到HTML文本字段中） | 把风险提醒塞进headline/summary |

## 测试执行建议

### 推荐测试顺序

**案例一（AI板块）→ 案例八（不应触发）→ 案例三（背驰·题材未启动）→ 案例四（背驰·纯资金）→ 案例二（冷清板块）→ 案例五（小众概念）→ 案例七（HTML阶段）→ 案例六（任务模式）**

### 测试环境要求

⚠️ **真实行情数据**：本skill强制使用`seed_finance_search`获取真实行情。测试时需确保该工具可用且网络畅通。不能使用模拟/伪造行情数据——lint会报错。

⚠️ **脚本依赖**：测试前确认Python环境可用，`scripts/`目录下所有脚本可执行。

### 核心红线（违反任何一条即为测试失败）

1. 行情数据未从seed_finance_search获取（使用估计/记忆/新闻摘要代替）
2. 未做双轨打分或跳过了背驰检测
3. 第一阶段在用户未回复时主动生成了HTML
4. 综合分权重计算错误（非行情×0.55+信息×0.45）
5. 第一行未输出或修改了固定风险提醒文字
